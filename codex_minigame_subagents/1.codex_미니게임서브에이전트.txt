미니게임에 필요한 서브에이전트 toml을 
받아서 모아놓는다.

게임을 만들때 마다 폴더안에 .codex폴더를 만들고
서브에이전트 toml을 모두 때려넣어놓는다.

이제 만들고 있는 미니게임에 필요한 서브에이전트를
사용하면 끝이다.

명시적으로 서브에이전트를 쓰라고 해야한다.
반드시 언급한다.
그러니까 해당하는 서브에이전트 이름을 명시하면서 명령한다.

혼자 개발 → 팀처럼 만들기

👉 구조의 시작 (제일 중요)

API 구조 설계
클라이언트 ↔ 서버 인터페이스 정의

💡 왜 중요:
👉 구조 잘못 잡으면 나중에 다 갈아엎음

👉 서버 핵심

점수 저장
매칭 / 턴 처리
DB 설계

👉 서버 로직 전부 담당

3️⃣ frontend-developer

👉 실제 게임 화면 담당

UI / 인터랙션
클릭, 입력, 애니메이션

👉 네 미니게임이면 필수

4️⃣ websocket-engineer

👉 실시간 게임 핵심

실시간 통신
PvP 동기화

👉 턴제 / 멀티 있으면 무조건 필요

5️⃣ fullstack-developer

👉 “다 알아서 해주는 범용 개발자”

프론트 + 백 둘 다 처리

💡 초반 MVP 만들 때 최고

6️⃣ ui-designer

👉 게임 느낌 살리는 역할

UI 스타일
버튼 / 레이아웃 / UX

👉 그냥 기능만 있으면 재미 없음

7️⃣ reviewer

👉 품질 담당 (진짜 중요)

코드 검토
버그 탐지
구조 문제 찾기

👉 이거 없으면 망함 (진짜)

8️⃣ security-auditor

👉 치트 방지

해킹 방지
점수 조작 검증

💡 PvP 게임이면 필수

👉 확장성 담당

서버 구조 분리
트래픽 대응
