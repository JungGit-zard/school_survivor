# Capacitor Android Firebase Google login fails in Play AAB

## Problem

Google login in the Play-distributed Android AAB can fail if the app uses Firebase Web SDK popup/redirect auth inside a Capacitor WebView. The user sees a blank browser page, localhost connection failure, or Firebase redirect state errors instead of returning to the game.

## Symptoms

- Firebase auth handler says the initial state is missing.
- External Chrome opens `localhost` and shows `ERR_CONNECTION_REFUSED`.
- Login works differently between local web/dev builds and Play internal testing AABs.

## What Didn't Work

- Switching between Firebase Web SDK `signInWithRedirect()` and `signInWithPopup()` is not enough for the AAB.
- Sending the user to the Play Integrity API setup page does not help. That page is unrelated to Firebase Google Sign-In SHA configuration.
- Using only the existing Firebase web app config is not enough. The Firebase project must also have an Android app registered for `com.jungyoon.zombieschool`.

## Solution

Use native Google sign-in for Capacitor Android and then bridge the token into Firebase JS auth:

1. Add `@capacitor-firebase/authentication`.
2. Configure Capacitor:

```json
{
  "plugins": {
    "FirebaseAuthentication": {
      "skipNativeAuth": true,
      "providers": ["google.com"]
    }
  }
}
```

3. In the app auth client, use native Google login only on Capacitor Android/iOS shells:

```js
const result = await FirebaseAuthentication.signInWithGoogle({ skipNativeAuth: true })
const credential = GoogleAuthProvider.credential(result.credential.idToken, result.credential.accessToken)
await signInWithCredential(auth, credential)
```

4. In `android/variables.gradle`, include Google support for the plugin:

```gradle
rgcfaIncludeGoogle = true
```

5. In Firebase Console, add an Android app with package:

```text
com.jungyoon.zombieschool
```

6. Register Play App Signing SHA-1 and SHA-256 on that Firebase Android app.

The known Play Console path for this project is:

```text
Google Play Console
-> select app
-> Google Play로 보호됨
-> Playstore 보호
-> expand with the right arrow
-> 앱 서명 키 보호
-> Play 앱 서명 관리
-> 앱 서명 키 인증서
```

Use the SHA-1 and SHA-256 from `앱 서명 키 인증서`. Do not use the Play Integrity API setup page.

7. After SHA registration, download a fresh `google-services.json` from the Firebase Android app and place it here:

```text
Developer/r3f_prototype/android/app/google-services.json
```

The correct file must contain both:

- Android OAuth client for `com.jungyoon.zombieschool`
- Web client, which generates `default_web_client_id`

## Why This Works

Capacitor Android runs the app in an internal WebView/local server. When OAuth escapes to external Chrome, Chrome's `localhost` is not the app's local server, so the Firebase Web redirect state cannot reliably return to the game. Native Google sign-in avoids that browser redirect path and gives the app an ID token directly. Firebase JS `signInWithCredential()` then creates the normal Firebase user state used by ranking, coins, nickname, and cloud progress.

The refreshed `google-services.json` matters because Android Google sign-in reads `default_web_client_id` from generated Android resources. That resource is generated by the Google Services Gradle plugin from `android/app/google-services.json`.

## Prevention

- For any future "Google login fails in AAB/mobile" report, check this solution first before trying popup/redirect changes.
- Verify the Firebase Android app exists and uses package `com.jungyoon.zombieschool`.
- Verify Play App Signing SHA-1 and SHA-256 are registered in Firebase.
- Re-download `google-services.json` after SHA changes.
- Verify with:

```powershell
npm run build
npx vitest run src/lib/firebaseAuth.test.js src/store/useAuthStore.cloudProgress.test.js --pool=threads --maxWorkers=1 --no-fileParallelism
npx cap sync android
$env:JAVA_HOME='C:\Program Files\Android\Android Studio\jbr'; $env:Path="$env:JAVA_HOME\bin;$env:Path"; .\gradlew.bat assembleDebug
```

- Confirm Gradle runs `:app:processDebugGoogleServices`.
- Confirm generated resource names include `default_web_client_id`, `google_app_id`, and `project_id`.
