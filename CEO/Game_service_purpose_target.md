# BangBang Survivor Service Purpose and Target

## 1. 게임이 제공하고자 하는 것

BangBang Survivor는 "좀비에 감염된 학교에서 빠져나가려는 주인공"의 5분 생존 경험을 제공하는 모바일 세로형 서바이버라이크 게임이다.

이 게임이 제공해야 하는 핵심 가치는 다음과 같다.

### 1-1. 짧고 명확한 생존 몰입

- 한 판은 5분으로 짧게 끝난다.
- 시작 30초는 조작 적응 구간이다.
- 1분 30초 이후부터 압박이 시작된다.
- 3분 이후에는 플레이어가 선택한 빌드의 효과가 체감된다.
- 5분 시점에는 교장 감염체 보스 처치 또는 스테이지 클리어로 명확하게 마무리된다.

즉, 플레이어는 짧은 시간 안에 "도망친다, 버틴다, 강해진다, 보스를 넘는다"는 흐름을 느껴야 한다.

### 1-2. 초보자도 이해하기 쉬운 조작

- 플레이어는 이동만 직접 조작한다.
- 공격은 자동으로 이루어진다.
- 레벨업 시 3개 선택지 중 하나를 고른다.

조작 부담을 줄이고, 재미의 중심을 "어디로 움직일지", "어떤 업그레이드를 고를지", "위험한 적을 어떻게 피할지"에 둔다.

### 1-3. 감염된 학교라는 익숙하지만 위험한 무대

게임의 무대는 일반적인 판타지 던전이 아니라 학교다.

제공해야 하는 감각:

- 운동장, 본관, 교실 물건, 사물함, 책상 같은 익숙한 학교 요소
- 감염 경고 테이프, 오염 웅덩이, 감염체 접근으로 생기는 긴장감
- 학생 생존자가 학교 장비로 버티는 카툰 액션 감각

이 게임은 실사 공포보다 "카툰풍 감염 학교 탈출 액션"에 가깝다.

### 1-4. 성장 선택의 재미

플레이어는 생존 중 레벨업을 통해 무기와 패시브를 선택한다.

핵심 선택:

- 연필 투척: 단일 대상과 보스 대응
- 책가방 휘두르기: 근접 방어와 소형 감염 학생 떼 대응
- 비상벨 충격파: 화면 정리와 다방향 압박 완화
- 전기 스턴건: 중형 감염체와 엘리트 대응

플레이어는 매 판마다 어떤 장비를 강화할지 고르며, 자신의 생존 방식이 달라진다고 느껴야 한다.

### 1-5. 읽기 쉬운 카툰 액션

시각 방향은 어둡고 복잡한 공포가 아니라 읽기 쉬운 카툰 액션이다.

중요 기준:

- 플레이어와 감염체의 실루엣이 모바일 화면에서 잘 보여야 한다.
- 적 접근 방향, 투사체, 장판, XP 오브, 회복 오브가 서로 헷갈리지 않아야 한다.
- Three.js 카툰 렌더링과 외곽선을 사용해 캐릭터 가독성을 높인다.
- Phaser는 2D 월드, UI, 이펙트, 충돌, 투사체를 안정적으로 관리한다.

## 2. 타겟 유저층

### 2-1. 1차 타겟

1차 타겟은 짧은 시간 동안 부담 없이 성장형 액션을 즐기고 싶은 모바일 캐주얼 액션 유저다.

특징:

- 5분 안팎의 짧은 플레이를 선호한다.
- 복잡한 조작보다 이동 중심 조작을 선호한다.
- 레벨업 선택, 무기 강화, 자동 공격이 주는 성장감을 좋아한다.
- 뱀서라이크, 서바이버라이크, 방치형 액션의 문법에 익숙하거나 쉽게 적응할 수 있다.

### 2-2. 2차 타겟

2차 타겟은 학교 탈출, 좀비 감염, 카툰 액션 같은 콘셉트를 좋아하는 라이트 게이머다.

특징:

- 공포 분위기에는 흥미가 있지만 너무 잔혹하거나 어두운 게임은 부담스러워한다.
- 캐릭터와 몬스터가 귀엽고 명확하게 보이는 카툰 그래픽을 선호한다.
- 한 판 안에서 시작, 위기, 성장, 보스 마무리가 분명한 게임을 선호한다.

### 2-3. 제외하거나 후순위로 둘 유저층

초기 버전에서는 아래 유저층을 핵심 타겟으로 잡지 않는다.

- 깊은 스토리 RPG를 기대하는 유저
- 고난도 수동 조작 액션을 기대하는 유저
- 실사 공포와 잔혹한 좀비 표현을 기대하는 유저
- 장시간 파밍과 복잡한 장비 세팅을 기대하는 하드코어 유저
- 멀티플레이 또는 온라인 경쟁을 핵심으로 기대하는 유저

## 3. 구현방법론

### 3-1. 첫 목표는 완성된 5분 플레이 루프

처음부터 많은 기능을 만들지 않는다.

첫 개발 목표:

1. 플레이어가 움직인다.
2. 감염된 학생이 추적한다.
3. 플레이어가 자동으로 연필을 던진다.
4. 적이 죽고 XP 오브를 떨어뜨린다.
5. XP를 먹으면 레벨업한다.
6. 업그레이드 3개 중 하나를 선택한다.
7. 시간이 흐를수록 웨이브가 강해진다.
8. 5분에 교장 감염체가 등장한다.
9. 보스를 처치하면 클리어된다.

이 기본 루프가 재미있고 안정적으로 돌아간 뒤 그래픽, 이펙트, 추가 몬스터, 고급 연출을 확장한다.

### 3-2. Phaser 4와 Three.js의 역할을 분리한다

Phaser 4는 게임의 실제 규칙을 담당한다.

Phaser 4 담당:

- 씬
- 입력
- 카메라
- 타일맵
- 충돌
- 적 스폰
- 투사체
- 무기 로직
- XP와 아이템 드랍
- UI
- 2D 이펙트
- 게임 상태 전환

Three.js는 캐릭터 시각 표현을 담당한다.

Three.js 담당:

- 플레이어 3D 모델
- 감염체 3D 모델
- 카툰 렌더링
- 외곽선
- 캐릭터 애니메이션
- Phaser 좌표와 시각 동기화

중요 원칙:

- 게임 판정의 진실은 Phaser에 둔다.
- Three.js 모델은 충돌 판정의 기준으로 쓰지 않는다.
- Three.js는 "보여주는 역할"에 집중한다.

### 3-3. 데이터 기반으로 만든다

몬스터, 무기, 웨이브, 업그레이드는 코드에 박아두지 않고 데이터로 분리한다.

분리해야 할 데이터:

- 감염체 데이터
- 무기 데이터
- 웨이브 데이터
- 업그레이드 데이터
- 드랍 데이터

이렇게 하면 나중에 밸런스를 바꿀 때 로직 코드를 많이 고치지 않아도 된다.

### 3-4. 초보자 혼자 개발 가능한 단위로 쪼갠다

개발은 큰 시스템을 한 번에 만들지 않고 작은 단계로 진행한다.

권장 순서:

1. 세로형 Phaser 씬 생성
2. `SchoolGroundScene` 구성
3. `StudentSurvivor` 이동 구현
4. Three.js 카툰 캐릭터 표시
5. Phaser 위치와 Three.js 모델 동기화
6. `InfectedStudent` 추적 구현
7. `PencilThrow` 자동 공격 구현
8. 충돌과 데미지 처리
9. XP 오브와 레벨업 구현
10. 웨이브와 보스 구현

각 단계는 실행해서 눈으로 확인 가능한 결과를 가져야 한다.

### 3-5. 성능은 초반부터 고려한다

서바이버라이크 게임은 적과 투사체가 많아지기 때문에, 초반부터 성능 기준을 둔다.

필수 기준:

- 화면 내 감염체 100마리 이상을 고려한다.
- 적, 투사체, XP 오브는 오브젝트 풀링을 사용한다.
- Three.js 캐릭터 모델도 가능하면 풀링한다.
- 작은 감염체는 디테일보다 실루엣과 색상 구분을 우선한다.
- 발사체 수 상한을 둔다.
- 3분 이후 화면 밀도가 높아져도 플레이어 위치가 묻히지 않아야 한다.

### 3-6. 검수 기준을 문서화한다

각 기능은 구현 후 다음 기준으로 검수한다.

- 게임 콘셉트에 맞는가
- 5분 생존 루프에 도움이 되는가
- 모바일 세로 화면에서 읽히는가
- Phaser와 Three.js 역할이 섞이지 않았는가
- 데이터 수정으로 밸런스 조정이 가능한가
- 초보자 기준으로 유지보수 가능한가
- 최소 1개 정상 경로와 1개 실패 경로를 확인했는가

## 4. 한 줄 정리

BangBang Survivor는 "학생 생존자가 좀비에 감염된 학교에서 5분 동안 버티고 탈출 기회를 만드는 모바일 세로형 카툰 서바이버라이크"이며, 구현은 Phaser 4로 생존 게임 로직을 만들고 Three.js 카툰 렌더링으로 캐릭터를 명확하게 보여주는 방식으로 진행한다.
