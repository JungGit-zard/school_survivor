# Admin Operations Control Scope

> 작성일: 2026-06-21  
> 역할: CEO / 제품 방향 기록  
> 상태: 결정

## 결정

어드민 페이지 1차 범위는 `운영자가 밸런스와 시즌/랭킹 보상을 조정하고, 그 결과가 현재 게임 클라이언트에 반영되는 로컬 운영 콘솔`로 정한다.

## 이유

- 현재 게임은 로컬 저장과 클라이언트 프로토타입 중심이므로 서버 권한보다 빠른 밸런스 실험이 더 중요하다.
- 랭킹은 아직 공식 서버 검증 경쟁 시스템이 아니므로, 보상/시즌 설정은 먼저 표시와 점수 정책 조정까지 연결한다.
- 실제 보상 지급과 부정행위 방지는 서버 검증 이후 확장한다.

## 1차 운영자가 조작할 수 있는 것

- Stage 1/2 생존 시간.
- 플레이어 시작 HP 보너스와 이동 속도 배율.
- 생존 마일스톤 골드 배율.
- 랭킹 시즌 ID/이름/상태/기간.
- Stage 2 보너스 점수와 클리어 보너스 점수.
- TOP 1, TOP 10, TOP 100 보상 골드와 배지명.

## 운영 리스크

- 생존 시간이 너무 짧으면 랭킹 점수와 클리어 판정이 왜곡된다.
- 속도/HP를 과하게 올리면 기존 난이도 문서와 플레이 감각이 달라진다.
- 랭킹 보상은 공식 지급처럼 표현하면 안 된다. 1차 UI에서는 `보상 안내`로 다룬다.
