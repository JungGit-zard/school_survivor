# BangBang Survivor — 구글 플레이 스토어 런칭 전략
작성일: 2026-05-02

---

## 1. 핵심 방향

**목표:** BangBang Survivor를 구글 플레이 스토어에 정식 등록해 안드로이드 유저에게 배포한다.

**기준:**
- 첫 출시는 완성도보다 **플레이 가능한 루프**를 빠르게 배포하는 것에 집중한다.
- 무료 다운로드 + 리워드 광고 모델로 시작한다.
- 다운로드 용량은 **50MB 이하**를 목표로 한다.

---

## 2. 기술 경로: 웹 앱 → 안드로이드 앱

현재 게임은 React + R3F(Three.js) 기반 웹 앱이다.
구글 플레이 등록을 위해 **안드로이드 앱으로 패키징**이 필요하다.

### 권장 방법: TWA (Trusted Web Activity) + Bubblewrap

| 항목 | 내용 |
|------|------|
| 방식 | 웹앱을 네이티브 Chrome 엔진으로 감싸 APK 생성 |
| 도구 | Google Bubblewrap CLI |
| 장점 | 코드 재작성 없음, 웹 성능 그대로 유지 |
| 단점 | 기기 호환성 일부 제한 (Chrome 72 이상 필요) |
| 광고 연동 | Google AdMob WebView 광고 또는 JS 브릿지로 연동 |

### 대안: Capacitor (차선책)

| 항목 | 내용 |
|------|------|
| 방식 | 웹앱을 WebView로 감싸 네이티브 앱 생성 |
| 도구 | Ionic Capacitor |
| 장점 | 네이티브 플러그인 접근 용이 (광고, 알림, 결제) |
| 단점 | 초기 설정 복잡, 안드로이드 스튜디오 필요 |

**1단계에서는 TWA + Bubblewrap 경로를 채택한다.**

## 3. 출시 전 필수 완성 항목

### 3-1. 게임 플레이 (필수)
- [ ] 5분 생존 루프 완성 (시작 → 웨이브 → 보스 → 클리어/게임오버)
- [ ] 모바일 터치 조이스틱 정상 동작
- [ ] 레벨업 선택 UI 모바일 터치 정상 동작
- [ ] 게임오버 후 재시작 기능
- [ ] 60fps 또는 안정적인 30fps 유지 (중저가 안드로이드 기준)

### 3-2. 스토어 등록 자산 (필수)
- [ ] 앱 아이콘: 512×512px PNG
- [ ] 피처드 이미지: 1024×500px
- [ ] 스크린샷: 세로형 최소 2장 (권장 4~8장)
- [ ] 짧은 설명: 80자 이내
- [ ] 전체 설명: 4000자 이내
- [ ] 콘텐츠 등급 설문 완료 (폭력 수위 표시)
- [ ] 개인정보처리방침 URL (필수 — 무료 호스팅 가능)

### 3-3. 기술 요건 (필수)
- [ ] Target SDK: Android 14 (API 34) 이상
- [ ] 64비트 지원 (TWA는 자동 충족)
- [ ] APK 또는 AAB 빌드 생성
- [ ] 서명 키스토어 생성 및 보관
- [ ] 구글 플레이 개발자 계정 등록 ($25 일회성)

### 3-4. 법적 요건 (필수)
- [ ] 개인정보처리방침 작성 및 호스팅
- [ ] 광고를 넣는 경우 AdMob 약관 동의
- [ ] 콘텐츠 등급: 만 7세 이상 또는 만 12세 이상 예상

## 4. 출시 단계 로드맵

### Phase 1 — 내부 테스트 (현재 ~ 2주)
- 게임 플레이 루프 완성
- 모바일 터치 컨트롤 안정화
- 성능 최적화 (중저가 안드로이드 30fps 이상)
- TWA 빌드 테스트

### Phase 2 — 비공개 테스트 (2주 ~ 4주)
- 구글 플레이 내부 테스트 트랙 등록
- 팀원 5~10명 테스트
- 버그 수정, 밸런스 조정
- 스토어 등록 자산 준비

### Phase 3 — 공개 출시
- 오픈 테스트 또는 정식 출시
- 리워드 광고 연동 (AdMob)
- 출시 후 1주일 리뷰 모니터링

## 5. 수익화 전략

**기본 모델:** 무료 다운로드 + 리워드 광고

| 광고 노출 시점 | 내용 |
|----------------|------|
| 게임오버 후 | "광고 보고 1회 부활" |
| 런 종료 후 | "광고 보고 획득 보상 2배" |
| 레벨업 시 | "광고 보고 선택지 추가 1개" |

**금지 사항:**
- 전투 중 강제 광고 금지
- 첫 클리어를 광고 없이 불가능하게 만드는 구조 금지

**추후 추가 검토:**
- 스킨/캐릭터 코스튬 IAP (게임플레이 영향 없는 외형)
- 광고 제거 IAP (1~2달러)

## 6. 스토어 포지셔닝

**핵심 키워드:** 서바이버, 좀비, 학교, 생존, 카툰 액션

**경쟁 포지셔닝:**
- 탕탕특공대(Survivor.io)의 짧은 세션 구조
- 학교+좀비라는 로컬 친숙 설정으로 차별화
- 한국 유저 1차 공략 후 글로벌 확장 검토

**스토어 짧은 설명 (초안):**
> 좀비에 감염된 학교에서 5분 동안 살아남아라! 자동 공격, 레벨업 선택, 보스 처치까지 — BangBang Survivor
