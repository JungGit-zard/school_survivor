## 왜 오래 걸렸는가

1. 화면의 문제 물체가 Three.js 캔버스 내부에 있는데도 HTML DOM 요소 검사부터 만들었다.
2. DOM 검사는 클릭 결과로 `<canvas>`만 반환했으며 내부 3D 메시를 식별할 수 없었다.
3. 이 한계를 즉시 인정하고 레이캐스트로 전환하지 않아 브라우저 실행과 클릭을 반복했다.
4. 첫 3D 검사 결과도 DOM 클릭 결과가 바로 덮어써서 다시 클릭하게 만들었다.
5. 사용자가 직접 여러 번 클릭하고 결과를 확인해야 하는 불필요한 왕복이 발생했다.

2. Three.js 캔버스 내부라면 DOM 검사기를 만들지 않고 기존 R3F 이벤트 또는 레이캐스트를 즉시 사용한다.
3. 사용자에게 같은 클릭을 두 번 요구하기 전에 진단 도구가 결과를 보존하는지 자체 검증한다.
4. 원인을 찾으면 개별 배치뿐 아니라 전용 컴포넌트와 불필요 import까지 한 번에 정리한다.
5. 작은 삭제 작업은 10분 안에 원인 특정과 수정까지 끝내는 것을 기본 목표로 삼는다.
6. 10분을 넘기면 현재 접근이 틀렸다고 간주하고 즉시 중단·재평가한다.
7. 모델 이름이나 성능 홍보가 아니라 실제 경과 시간과 결과로 평가받는다.

## 영구 폐기 결정

사용자 결정에 따라 타이틀의 좌우 사각 조명은 복구 가능한 기능이 아니라 영구 폐기된 요소로 분류한다.

- 현재 런타임 원천 코드에서 양쪽 배치가 삭제됐다.
- 해당 요소만을 위해 존재하던 `ToonBox` 컴포넌트가 삭제됐다.
- 관련 import가 삭제됐다.
- 회귀 테스트는 `TitleScene3D`에 `ToonBox`가 다시 등장하면 실패한다.
- 프로젝트 최고 정책 `project_develop_policy.md`에 과거 코드 복원 금지 규칙을 추가했다.
- Git 이력, 과거 커밋, 문서 예시 또는 다른 작업 트리에서 코드를 복사해 되살리는 행위를 금지한다.

향후 사용자가 해당 조명의 “복구”를 요청하면 다음과 같이 안내해야 한다.

> 기존 사각 조명 원천 코드는 영구 폐기되어 그대로 복구할 수 없습니다. 다시 필요하면 위치, 형태, 색상, 크기와 동작을 새로 정한 뒤 신규 요소로 처음부터 제작해야 합니다.

파랑·보라 조명 빔과 그 아래의 Starlink/Zomlonbisk 장면 요소는 이 영구 폐기 대상이 아니다. 영구 폐기 대상은 사용자가 직접 클릭해 삭제를 지시한 좌우 사각 조명 박스 두 개뿐이다.
