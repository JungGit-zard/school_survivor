## 1. 스테이지3 아이덴티티 / 시그니처 기믹

### 상승 계보
- **스테이지1 (교실 생존)** — 추격/근접. E01 스웜 + E02 탱커 + E05 차저. 단일 축(밀어붙임) 압박.
- **스테이지2 (복도 투사체 시험)** — 원거리 E04 도입. 좁은 복도에서 탄막 회피 + 근접. 이동/회피 두 축.
- **스테이지3 (총력전 / 혼돈)** — **3축 동시 압박**: 차저(E05) + 거대(E06) + 원거리(E04)가 **한 화면에 겹쳐** 들어온다. 스1의 근접·스2의 원거리를 통합하고, 여기에 거대(E06)를 상시 위협으로 격상.

### 시그니처: "3축 겹침(triple-threat overlap)" + 개방형 아레나
스테이지3의 정체성은 **단일 신규 적이 아니라 "동시성"**이다. 지금까지 순차적으로 배웠던 위협들(러너→탱커→원거리→차저→거대)이 후반부에 **동시에** 밀려온다:
- **E06 거대**를 스1(피크 비중 0.03)·스2(피크 0.18)보다 **상시·고비중(피크 0.15)**으로 격상 → 항상 킷하며 돌아야 하는 앵커.
- **E05 차저**가 그 킷 경로를 응징(피크 0.25).
- **E04 원거리**가 멈춰서면 응징(상시 0.08~0.18) → 서 있어도 죽고 돌아도 죽는 딜레마.
- **개방형 아레나(D1)**라서 스폰 링(4.0~6.5) 360° 리빌이 사방에서 성립 → 복도처럼 한 축으로 도망칠 벽이 없다. `ring`/`gauntlet` formation이 개방 공간에서 완전 포위로 작동.

### 신규 자산 없이 성립 (fallback 명시)
- **잡몹**: 전량 기존 E01~E06. 신규 적 불필요.
- **formation**: `swarm/ring/pincer/column/gauntlet` **5종 모두 이미 구현됨**(`Enemies.jsx` `formationSpawnPositions`). 신규 코드 0.
- **보스**: **더블 보스 B01+B02**(둘 다 차저, 자산 존재) 동시 등장으로 실질 난이도 상승. B03 자산 없이 성립.
- **이상적 확장(선택)**: B03 신규 — 두 기존 보스가 모두 차저이므로, **비-차저형(원거리 포격 또는 소환형)** B03이 있으면 "총력전"을 더 완성. 단 threemini 자산 + 신규 거동 코드 필요 → D2/D3 승인 사항.

## 3. 난이도 곡선 근거

### 3-1. 왜 이 target/구성인가 (구간별)
- **온보딩 압축(0–16s)**: 스1 40s·스2 24s 단일좀비 구간을 16s로. 이미 2스테이지를 깬 숙련 플레이어 전제. 첫 웨이브부터 E03 러너를 섞어 "다르다"를 즉시 전달.
- **조기 도입 사슬(34/52/108s)**: 원거리(E04)·차저(E05)·거대(E06)를 스2 대비 각각 **~38s / ~68s / ~60s 앞당김**. 세 위협의 도입 간격을 좁혀 후반 "겹침"의 재료를 일찍 깔아둔다.
- **호흡 구간(92–108s)**: 순수 완화가 아니라 **탱커(E02 0.45)벽**으로 전환 — 리듬은 바꾸되 난이도는 유지. 거대(E06) 도입 직전에 배치해 대비 효과 극대화.
- **피크(132–150s, target32)**: 보스 직전, E05 0.25 + E06 0.15 + E04 0.10 동시 = 3축 최대 겹침. 여기가 트래시 최고 압력점.
- **보스 구간 감량(150s+)**: 더블 보스(2300 HP)가 화면을 지배하므로 트래시 target을 16~20으로 급감. 안 그러면 보스+트래시 동시 과부하로 불공정+렌더 부담. E04 원거리만 견제로 남겨 "멈추면 죽음" 유지.
- **탈출 스프린트(218–240s)**: E01 0.44로 뚫고 나갈 여지 확보(스1 224–240 E01 0.25, 스2 E01 0.20와 동일 철학) — 마지막은 "탈출 가능한 혼돈".

### 3-3. 20~40초 랜덤 간격 표집 분산 감안
- **핵심 사실**: 첫 웨이브 t=0, 이후 간격 평균 30s → 240s에 **평균 ~8~9웨이브**(최소 6, 최대 12). phase는 12개인데 웨이브는 ~8회 → **모든 phase가 실현되지 않는다.** 발화 시각의 활성 phase(`findLast(start)`)만 표집된다.
- **좁은 phase 스킵 위험**: 16~18s 폭 phase(온보딩·호흡)는 운 나쁜 draw(20~40)에서 **웨이브가 한 번도 안 떨어질 수 있다.** 이는 스1/스2도 감수 중인 구조.
- **본 설계의 방어책 — 타입 지속(persistence)**: 한 번 도입한 위협 타입을 **다음 여러 phase에 계속 실어** 특정 intro phase가 스킵돼도 곧 다른 phase에서 그 타입이 등장하게 설계했다.
  - E04: 34–52 도입 후 52·72·132·150·172·218 구간에도 존재 → 어느 웨이브를 표집해도 후반엔 반드시 원거리 조우.
  - E05: 52 도입 후 72·92·108·132·150·172·196·218 지속.
  - E06: 108 도입 후 132·172 지속.
  - 따라서 **"어떤 8개 phase가 표집되든 3축 겹침으로 수렴"**하는 단조 상승 곡선. 개별 phase 스킵이 곡선을 깨지 않는다.
- **피크 보장**: 132–150(피크)는 18s로 다소 좁지만, 바로 앞 108–132(24s, E06 포함)와 구성이 연속이라 둘 중 하나는 거의 확실히 표집된다. balanceqa가 시드 다수로 "거대 미조우 런" 발생률을 확인하는 것을 권장(3-4 리스크).

### 4-1. STAGE3_BURST_EVENTS 제안 (burstEvents.js)

```js

export const STAGE3_BURST_EVENTS = [

  { sec:  34, type: 'E04', count:  1 },                         // 원거리 조기 등장 신호
  { sec:  52, type: 'E05', count:  2 },                         // 차저 조기 등장 신호
  { sec: 108, type: 'E06', count:  1 },                         // 거대 조기 등장 신호
  { sec: 132, type: 'E06', count:  1 },                         // 피크 거대 1기 추가
  // ── 더블 보스: 같은 sec에 B01+B02 → getBossSpawnSec=min=150, 2기 동시 ──
  { sec: 150, type: 'B01', count:  1 },
  { sec: 150, type: 'B02', count:  1 },
  { sec: 196, type: 'E05', count:  3 },                         // 마틸다 직전 차저 러시
  // ── formation 스파이크(균일 압력 깨기) — 예고는 STAGE3_SPAWN_TELEGRAPHS와 정렬 ──
  { sec:  44, type: 'E03', count:  6, formation: 'ring' },      // 개방 아레나 첫 완전 포위
```

## 5. balanceqa 검증 필요 리스크 지점 (핸드오프 노트)

1. **[최우선] 더블 보스 DPS 체크 & 화면 가독성(D2)**: B01+B02 = 2300 HP를 150~240s(90s) 안에 처치 가능한가? 스테이지3 진입 시점의 전형적 무기/업그레이드 빌드로 DPS 측정 필요. 두 차저가 **동시에 warn→charge**하면 회피 불가 상황(협공 돌진)이 생길 수 있음 — 겹침 빈도·회피 가능성 검증. 필요 시 보스 등장 시각을 132s로 앞당겨 창을 108s로 확대.
2. **더블 보스 + 마틸다 3중 겹침(196~240s)**: 마틸다(210) 등장 시 보스 2기가 아직 생존 중이면 추격자 3기. matildaSec를 210으로 늦췄지만 여전히 겹칠 수 있음 — 이 창의 체감 공정성 판정.
3. **HP ×1.0 전환 실효 난이도(D4)**: 스2 ×0.8 → 스3 ×1.0은 잡몹 처치 시간 +25%. E06(320 HP·상시) + E02(70 HP·고비중)가 겹치는 108~150 구간에서 "잡아도 안 줄어드는" 스택 과부하가 나는지. 동시 생존 적 수 상한(모바일 렌더) 확인.
4. **표집 분산 — 거대/원거리 미조우 런**: 20~40s 랜덤에서 특정 phase 스킵 시 E06/E04를 후반까지 못 만나는 시드가 있는가? 다수 시드(예: 50런) 몬테카를로로 "E06 첫 조우 시각" 분포·최악 시드 확인. 타입 지속 설계가 실제로 방어하는지 검증.
5. **피크(132–150) 동시 적 수**: target32 웨이브(16마리) + 잔존 + 형태 버스트(176 gauntlet 등) 겹칠 때 순간 최대 동시 좀비 수. 스1/스2 최대치 대비 초과분과 모바일 프레임 영향.
6. **개방 아레나 formation 재튜닝(D1)**: `RING_MAX_RADIUS(5)`·`PINCER_Z_OFFSET(10)`·`GAUNTLET_WALL_INSET`이 halfX 7.5(복도) 기준 → halfX 12 개방 맵에서 포위/협공 체감이 유지되는지(반경이 화면 밖으로 나가 포위감 상실 위험). threemini와 공동.
7. **E04 cap 로직**: `Enemy.jsx`에 E04 동시 상한(cap) 로직 존재. 스3는 E04를 조기·다구간 배치하므로 cap이 후반 원거리 비중을 의도보다 깎지 않는지 확인.

## 요약 (한눈에)

- **아이덴티티**: 스1(근접)→스2(원거리)의 통합 상승 = **"3축 겹침 총력전"**(차저 E05 + 거대 E06 + 원거리 E04 동시) + **개방형 아레나**(사방 포위).
- **핵심 곡선**: 온보딩 16s로 압축 → 원거리34s·차저52s·거대108s **조기 도입** → 132–150 피크(target32) → 150 **더블 보스 B01+B02**(2300 HP) → 마틸다 210 → 탈출 스프린트.
- **상승 레버(질적)**: HP ×1.0(+25%), 위협 조기 도입, E06 상시화, 더블 보스, 전방위 포위. **마릿수(target 16~32)는 스2와 동급으로 억제** — 모바일 공정성 유지.
- **신규 자산 0으로 성립**: 기존 E01~E06 + formation 5종(구현됨) + 더블 보스. B03은 선택적 이상 확장.
- **승인 필요**: D1 맵/테마, D2 보스(더블 vs B03 신규), D4 HP ×1.0, D5 언락 — 상단 표 참조.
