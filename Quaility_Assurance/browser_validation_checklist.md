# Browser Validation Checklist

Last updated: 2026-05-16

## 1. 목적

브라우저에서 실제 게임이 보이고 조작되는지 확인하기 위한 반복 체크리스트다.

## 2. 기본 명령

```powershell
cd Developer/r3f_prototype
npm run dev -- --host 127.0.0.1 --port 5173
```

기본 URL:

```text
http://127.0.0.1:5173/
```

gstack browse 실행 파일:

```text
C:\Users\admin\.claude\skills\gstack\browse\dist\browse.exe
```

## 3. 매번 확인할 항목

### 로딩

- [ ] 서버가 `200` 응답을 반환한다.
- [ ] 초기 화면이 렌더링된다.
- [ ] 타이머, 레벨, HP, XP, 골드가 보인다.
- [ ] 플레이어 모델이 보인다.

### 콘솔

- [ ] 치명적인 JavaScript 오류가 없다.
- [ ] WebGL 경고가 반복적으로 플레이를 망치지 않는다.

### 네트워크

- [ ] 주요 요청이 `200` 또는 캐시성 `304`로 끝난다.
- [ ] 누락된 모듈 요청이 없다.

### 반응형

- [ ] `375x812` 모바일 캡처
- [ ] `390x844` 모바일 캡처
- [ ] `768x1024` 태블릿 캡처
- [ ] `1280x720` 데스크톱 캡처

### 모바일 조작

- [ ] `snapshot -i`에서 조작 가능한 UI가 잡힌다.
- [ ] 조이스틱이 표시된다.
- [ ] 터치 이동이 가능하다.
- [ ] pause 버튼이 있다.
- [ ] pause 후 resume이 가능하다.

### HUD

- [ ] 하단 HP/XP가 안전 영역과 충돌하지 않는다.
- [ ] 레벨업 카드가 390px 화면에서 잘리지 않는다.
- [ ] 게임오버 모달이 잘리지 않는다.
- [ ] 클리어 모달이 잘리지 않는다.

## 4. 현재 알려진 실패

- 모바일 pause 버튼은 `snapshot -i`에서 확인됐다.
- pause 버튼 클릭 시 `PAUSED`와 `계속하기` 버튼이 확인됐다.
- 조이스틱은 실제 화면에 연결됐다.
- 하단 HUD 안전 영역과 실제 터치 드래그 이동감은 추가 실기기 확인이 필요하다.

## 5. 통과 기준

브라우저 검증을 완료로 기록하려면 다음을 만족해야 한다.

- `npm test` 통과
- `npm run build` 통과
- 모바일/태블릿/데스크톱 캡처 확인
- 콘솔 치명 오류 없음
- 모바일에서 이동과 pause/resume 가능
- 레벨업/결과 모달 잘림 없음
