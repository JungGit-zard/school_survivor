# Stage 1 Classroom Desk Layout QA Gate

작성일: 2026-06-07
대상: Stage 1 `교실 생존` 책상 배치안
범위: `stageObjectPlacements` 기준 Stage 1 책상 배치 데이터 검토만 포함. 구현 변경 없음.
상태: QA 기준 정의 완료, 런타임 검증 미실행

## 1. 분석 범위

- Stage 1 핵심 목표는 `300초 생존`이다.
- Stage 1 핵심 플레이 감각은 `중앙 회피 공간 유지`다.
- 현재 책상은 `시각 전용 배경 오브젝트`이며 `충돌체 없음`이다.
- 현재 임시 좌표는 다음 4개다.
  - `[-18, -20]`
  - `[19, -16]`
  - `[-21, 13]`
  - `[20, 18]`
- 현재 코드 기준 플레이 구역은 대략 `±48` 경계다.
- 현재 자동 테스트에는 `Stage 1 책상이 중앙부에 들어오지 않는지`를 보는 최소 가드가 이미 있다.
  - 근거: `Developer/r3f_prototype/src/components/StageObjects/stageObjectPlacements.test.js`

## 2. 핵심 QA 판단

### 확인된 사실

- 현재 4개 좌표는 모두 중심점 `(0, 0)`에서 충분히 바깥쪽에 있다.
- 현재 4개 좌표는 모두 맵 외곽 경계 `±48` 안쪽에 있다.
- 현재 좌표 배치는 사실상 `4개 외곽 사분면 분산` 구조다.

### 주요 위험

1. 책상이 충돌하지 않는데 화면상으로는 길을 막는 것처럼 보일 수 있다.
2. 모바일 세로 화면에서는 하단 조이스틱, HUD, 카메라 원근 때문에 하단 중앙 책상이 실제보다 더 크게 느껴질 수 있다.
3. 앞으로 소품이 늘어나면 지금의 "중앙 비우기"가 StageObjects 데이터에서 조용히 무너질 수 있다.
4. 자동 테스트가 현재는 `중앙 침범`만 막고 있어서 `군집`, `화면 가림`, `미래 오브젝트 추가 여유`는 잡지 못한다.

### 최소 권장 완화책

- Stage 1 책상 배치안은 "외곽 장식, 중앙 생존 공간 보존"을 QA 게이트로 먼저 고정한다.
- 추가 오브젝트 검토도 같은 게이트를 통과해야 한다.
- 특히 모바일 세로 화면에서는 "하단 중앙 시야 비우기"를 별도 수동 게이트로 둔다.

## 3. 배치안이 통과해야 할 검증 기준

### A. 구조 기준

1. 중앙 생존 구역을 시각적으로 비워야 한다.
   - 최소 기준: 책상 기준 좌표가 `x = -12~12` 그리고 `z = -12~12` 안에 들어오면 실패다.
   - 이유: 현재 자동 테스트도 같은 의도의 중앙 회피 규칙을 사용한다.
2. 외곽 장식 역할을 유지해야 한다.
   - 책상은 중심보다 외곽 사분면에 배치되어야 한다.
   - 현재 4개 책상 기준으로는 `좌상/우상/좌하/우하` 분산이 가장 안전하다.
3. 미래 오브젝트 여유를 남겨야 한다.
   - 특정 한 변이나 한 코너에 큰 소품이 몰려 있으면 실패 후보다.
   - 같은 영역에 책상과 차기 대형 소품이 겹칠 수 있으므로 "한 화면 구역 과밀"을 금지한다.
4. 맵 경계에 너무 붙지 않아야 한다.
   - 책상 기준 좌표가 `|x| > 40` 또는 `|z| > 40`이면 실패 후보다.
   - 이유: 맵 경계 `±48` 대비 카메라 여유와 후속 소품 여유가 줄어든다.

### B. 플레이 가독성 기준

1. 플레이어 시작 직후 중앙 탈출 경로가 비어 보여야 한다.
2. 돌진 적 경고선, 적 체력바, 플레이어 실루엣을 책상이 가리지 않아야 한다.
3. 책상이 "통과 가능 장식"인지 "막힌 장애물"인지 혼동을 강하게 만들면 실패다.
4. 300초 루프 동안 책상이 전투 정보보다 더 먼저 눈에 들어오면 실패 후보다.

### C. 데이터 품질 기준

1. 모든 Stage 1 책상 데이터는 고유 `id`를 가져야 한다.
2. `type`은 StageObjects 규격값만 사용해야 한다.
3. 좌표, 회전, 스케일이 빠진 항목이 있으면 실패다.
4. 같은 좌표 또는 지나치게 가까운 중복 배치가 있으면 실패다.

## 4. 모바일 세로 화면에서 체크할 장면

권장 해상도:
- `375x812`
- `390x844`
- 가능하면 `320x568`도 추가 확인

필수 확인 장면:

1. 시작 직후 0~15초
   - 플레이어가 중앙 근처에서 스폰됐을 때 책상이 바로 도주 경로를 막아 보이지 않는지 확인
   - 조이스틱이 나타난 뒤 하단 화면에서 책상과 조작 UI가 시각적으로 엉키지 않는지 확인
2. 중앙 복귀 장면
   - 좌상, 우상, 좌하, 우하 외곽을 각각 한 번 훑고 중앙으로 돌아왔을 때 중앙이 계속 비어 보이는지 확인
3. 돌진 적 압박 구간 150~210초
   - E05 돌진 경고와 플레이어 회피선이 책상 때문에 읽기 어려워지지 않는지 확인
4. 보스 경고 직전 230~240초
   - 중앙 대기, 측면 이동, 하단 회피 모두에서 책상이 공포 연출을 망치지 않는지 확인
5. 저체력 장면
   - 화면 가장자리 붉은 비네트가 켜진 상태에서도 책상이 플레이어 위치를 과하게 숨기지 않는지 확인

## 5. 자동 테스트로 잡을 수 있는 기준

자동 테스트 적합 항목:

1. Stage 1 책상 데이터 존재 여부
2. 모든 항목의 `id`, `type`, `position`, `rotation`, `scale` 존재 여부
3. 중앙 생존 구역 침범 금지
   - 예: `x/z` 모두 `-12~12` 안에 들어오는 좌표가 없어야 함
4. 맵 경계 여유
   - 예: `|x| <= 40`, `|z| <= 40`
5. 중복 좌표 금지
6. 최소 간격 기준
   - 예: 대형 소품끼리 너무 가까운 배치 금지
7. Stage 1 책상 수 상한
   - 현재 단계에서는 `4개 유지` 또는 `기획 승인된 상한` 초과 금지

자동 테스트만으로 부족한 항목:

- 모바일 세로 화면에서의 실제 가림
- "막힌 것처럼 보이는지" 같은 시각적 오해
- 조이스틱, HUD, 적 경고선과의 실제 화면 충돌

## 6. 수동 플레이테스트 체크리스트

- [ ] 시작 10초 안에 중앙 회피 공간이 비어 보인다.
- [ ] 플레이어가 책상 근처를 지나갈 때 "왜 안 막히지?"라는 시각적 위화감이 강하지 않다.
- [ ] 책상이 플레이어 몸, 적 체력바, 돌진 경고선보다 먼저 눈에 들어오지 않는다.
- [ ] 하단 중앙 시야가 조이스틱과 책상 때문에 답답하지 않다.
- [ ] 좌우 외곽으로 이동할 때 책상이 화면 가장자리에서 잘려도 부자연스럽지 않다.
- [ ] 중앙에서 8자 이동을 반복해도 회피선 읽기가 유지된다.
- [ ] 150~210초 돌진 압박 구간에서 책상이 회피 판단을 방해하지 않는다.
- [ ] 230~240초 보스 경고 직전에도 중앙 대기 공간이 유지되어 보인다.
- [ ] 300초 완주 또는 자연 게임오버까지 책상 때문에 "맵이 좁다"는 인상이 과하게 생기지 않는다.
- [ ] 향후 사물함, 의자 더미, 콘 같은 추가 소품을 놓을 외곽 여유가 남아 있다고 판단된다.

## 7. Go / No-Go 기준

## Go

- 중앙 생존 구역 침범이 없다.
- 자동 데이터 가드가 모두 통과한다.
- 모바일 세로 화면 `375x812`, `390x844`에서 하단 중앙 시야 답답함이 없다.
- 150~240초 핵심 압박 구간에서 회피 정보 가독성이 유지된다.
- 플레이테스터가 책상을 "배경 장식"으로 받아들이고, "보이지 않는 가짜 장애물"로 오해하지 않는다.

## No-Go

- 책상 1개라도 중앙 생존 구역 안에 들어온다.
- 하단 중앙 화면에서 조이스틱/HUD와 겹쳐 회피 시야를 줄인다.
- 책상 때문에 돌진 경고선, 플레이어, 적 체력바 가독성이 깨진다.
- 외곽이 아닌 특정 한쪽으로 배치가 몰려 향후 소품 확장 여유가 없어진다.
- 플레이테스트 중 "막혀 보이는데 실제로는 통과됨" 피드백이 반복된다.

## 8. 현재 임시 배치에 대한 QA 의견

- 현재 임시 좌표 4개는 `중앙 회피`라는 1차 목적에는 맞는 편이다.
- 특히 4사분면 분산 구조라서 "중앙 비우기" 관점에서는 출발점으로 무난하다.
- 다만 이것만으로 QA 통과를 줄 수는 없다.
  - 이유 1: 현재 자동 가드는 중앙 침범만 막는다.
  - 이유 2: 모바일 세로 화면의 하단 시야 압박은 데이터만 보고 확정할 수 없다.
  - 이유 3: 앞으로 소품이 추가되면 지금 좌표가 기준점이 되어 과밀화될 수 있다.

## 9. 검증 상태와 잔여 리스크

### 이번에 확인한 것

- 문서/코드 기준으로 현재 Stage 1 책상은 중앙에서 떨어져 있다.
- 현재 Stage 1 책상은 시각 전용 배경이라는 전제가 명확하다.
- 현재 자동 테스트에는 중앙 회피 최소 가드가 이미 존재한다.

### 아직 런타임 검증이 필요한 것

- 모바일 세로 화면 실제 시야
- 조이스틱, HUD, 경고선과의 화면 겹침
- 300초 플레이에서의 시각 피로도
- 책상 비충돌 상태가 플레이어에게 주는 혼동 정도

### 잔여 리스크 우선순위

- P1: 모바일 세로 화면 하단 중앙 시야 압박
- P1: 충돌 없음인데 막힌 것처럼 보이는 시각 오해
- P2: 후속 오브젝트 추가 시 외곽 과밀화
- P2: 자동 테스트가 화면 가독성 문제를 놓치는 구조

## 10. 후속 권장 액션

1. 이 문서를 Stage 1 책상 배치 QA 게이트의 기준 문서로 사용한다.
2. 다음 배치안 검토 때는 반드시 모바일 세로 화면 스크린샷 또는 실기 화면을 붙여 함께 판단한다.
3. 추가 소품이 생기면 책상만 따로 보지 말고 `StageObjects 전체 배치 밀도` 기준으로 다시 QA한다.
