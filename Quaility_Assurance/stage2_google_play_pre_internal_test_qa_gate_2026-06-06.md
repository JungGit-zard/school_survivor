## 3. 핵심 발견과 우선순위

### P0. 저장 무결성 위험

- 설명: 게임오버 뒤 `damagePlayer`가 다시 불리면 결과 기록이 두 번 저장될 수 있다.
- 사용자 영향: 최고 기록, 클리어 횟수, 해금 진행도가 잘못 올라갈 수 있다.
- 판단: 테스터 모집 전에 반드시 차단해야 한다.
- 검증 경로:
  - 체력 0 직후 추가 피격이 발생하는 상황 재현
  - 결과 화면 진입 후 기록 저장 호출 횟수 확인
  - 재시작 후 저장 데이터가 1회만 반영되는지 확인

### P0. Stage 2 공정성 위험

- 설명: E04 탄환 상한이 화면 전체 기준이 아니라 적 개체 기준이면 복도 스테이지가 피할 수 없는 패턴으로 무너질 수 있다.
- 사용자 영향: 억울한 피격, 회피 불가 구간, 밸런스 붕괴
- 판단: 내부 테스트에 올리면 피드백이 밸런스가 아니라 버그/불공정으로 오염된다.
- 검증 경로:
  - 90초 이후 E04 다수 출현 구간에서 동시 활성 탄환 총량 측정
  - 보스 압박 구간에서 탄환 수가 추가로 폭증하지 않는지 확인
  - 좁은 복도 중앙 안전 레인이 실제로 유지되는지 확인

### P0. 비플레이 상태 진행 위험

- 설명: `pause`, `levelup`, `gameover` 중에도 일부 투사체가 계속 움직이거나 피해를 줄 수 있다.
- 사용자 영향: 메뉴를 열었는데 맞아 죽거나, 레벨업 중 피격되는 치명적 불신
- 판단: 발견 즉시 No-Go
- 검증 경로:
  - 탄환이 화면에 있는 상태에서 pause
  - 탄환 직전 프레임에서 levelup 진입
  - gameover 직후 추가 피격, 추가 사운드, 추가 저장 여부 확인

### P0. 릴리스 빌드 치트 노출 위험

- 설명: 타이틀의 개발 치트가 릴리스 AAB에서 보이면 테스터가 정상 진행 대신 우회 진행을 하게 된다.
- 사용자 영향: 테스트 데이터 오염, 진행도 신뢰 상실, Play 심사 인상 악화 가능성
- 판단: 테스터 모집 전 반드시 제거 또는 완전 비노출
- 검증 경로:
  - 클린 설치 상태 타이틀 화면 확인
  - 설정/옵션/숨김 제스처 포함 전체 UI 확인
  - 릴리스 빌드에서 무기 해금 진행이 정상 플레이로만 열리는지 확인

- [ ] E04 탄환 총량이 화면 전체 기준 상한을 넘지 않는다.
- [ ] E04가 플레이어 근접 거리에서 즉시 발사하지 않는다.

- [ ] 탄환이 날아오는 중 `pause`하면 탄환 위치, 타이머, 피격 판정이 모두 멈춘다.

- [ ] `gameover` 직후 추가 피격, 추가 사망 연출, 추가 저장이 발생하지 않는다.
- [ ] `resume` 후 정지 전 위치에서 자연스럽게 이어진다.
- [ ] Android 홈 버튼, 앱 전환, 화면 꺼짐 후 복귀에서도 자동 일시정지가 안전하게 동작한다.

### C. 저장/결과/해금 무결성 검증

- [ ] Stage 2 실패 기록이 1회만 저장된다.
- [ ] Stage 2 클리어 기록이 1회만 저장된다.
- [ ] `stage1Clears`, Stage 2 최고 생존 시간, Stage 2 클리어 횟수가 서로 섞이지 않는다.
- [ ] 구버전 저장 데이터에서 Stage 2 관련 값이 깨지지 않고 기본값으로 보정된다.
- [ ] 연속 재시작 5회 이상 후에도 기록 누락, 중복, 해금 꼬임이 없다.
- [ ] 앱 종료 후 재실행해도 가장 최근 결과가 그대로 유지된다.

### D. Stage 1 회귀 검증

- [ ] Stage 1 웨이브와 버스트에 E04가 다시 나타나지 않는다.
- [ ] Stage 1 보스가 Stage 2 전용 투사체 규칙을 가져오지 않는다.
- [ ] Stage 1 클리어/생존 기반 해금 규칙이 기존과 같게 유지된다.
- [ ] Stage 1 결과 저장이 Stage 2 추가 이후에도 정상 동작한다.

### E. 타이틀/메뉴/릴리스 노출 검증

- [ ] 릴리스 AAB 첫 실행 시 개발 치트가 보이지 않는다.
- [ ] 설정 화면, 숨김 탭, 긴 터치, 키보드 입력 등 우회 경로에도 치트가 없다.

- [ ] 앱 설치 후 첫 실행부터 검은 화면, WebView 멈춤, 포커스 유실이 없다.
- [ ] 백버튼 동작이 의도치 않은 종료나 저장 손실을 만들지 않는다.

- [ ] 실제 기기에 설치 가능한 서명된 AAB/APK 산출물을 확인했다.

- [ ] 디버그용 문구, 디버그 버튼, 테스트용 저장값 주입 코드가 노출되지 않는다.

## 8. Go / No-Go 기준

## Go

아래를 모두 만족하면 Google Play 내부 테스트 업로드와 테스터 모집 진행 가능이다.

1. 위 P0 네 가지 위험이 모두 수정 완료 또는 재현 불가로 검증됐다.
2. Stage 2 정상 경로, 실패 경로, Stage 1 연동 경로 스모크 테스트가 모두 통과했다.
3. 실제 Android 기기 최소 2대에서 릴리스 빌드 1판 이상 완료가 가능했다.
4. 저장 기록 중복, 누락, 스테이지 간 섞임이 없다.
5. 릴리스 빌드에서 개발 치트와 디버그 UI가 보이지 않는다.
6. 남아 있는 이슈가 있어도 P1 이하이며, 진행도/공정성/저장 무결성/앱 실행 자체를 깨지 않는다.

## No-Go

아래 중 하나라도 걸리면 AAB 업로드 또는 테스터 모집을 멈춘다.

1. 게임오버 후 기록이 중복 저장된다.
2. `pause`, `levelup`, `gameover` 중 탄환이 움직이거나 피해를 준다.
3. E04 탄환이 화면 전체 상한을 넘겨 회피 불가 패턴을 만든다.
4. 릴리스 빌드에서 개발 치트가 노출되거나 진행도를 우회할 수 있다.
5. Stage 1 기록과 Stage 2 기록이 섞이거나 손상된다.
6. 실제 Android 기기에서 크래시, 검은 화면, 입력 불능, 복귀 불능이 재현된다.
7. 300초 세션 또는 1회 게임오버까지 기본 플레이 루프를 안정적으로 끝내지 못한다.

## 9. 이번 문서 기준의 가장 작은 권장 조치

가장 작은 순서로는 아래가 맞다.

1. P0 네 가지를 먼저 막는다.
2. 그 다음 Stage 2 스모크 3종만 다시 돈다.
3. 그 다음 Android 실제 기기 2대 릴리스 검증으로 넘어간다.
4. 마지막에 AAB 업로드와 테스터 모집 문구를 연다.

이 순서가 가장 짧게 위험을 줄인다. 반대로 지금 상태에서 바로 테스터를 받으면 저장 무결성과 공정성 피드백이 섞여 원인 분리가 어려워진다.
