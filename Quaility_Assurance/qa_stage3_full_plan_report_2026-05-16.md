# QA 3단계 전체 계획/리포트 - 2026-05-16

## 1. 문서 목적

이 문서는 BangBang Survivor 현재 구현을 대상으로 한 3단계 QA 전체 계획/리포트이다.

QA는 Quality Assurance의 줄임말이며, 게임이 기획 의도대로 동작하는지 확인하고 실패 위험을 기록하는 작업이다. 이 문서는 코드 수정 없이 현재 구현 상태를 검증하고, 다음 검증 순서를 정리한다.

## 2. 검증 범위

- 대상 위치: `Developer/r3f_prototype`
- 대상 형태: React Three Fiber 기반 3D 생존 게임 프로토타입
- 검증 기준: 5분 Stage 1 플레이 흐름, 레벨업, 드랍, 골드, 보스, 리셋, 무기 업그레이드
- QA 기록 위치: `Quaility_Assurance/`
- 금지 사항: 코드 및 소스 수정 없음

이번 문서에서 자동으로 확인한 것은 테스트와 빌드 가능 여부다. 실제 5분 플레이, 보스 처치, 리셋 반복 같은 플레이 검증은 아직 수동 체크리스트로 남긴다. 검증하지 않은 기능은 완료로 기록하지 않는다.

## 3. 확인한 프로젝트 정책

- `project_develop_policy.md`: QA 테스트 계획, 검증 결과, 버그 위험, 리뷰 기록은 반드시 `Quaility_Assurance/` 폴더에 기록해야 한다.
- `AGENTS.md`: 작업 전 파일 구조 확인, 의미 있는 변경 후 `git status --short --branch` 확인, 코드 변경 금지 요청 준수.
- `SESSION_CONTINUITY.md`: 세션 메모리 규칙은 별도 단일 문서로 관리된다. 이번 QA 문서에는 세션 메모리 규칙을 중복 작성하지 않았다.

## 4. 현재 구현 기능 목록

직접 파일을 읽어 확인한 현재 주요 구현 기능은 다음과 같다.

- 앱 구조: `App.jsx`에서 iPhone 12 Pro 비율의 세로형 게임 화면을 구성하고, `gameKey` 변경 시 Rapier Physics 트리를 재마운트한다.
- 게임 진행: `Game.jsx`에서 `elapsedMs`를 증가시키며 5분 경과 시 스테이지 클리어 상태로 전환한다.
- 플레이어: 이동, 체력, 피격 무적, 카메라 추적 흐름이 구현되어 있다.
- 적 스폰: `Enemies.jsx`에서 시간대별 웨이브와 버스트 이벤트로 E01, E02, E03, E05, E06, B01을 스폰한다.
- 적 행동: `Enemy.jsx`에서 기본 추격형, E05/B01 돌진형, E04 원거리형 코드가 존재한다.
- 보스: B01은 4분에 등장하며, 현재 스탯상 XP는 0이고 추격/돌진 패턴을 사용한다.
- 드랍: 일반 적은 확률적으로 교과서를 드랍하고, E06/B01은 보너스 교과서와 골드를 드랍한다.
- 골드: 시간 기반 골드 코인이 25-35초 간격으로 생성되고, 엘리트/보스 처치 시 추가 골드가 생성된다.
- 레벨업: XP 획득 시 레벨업 상태로 전환되고, HUD에서 최대 3개의 업그레이드 카드를 선택한다.
- 무기: 연필, 30cm 자, 텀블러, 플라스크, 벨, 전기, 보조배터리, 스타링크, 오니기리 구현 파일이 존재한다.
- 업그레이드: `upgrades.js`에서 무기 해금, 피해 증가, 스탯 증가, 플레이어 강화 조건을 관리한다.
- VFX: `VFXLayer.jsx`, `itemEffects.js`, `vfxEvents.js` 기반의 시각 효과 이벤트가 존재한다.
- 리셋: `resetGame`은 Zustand 스토어의 플레이어/무기/상태/시간/보스/세션 골드와 `gameKey`를 초기화한다.

## 6. 심각도별 QA 리스크

심각도 기준:

- Critical: 플레이 진행, 클리어, 재시작이 막히는 수준
- High: 핵심 성장/보상/전투 밸런스가 크게 어긋나는 수준
- Medium: 특정 조건에서 체감 품질이나 회귀 위험이 있는 수준
- Low: 당장 실패는 아니지만 추적할 필요가 있는 수준

### Critical

현재 자동 검증만으로 확정된 Critical 실패는 없다. 단, 아래 High 리스크가 실제 플레이에서 재현되면 Critical로 승격될 수 있다.

### High

1. `gainXp`가 한 번에 한 레벨업만 처리한다.
   - 위치: `Developer/r3f_prototype/src/store/useGameStore.js`
   - 위험: 교과서 여러 개를 빠르게 먹거나 큰 XP를 한 번에 얻으면 XP가 다음 필요량을 넘어도 레벨업이 한 번만 처리될 수 있다.
   - 영향: 성장 템포가 꼬이고, 레벨업 카드 선택 횟수가 실제 XP 누적량보다 적어질 수 있다.
   - QA 기준: 큰 XP를 한 번에 획득했을 때 필요한 만큼 연속 레벨업 처리되거나, 의도적으로 1회만 처리한다면 초과 XP와 다음 레벨업 흐름이 명확해야 한다.

2. B01 보스 보너스 교과서가 `dropData.xp` 값을 사용하지만 B01 XP가 0이다.
   - 위치: `Developer/r3f_prototype/src/components/Enemy.jsx`, `Developer/r3f_prototype/src/components/Enemies.jsx`
   - 위험: B01 처치 시 보너스 교과서 3개가 생성되어도 각 교과서 XP가 0일 수 있다.
   - 영향: 보스를 잡아도 성장 보상이 체감되지 않는다.
   - QA 기준: B01 처치 후 교과서가 실제 XP를 제공하는지 확인해야 한다.

3. 후반 레벨업 카드가 3개 미만이거나 no-op 카드가 남을 수 있다.
   - 위치: `Developer/r3f_prototype/src/components/HUD.jsx`, `Developer/r3f_prototype/src/lib/upgrades.js`
   - 위험: 무기 4개 제한, 무기 레벨 상한, 스탯 상한 이후 선택지가 부족해질 수 있다. no-op은 눌러도 효과가 없는 선택지를 뜻한다.
   - 영향: 후반 성장 선택이 빈약하거나 플레이어가 선택한 보상이 실제로 적용되지 않을 수 있다.
   - QA 기준: 5분 플레이 후반에도 카드 3개가 모두 유효해야 하며, 3개 미만이 의도라면 UI가 자연스럽게 처리해야 한다.

4. `resetGame`/`gameKey` 리셋이 `refs.js` 전역 상태까지 초기화하지 않을 수 있다.
   - 위치: `Developer/r3f_prototype/src/store/useGameStore.js`, `Developer/r3f_prototype/src/lib/refs.js`
   - 위험: Zustand 스토어와 Physics 트리는 초기화되지만 `playerPos`, `playerFacing`, `bagSwingState`, `enemyBodies`, `joystickDir` 같은 전역 참조가 남을 수 있다.
   - 영향: 재시작 후 플레이어 위치, 조이스틱 입력, 적 참조, 자 쿨다운 등이 이전 판의 상태를 끌고 올 수 있다.
   - QA 기준: 게임오버/클리어 후 여러 번 재시작해도 위치, 카메라, 적, 무기 쿨다운, 입력이 모두 초기 상태여야 한다.

### Medium

5. Stage 1에서 E04 투사체 코드는 스폰 테이블 밖에 있지만 실행 가능한 형태로 남아 있다.

- B01 처치 시 골드 5개가 추가 드랍되는지 확인한다.

### 7.8 골드 경제

- 시간 기반 골드 코인이 25-35초 사이 간격으로 생성되는지 확인한다.
- 5분 동안 시간 기반 골드 코인이 최소/평균 몇 개 생성되는지 3회 이상 기록한다.
- 골드 코인 획득 시 세션 골드가 증가하는지 확인한다.
- 게임오버/클리어 화면에 세션 골드와 누적 골드가 맞게 표시되는지 확인한다.
- 브라우저 새로고침 후 누적 골드가 유지되는지 확인한다.

### 7.9 리셋과 반복 플레이

- 게임오버 후 다시 시작을 누른다.
- 클리어 후 다시 시작을 누른다.
- 3회 이상 반복해 플레이어 위치, HP, XP, 레벨, 무기, 적, 골드 세션이 초기화되는지 확인한다.
- 이전 판의 적, 투사체, 골드, 교과서, VFX가 남지 않는지 확인한다.
- `bagSwingState`, `enemyBodies`, `joystickDir` 영향으로 보이는 이상 동작이 없는지 확인한다.

- 게임오버와 클리어 후 재시작이 3회 이상 안정적으로 동작한다.
- XP 획득과 레벨업이 초과 XP 상황에서도 플레이어에게 손해 없이 처리된다.
- B01 처치 보상이 실제 XP와 골드로 반영된다.
- 후반 레벨업 카드가 모두 유효하거나, 카드 수가 부족할 때 UI가 명확하게 처리한다.
- Stage 1에서 E04가 등장하지 않는다.
- 시간 기반 골드 코인 수량이 기획 의도 범위 안에 있다.

### 실패 기준

- 자동 테스트 또는 빌드가 실패한다.
- 5분 전에 게임 진행이 멈추거나 조작이 불가능해진다.
- 레벨업 모달에서 선택 가능한 유효 카드가 없는데 진행이 막힌다.
- 보스 보상 교과서가 XP 0으로 동작한다.
- 재시작 후 이전 판의 위치, 적, 입력, VFX, 쿨다운 상태가 남는다.
- Stage 1에서 E04가 의도 없이 등장하거나 투사체가 플레이어를 공격한다.
- 5분 보상 체감이 기획 의도보다 현저히 낮고 반복 재현된다.

## 9. 다음 단계 권장

1. QA 우선순위 1: `gainXp` 초과 XP 처리와 연속 레벨업 재현 테스트를 추가한다.
2. QA 우선순위 2: B01 처치 후 교과서 XP가 실제로 증가하는지 수동 플레이 또는 테스트 훅으로 검증한다.
3. QA 우선순위 3: 후반 레벨업 카드 풀이 고갈되는 시뮬레이션 테스트를 작성한다.
4. QA 우선순위 4: 재시작 3회 반복 수동 테스트로 `refs.js` 전역 상태 잔류 여부를 확인한다.
5. QA 우선순위 5: Stage 1 5분 플레이 중 E04 미등장 여부를 확인하고, 회귀 방지 테스트 또는 명시적 가드를 검토한다.
6. QA 우선순위 6: 시간 기반 골드 코인 생성 횟수를 3회 이상 기록해 최소 보상 체감을 판단한다.
7. 성능 후속: 빌드 번들 크기 경고에 대해 모바일 로딩 시간과 프레임 드랍을 별도 성능 QA로 분리한다.

## 10. 현재 결론

현재 자동 검증 기준으로는 테스트와 빌드가 통과했다. 그러나 자동 테스트 범위가 업그레이드 유틸에 치우쳐 있어 실제 Stage 1 전체 플레이 품질을 보장하지는 않는다.

가장 큰 QA 리스크는 XP/레벨업, B01 보상, 후반 카드 유효성, 재시작 초기화다. 이 4개는 플레이어 성장과 반복 플레이 안정성에 직접 연결되므로 다음 검증 순서에서 먼저 확인해야 한다.
