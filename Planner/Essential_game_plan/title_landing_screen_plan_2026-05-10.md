# Escape! zombie school 타이틀/랜딩 화면 구성기획안

작성일: 2026-05-10  
문서 위치: `Planner/`  
대상: Escape! zombie school 1차 서비스용 타이틀 화면 / 랜딩 화면  
핵심 주제: **Escape! zombie school**

## 1. 기획 목적

타이틀 화면은 게임을 실행한 유저가 1초 안에 다음 내용을 이해하게 만드는 첫 화면이다.

- 여기는 평범한 학교가 아니라 좀비에 감염된 학교다.
- 플레이어는 학생 생존자다.
- 목표는 학교에서 탈출하는 것이다.
- 게임 분위기는 실사 공포가 아니라 카툰풍 감염 학교 탈출 액션이다.

따라서 타이틀 문구는 반드시 **"Escape! zombie school"** 이 전면에 보여야 한다.

## 2. 화면 콘셉트

### 핵심 콘셉트

**감염된 학교 정문 앞, 탈출 직전의 학생 생존자**

화면은 학교 정문 또는 복도 출입구를 배경으로 잡는다. 문 뒤쪽에서는 감염된 학생 좀비들이 몰려오고, 앞쪽에는 주인공이 책가방과 연필을 들고 도망칠 준비를 하고 있다.

이 장면은 게임의 내용을 그대로 압축한다.

- 학교: 교문, 복도, 교실 문, 안내판, 신발장, 책상 잔해
- 감염: 초록 감염 액체, 금 간 바닥, 경고 테이프, 비상등
- 탈출: 열린 문, 바깥 빛, 화살표 안내판, 주인공의 도망 방향
- 액션: 몰려오는 좀비, 주인공의 준비 자세, 무기 실루엣

## 3. 타이틀 문구

### 메인 타이틀

```text
Escape! zombie school
```

### 보조 문구 후보

1. `5분만 버티면, 교문이 열린다`
2. `감염된 학교에서 살아남아라`
3. `책가방 하나 들고 시작하는 탈출 액션`
4. `종이 울리기 전까지 도망쳐라`

1차 추천:

```text
5분만 버티면, 교문이 열린다
```

이 문구는 현재 5분 스테이지 구조와 가장 잘 맞는다.

## 4. 첫 화면 구성

### 4-1. 전체 레이아웃

모바일 세로 화면 9:16 기준.

| 영역 | 화면 위치 | 내용 |
| --- | --- | --- |
| 상단 15% | 위쪽 | 작은 로고 또는 서비스명 보조 표기 |
| 중앙 35% | 중앙 상단 | 메인 타이틀 `Escape! zombie school` |
| 중앙 30% | 중앙 | 주인공 + 감염 학교 배경 + 좀비 실루엣 |
| 하단 20% | 하단 | 시작 버튼, 이어하기, 설정 |

### 4-2. 메인 타이틀 배치

- 타이틀은 화면 중앙 상단에 크게 배치한다.
- 글자는 두껍고 읽기 쉬워야 한다.
- `Escape!`와 `zombie school`을 줄바꿈한다.

권장 형태:

```text
Escape!
zombie school
```

이유:

- 모바일 세로 화면에서 글자가 커진다.
- "Escape!"가 목표 행동으로 먼저 읽힌다.
- "zombie school"이 감염된 학교 콘셉트를 바로 전달한다.

### 4-3. 버튼 구성

필수 버튼:

```text
게임 시작
```

보조 버튼:

```text
이어하기
설정
```

1차 구현에서는 아래처럼 단순하게 간다.

```text
[ 게임 시작 ]
[ 설정 ]
```

`이어하기`는 저장 데이터가 있을 때만 표시한다.

## 5. 비주얼 연출

### 5-1. 배경

권장 배경:

- 어두운 학교 복도 또는 학교 정문
- 바닥에는 금 간 타일과 초록 감염 흔적
- 벽에는 `출입금지`, `감염주의`, `비상구` 같은 학교 안내판
- 화면 끝에는 좀비들이 몰려오는 그림자

주의:

- 너무 진짜 공포 영화처럼 만들지 않는다.
- 피, 잔혹 표현, 실사 공포보다 카툰 액션 느낌을 우선한다.
- 배경이 너무 복잡해서 타이틀 글자가 묻히면 안 된다.

### 5-2. 주인공

주인공은 화면 하단 중앙 또는 우측 하단에 배치한다.

모습:

- 교복 또는 학교 느낌의 복장
- 파란색 또는 하늘색 포인트
- 책가방 착용
- 손에는 연필 또는 30cm 자
- 도망치기 직전의 자세

역할:

- 유저가 조작할 캐릭터임을 알려준다.
- 좀비와 색상으로 확실히 구분된다.

### 5-3. 좀비

좀비는 배경 중후경에 배치한다.

모습:

- 감염된 학생 좀비 3~5명
- 큰 체육교사 좀비 1명 실루엣
- 눈 또는 감염 부위에 초록/보라 포인트

주의:

- 타이틀 화면에서는 좀비가 주인공을 완전히 가리면 안 된다.
- 좀비는 무섭지만 귀엽고 읽기 쉬운 카툰 실루엣이어야 한다.

## 6. 색상 방향

기존 그래픽 팔레트를 따른다.

| 역할 | 색상 방향 | 예시 |
| --- | --- | --- |
| 플레이어 | 하늘색/파란색 | `#59c7ff` |
| 감염 흔적 | 탁한 초록 | `#41745a`, `#95bf91` |
| 위험/경고 | 주황/붉은색 | `#e99039`, `#d32836` |
| 배경 | 어두운 회색/나무색 | `#2d2738`, `#805947` |
| 타이틀 글자 | 밝은 크림색 + 어두운 외곽선 | `#f8f7f2`, `#050209` |

타이틀 글자는 밝은 크림색을 기본으로 하고, 어두운 외곽선을 넣는다. `탈출` 단어에는 주황색 또는 붉은색 강조를 줄 수 있다.

## 7. 애니메이션 방향

1차 구현은 과하지 않게 간다.

권장 애니메이션:

- 타이틀 글자: 0.4초 동안 살짝 커지며 등장
- 배경 비상등: 1.2초 간격으로 약한 붉은 점멸
- 감염 액체: 천천히 흐르는 듯한 초록 빛
- 주인공: idle 숨쉬기 애니메이션
- 좀비 실루엣: 느리게 흔들리는 움직임
- 시작 버튼: 약한 하늘색 테두리 펄스

금지:

- 전체 화면을 계속 번쩍이게 만들지 않는다.
- 타이틀 글자를 너무 흔들어 읽기 어렵게 하지 않는다.
- 시작 버튼보다 장식 효과가 더 눈에 띄면 안 된다.

## 8. UX 흐름

### 최초 진입

1. 게임 로고/타이틀 등장
2. 배경과 캐릭터가 0.5초 안에 보임
3. `게임 시작` 버튼 활성화
4. 유저가 누르면 바로 Stage 1 시작

### 저장 데이터가 있을 때

```text
[ 이어하기 ]
[ 새로 시작 ]
[ 설정 ]
```

단, 1차 서비스에서는 저장 구조가 단순할 수 있으므로 기본은 `게임 시작` 하나를 우선한다.

## 9. 사운드 방향

타이틀 화면 사운드가 들어간다면 다음 방향을 추천한다.

- 멀리서 들리는 학교 종소리
- 낮게 깔리는 감염된 학교 분위기음
- 좀비 소리는 아주 작게, 반복 피로감이 없게
- 버튼 선택음은 밝고 짧게

메인 사운드 콘셉트:

```text
익숙한 학교 종소리가 이상하게 뒤틀려 들리는 느낌
```

## 10. 구현 방식 제안

현재 프로젝트가 R3F 기반이므로 타이틀 화면도 같은 기술 흐름을 유지한다.

권장 구현:

- React DOM: 타이틀 글자, 버튼, 메뉴
- R3F/Three.js: 배경 3D 오브젝트, 주인공, 좀비 실루엣
- Zustand: `phase = 'title'` 상태 추가

예상 파일:

```text
Developer/r3f_prototype/src/components/TitleScreen.jsx
Developer/r3f_prototype/src/components/TitleScene3D.jsx
Developer/r3f_prototype/src/store/useGameStore.js
```

구현 단계:

1. `phase: 'title'` 추가
2. 앱 시작 시 TitleScreen 표시
3. `게임 시작` 클릭 시 기존 `playing` 상태로 전환
4. TitleScene3D에 주인공/좀비/학교문 배치
5. 이후 설정/이어하기 버튼 추가

## 11. 화면 카피 최종안

### 추천안 A: 가장 명확한 버전

```text
Escape! zombie school
탈출

5분만 버티면, 교문이 열린다

[ 게임 시작 ]
```

### 추천안 B: 액션 강조 버전

```text
Escape! zombie school
탈출

책가방 들고 튀어라!

[ 게임 시작 ]
```

### 추천안 C: 학교 분위기 강조 버전

```text
Escape! zombie school
탈출

종이 울리면, 감염자들이 몰려온다

[ 게임 시작 ]
```

1차 서비스 추천은 **A안**이다.

이유:

- 5분 생존 규칙을 바로 전달한다.
- 탈출 목표가 명확하다.
- 과하게 장난스럽지 않아 게임 분위기와 맞는다.

## 12. 구현 수용 기준

타이틀 화면 구현 완료 기준:

- `Escape! zombie school`이 첫 화면에서 가장 먼저 읽힌다.
- 유저가 게임 목표를 "감염된 학교에서 탈출"로 이해한다.
- `게임 시작` 버튼을 1초 안에 찾을 수 있다.
- 모바일 9:16 화면에서 글자와 버튼이 겹치지 않는다.
- 주인공, 좀비, 학교 배경이 모두 보인다.
- 실사 공포가 아니라 카툰풍 감염 학교 액션으로 보인다.
- 시작 버튼 클릭 후 Stage 1로 자연스럽게 진입한다.
