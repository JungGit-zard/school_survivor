# Escape! zombie school 패시브 업그레이드 / 코인 경제 기획

작성일: 2026-05-17  
작성 위치: `Planner/Essential_game_plan/`  
목적: 한 판에서 벌 수 있는 코인량, 코인상점 가격, 추천 구매 순서, 주인공 영구 능력치를 정의한다.

---

## 1. 기획 결론

한 판 5분 완주 기준 기대 코인은 **약 32개**로 잡는다.

| 플레이 결과 | 예상 코인 | 설명 |
|---|---:|---|
| 1분 전 사망 | 0-2 | 시간 코인을 거의 못 먹는 실패 구간 |
| 1분 생존 | 3-5 | 시간 코인 약 2-4 + 1분 보너스 1 |
| 3분 생존 | 10-14 | 시간 코인 약 6 + 마일스톤 1+3 |
| 4분 보스 조우 | 16-22 | 시간 코인 약 8 + 마일스톤 1+3+4 + E06 보너스 가능 |
| 5분 클리어, 보스 미처치 | 25-29 | 시간 코인 약 9-13 + 마일스톤 16 |
| 5분 클리어, 보스 처치 | **30-34** | 위 보상 + B01 보스 코인 5 |

정본 목표값:

```text
일반 완주 기대값 = 28코인
좋은 완주 기대값 = 32코인
숙련 완주 기대값 = 34코인 이상
```

초보자가 한 판을 망쳐도 3-5코인은 남고, 5분을 버티면 최소 1개 업그레이드가 가능한 구조를 목표로 한다.

## 2. 현재 코인 수입 근거

현재 구현/기획 기준:

| 수입원 | 수량 | 비고 |
|---|---:|---|
| 시간 기반 코인 | 약 10 | 25-35초마다 1개, 5분 평균 약 10개 |
| 생존 마일스톤 | 16 | 1분 +1, 3분 +3, 4분 +4, 5분 +8 |
| E06 엘리트 | +1 | 처치 시 보너스 |
| B01 보스 | +5 | 처치 시 보너스 |
| 합계 | 약 27-32 | 보스 처치 여부에 따라 변동 |

설계상 중요한 점:

- 코인은 킬 수보다 **생존 시간**에 더 크게 묶는다.
- 이유: 출근길 타깃은 갑자기 끊겨도 보상이 남아야 한다.
- 보스 처치 보너스는 숙련 보상으로 두되, 보스 미처치 클리어도 손해가 너무 크면 안 된다.

## 3. 뱀파이어 서바이버 참고 변환

Vampire Survivors의 PowerUp 구조를 그대로 복사하지 않고, Escape! zombie school의 5분 모바일 세션에 맞게 줄인다.

| V.S 계열 개념 | 이 게임 변환 | 초보 설명 |
|---|---|---|
| Might | 공격력 | 모든 무기 데미지가 조금 오른다 |
| Armor | 방어력 | 맞을 때 받는 피해가 줄어든다 |
| Move Speed | 이동속도 | 좀비 사이를 빠져나가기 쉬워진다 |
| Magnet | 회수 반경 | 교과서/코인이 더 멀리서 끌려온다 |
| Growth | XP 획득량 | 레벨업이 빨라진다 |
| Greed | 코인 획득량 | 한 판에서 코인을 더 번다 |
| Cooldown | 쿨다운 감소 | 무기가 더 자주 발동한다 |
| Amount | 발사체/타격 수 | 일부 무기가 더 많이 나간다 |
| Luck | 선택 운 | 좋은 카드/추가 선택지 확률 후보 |
| Curse | 위험도 증가 | 1차 상점에서는 제외. 초보자에게 불친절함 |

1차 서비스에서는 복잡한 능력치를 줄여 **8개 패시브**만 쓴다.

## 4. 주인공 영구 능력치 1차안

### 4-1. 1차 필수 패시브 8종

| 키 | 이름 | 최대 레벨 | 1레벨당 효과 | 최대 효과 | 역할 |
|---|---|---:|---|---|---|
| `might` | 공격력 | 5 | 무기 데미지 +4% | +20% | 처치 속도 증가 |
| `armor` | 방어력 | 5 | 받는 접촉 피해 -1 | -5 | 실수 완충 |
| `maxHp` | 체력 | 5 | 최대 HP +6 | +30 | 초보 생존 |
| `moveSpeed` | 이동속도 | 5 | 이동속도 +3% | +15% | 포위 탈출 |
| `magnet` | 회수 반경 | 5 | 교과서/코인 흡수 반경 +8% | +40% | 성장 누락 감소 |
| `growth` | 학습력 | 5 | XP 획득량 +5% | +25% | 레벨업 가속 |
| `greed` | 저금통 | 5 | 코인 획득량 +5% | +25% | 반복 플레이 보상 |
| `cooldown` | 손놀림 | 5 | 무기 쿨다운 -3% | -15% | 전체 화력 상승 |

### 4-2. 2차 후보

| 키 | 이름 | 이유 |
|---|---|---|
| `amount` | 추가 발사 | 강하지만 밸런스 영향이 커서 2차 |
| `luck` | 행운 | 4번째 카드 확률과 연결 가능하지만 UI 설명이 필요 |
| `revive` | 보건실 찬스 | 광고/리워드 설계와 충돌 가능 |
| `reroll` | 카드 다시 뽑기 | 5분 세션에는 편하지만 초기 UI가 복잡해짐 |

## 5. 가격 곡선

한 판 완주 기대값을 32코인으로 잡으면, 첫 구매는 1판 안에 가능해야 한다. 대신 후반 레벨은 여러 판을 요구한다.

### 5-1. 기본 가격표

| 레벨 | 가격 | 완주 기준 체감 |
|---:|---:|---|
| Lv.1 | 20 | 첫 완주 전후로 바로 구매 |
| Lv.2 | 45 | 1-2판 추가 |
| Lv.3 | 90 | 누적 4-5판 |
| Lv.4 | 160 | 목표를 갖고 모으는 구간 |
| Lv.5 | 260 | 장기 성장 보상 |
| 합계 | 575 | 한 능력 만렙 약 18판 완주 |

### 5-2. 능력별 가격 보정

일부 능력은 게임 영향이 크므로 기본 가격에 배율을 둔다.

| 능력 | 가격 배율 | 이유 |
|---|---:|---|
| 체력 / 방어력 / 자석 | ×1.0 | 초보자에게 필요한 기본 안정성 |
| 이동속도 / 학습력 / 저금통 | ×1.1 | 성장 속도와 경제에 영향 |
| 공격력 / 쿨다운 | ×1.25 | 전투 밸런스 영향이 큼 |

예시:

- `magnet` Lv.1 = 20코인
- `moveSpeed` Lv.1 = 22코인
- `might` Lv.1 = 25코인

## 6. 추천 구매 순서

### 6-1. 초보자 추천

| 순서 | 구매 | 이유 |
|---:|---|---|
| 1 | `magnet` Lv.1 | 교과서와 코인을 덜 놓쳐서 성장 체감이 바로 좋아진다 |
| 2 | `moveSpeed` Lv.1 | 포위됐을 때 빠져나오기 쉬워진다 |
| 3 | `maxHp` Lv.1 | 실수 한두 번을 버틴다 |
| 4 | `might` Lv.1 | 기본 연필 처치 속도가 오른다 |
| 5 | `growth` Lv.1 | 레벨업이 빨라져 무기 빌드가 안정된다 |

초보자 첫 5판 목표:

```text
magnet 1
moveSpeed 1
maxHp 1
might 1
growth 1
```

이 구성이 가장 무난하다. 이유는 "더 세게 때리기"보다 "아이템을 놓치지 않고, 포위에서 빠져나오고, 실수를 버티는 것"이 초반 완주율을 더 크게 올리기 때문이다.

### 6-2. 공격형 추천

```text
might 1 -> cooldown 1 -> magnet 1 -> growth 1 -> might 2
```

숙련 플레이어용. 적을 빨리 지우면 안전해지지만, 초보자에게는 이동 실수 보완이 부족하다.

### 6-3. 파밍형 추천

```text
magnet 1 -> greed 1 -> growth 1 -> magnet 2 -> greed 2
```

반복 플레이 보상을 빨리 늘리고 싶은 플레이어용. 단, 전투력이 늦게 올라 4분 이후가 어려울 수 있다.

## 7. 한 판 보상과 구매 체감 시뮬레이션

### 7-1. 평균 완주 플레이어

가정: 한 판 30코인 획득.

| 누적 판수 | 누적 코인 | 권장 구매 |
|---:|---:|---|
| 1판 | 30 | `magnet` Lv.1 구매 |
| 2판 | 40 남음 | `moveSpeed` Lv.1 또는 `maxHp` Lv.1 구매 |
| 3판 | 약 48 남음 | `growth` Lv.1 구매 |
| 4판 | 약 56 남음 | `armor` Lv.1 구매 |
| 5판 | 약 66 남음 | `might` Lv.1 구매 |

첫 5판 안에 4-5개 능력의 Lv.1을 열 수 있다. 이 정도가 모바일 5분 게임의 초반 유지율에 적당하다.

### 7-2. 자주 사망하는 플레이어

가정: 한 판 8-12코인 획득.

- 2-3판에 첫 업그레이드 1개 구매 가능.
- 첫 구매는 반드시 `magnet` 또는 `maxHp`가 좋다.
- 실패해도 코인이 남으므로 "아예 손해 본 판" 느낌을 줄인다.

### 7-3. 숙련 플레이어

가정: 한 판 34코인 이상 획득.

- 1판마다 Lv.1급 구매 가능.
- 3-4판마다 Lv.2 구매 가능.
- 보스 처치 보너스가 장기 성장 속도를 확실히 앞당긴다.

## 8. UI 기획

### 8-1. 코인상점 카드 정보

각 카드에는 아래만 표시한다.

```text
공격력 Lv.1
무기 데미지 +4%
20 코인
[구매]
```

초보자용 설명 원칙:

- `Might`, `Growth` 같은 영어 능력명은 내부 키로만 쓴다.
- 화면에는 `공격력`, `학습력`, `저금통`처럼 바로 이해되는 이름을 쓴다.
- 숫자는 "현재 효과 → 다음 효과"로 보여준다.

예시:

```text
회수 반경 Lv.2
교과서/코인 흡수 반경 +8% -> +16%
45 코인
```

### 8-2. 구매 제한

- 코인이 부족하면 버튼은 비활성화.
- 최대 레벨이면 `최대` 표시.
- 구매 직후 `goldTotal`에서 즉시 차감.
- 능력 레벨은 `localStorage`에 저장.

권장 저장 키:

```text
school_survivor:passiveUpgrades
```

형식:

```json
{
  "might": 1,
  "armor": 0,
  "maxHp": 1,
  "moveSpeed": 1,
  "magnet": 1,
  "growth": 0,
  "greed": 0,
  "cooldown": 0
}
```

## 9. 구현 적용 기준

### 9-1. 런 시작 시 적용

게임 시작 시 `passiveUpgrades`를 읽어서 기본 능력치에 반영한다.

| 능력 | 적용 위치 후보 |
|---|---|
| `might` | 무기 데미지 계산 공통 헬퍼 |
| `armor` | 플레이어 피격 처리 |
| `maxHp` | `useGameStore` 초기 HP |
| `moveSpeed` | `Player.jsx` 이동 속도 |
| `magnet` | `lib/pickup.js` 흡수 반경 |
| `growth` | `gainXp` 또는 교과서 XP 계산 |
| `greed` | `gainGold` 계산 |
| `cooldown` | 무기 쿨다운 계산 공통 헬퍼 |

### 9-2. 밸런스 안전장치

- `might`와 `cooldown`은 합쳐서 초반 화력을 너무 크게 올리지 않게 최대치 제한.
- `armor`는 피해를 0으로 만들 수 없다. 최소 1 피해는 남긴다.
- `moveSpeed`는 최대 +15%까지만. 너무 높으면 조작이 어려워진다.
- `greed`는 보상 경제를 망가뜨리지 않도록 1차 최대 +25%.

## 10. 최종 추천안

1차 구현 우선순위:

1. `magnet`
2. `moveSpeed`
3. `maxHp`
4. `might`
5. `growth`
6. `armor`
7. `cooldown`
8. `greed`

1차 구현에서 반드시 넣을 최소 세트:

```text
magnet / moveSpeed / maxHp / might / growth
```

상점 가격의 기준:

```text
한 판 평균 32코인
첫 구매 20-25코인
Lv.2 구매 45-55코인
한 능력 만렙 약 575-720코인
```

이 구조는 한 판만 해도 성장 보상이 보이고, 5판 정도 하면 주인공이 확실히 편해지며, 장기적으로는 여러 능력을 천천히 채우는 방향이다.

## 11. QA 체크리스트

- [ ] 5분 완주 3회 평균 코인이 28-34 범위에 들어오는가.
- [ ] 3분 사망 플레이어도 10코인 전후를 얻는가.
- [ ] 첫 업그레이드가 1-2판 안에 가능한가.
- [ ] `magnet` Lv.1 구매 후 교과서/코인 회수가 체감되는가.
- [ ] `maxHp` 또는 `armor` Lv.1 구매 후 초보자가 한 번 더 버티는가.
- [ ] `might` + `cooldown`을 동시에 올려도 0-60초 구간이 너무 쉬워지지 않는가.
- [ ] `greed`를 올린 계정의 5분 코인 수입이 지나치게 폭증하지 않는가.
