# 타이틀 화면 다중 에이전트 구상 검토

작성일: 2026-05-23  
대상: `Escape! zombie school` 타이틀 화면  
참여 관점: 그래픽디자이너, 기획자, 개발자  
방법론: Agent Room routing + Superpowers brainstorming

## 집행자 판단

이번 안건은 새 타이틀 화면 구상과 UX 방향을 정리하는 작업이다.

따라서 1차 방법론은 `Superpowers brainstorming`이고, 서브에이전트는 다음 역할로 사용했다.

- `graphic_designer`: 비주얼 콘셉트, 시선 흐름, 카툰 렌더링 방향
- `product-manager`: 유저 이해, MVP 범위, 카피와 버튼 구성
- `game-developer`: 현재 R3F 구조, 구현 위험, 모바일 성능과 검증

서브에이전트는 방법론을 대체하지 않고, 기존 기획과 프로젝트 정책 안에서 의견을 냈다.

## 공통 결론

타이틀 화면은 마케팅 랜딩 페이지가 아니라 `첫 판 진입을 빠르게 만드는 압축 화면`이어야 한다.

유저가 1초 안에 이해해야 할 내용:

- 학교가 감염됐다.
- 주인공은 학생 생존자다.
- 목표는 학교에서 탈출하는 것이다.
- 한 판은 5분 생존 구조다.
- 지금 누를 것은 `게임 시작`이다.

## 추천 콘셉트

추천 장면:

```text
거의 열린 교문 또는 출구 빛 앞에서,
학생 생존자가 탈출하려 몸을 틀고 있고,
뒤쪽 복도 양옆에서 감염 학생들이 몰려오는 장면
```

핵심은 `출구`, `학생 생존자`, `감염 학생`, `게임 시작`이 한 화면에서 바로 읽히는 것이다.

## 학생 주인공 외형 참고

첨부한 참고 이미지의 학생은 다음 인상을 타이틀 주인공에 반영한다.

- 분홍색 머리
- 붉은 교복 재킷
- 흰 셔츠와 리본
- 파란 체크 치마
- 파란 책가방
- 좀비에게 쫓기며 앞으로 뛰는 긴박한 표정

다만 이미지를 그대로 복제하지 않고, 프로젝트 규칙에 맞춰 3D 카툰 렌더링 주인공으로 단순화한다.

기획상 중요한 점은 `예쁜 학생 캐릭터`보다 `겁먹었지만 도망치는 학생 생존자`로 보이는 것이다. 첫 화면에서 유저가 “이 캐릭터를 조작해서 학교를 탈출하는구나”라고 바로 이해해야 한다.

## 추천 카피

1차 추천:

```text
Escape! zombie school
감염된 학교에서 5분만 버티면, 교문이 열린다
[ 게임 시작 ]
```

짧은 대안:

```text
Escape! zombie school
5분만 버텨. 교문이 열린다
[ 게임 시작 ]
```

추천은 첫 번째다. 세계관, 목표, 플레이 시간 약속이 한 줄에 같이 들어가기 때문이다.

## MVP 버튼 구성

필수:

- `게임 시작`

조건부:

- 저장 데이터가 있을 때만 `이어하기`

후순위:

- `설정`

MVP에서 제외:

- 코인상점
- 업적
- 도감
- 랭킹
- 출석 보상
- 긴 조작 설명

코인상점은 타이틀이 아니라 `한 판 끝남 -> 획득 코인 확인 -> 상점 진입` 흐름에 둔다.

## 처음 3초 경험

0~1초:

- 타이틀과 감염 학교 분위기가 먼저 읽힌다.
- 유저가 `좀비 학교 탈출 게임`이라고 이해한다.

1~2초:

- 부제에서 `5분`과 `교문`을 읽는다.
- 한 판 길이가 짧다는 인상을 받는다.

2~3초:

- 시선이 하단 `게임 시작` 버튼으로 내려간다.
- 메뉴 탐색보다 첫 판 시작을 먼저 하게 된다.

## 성공 기준

- 첫 화면에서 `Escape! zombie school`이 가장 먼저 읽힌다.
- 유저가 `감염된 학교에서 탈출하는 게임`이라고 이해한다.
- `게임 시작` 버튼을 1초 안에 찾을 수 있다.
- 모바일 9:16 화면에서 타이틀, 캐릭터, 버튼이 겹치지 않는다.
- 코인상점이 없어도 첫 판 진입이 자연스럽다.
- 실사 공포가 아니라 3D 카툰 감염 학교 액션으로 보인다.
