## 4. 코드: 오프닝 프론트로드 (Enemies.jsx, stage3 한정)

`waveSizeForStageAtTime`에 stage3 분기 추가. 스1(×1.3)·스2(×3) 분기는 **거동 불변**.

```
if (stageId === 'stage3') return waveTime === 0 ? size * 2 : size
```

- t=0 웨이브만 ×2(그 외 웨이브는 배율 없음). 스3 오프닝 = 10→20 + 버스트 E01×12 = 32 → 스1 오프닝(32)·스2(37)과 동급으로 정렬(발견 C 오프닝 희박 해소).
- 20~40s 랜덤 간격상 0~16s에는 t=0 웨이브만 발화하므로(다음 웨이브는 최소 20s 후) 프론트로드는 정확히 첫 웨이브 1회만 적용된다.
- 지속 배율(sustained multiplier)은 **채택하지 않음**: ×1.44 HP가 이미 처치시간 +44%로 동시개체를 끌어올렸으므로(발견 D), 지속 배율까지 곱하면 108~150 스택 과부하 위험. 밀도는 프론트로드(오프닝)와 target(곡선)으로만 조절해 수치가 투명하도록 유지.

## 5. 25초 프리보스 고정 스케줄(발견 B) — 폐기 판정

`Planner/stage3_preboss_25s_spawn_schedule_plan_2026-07-16.md`(0/25/50/75/100/125 고정 75마리)는 **이번 재설계에서 채택하지 않는다.**

폐기 사유:
1. **목적 이미 충족.** 그 안의 핵심 목적(위협 타입 첫 조우 운 누락 방지)은 현행 결정론 버스트(E04@34·E05@52·E06@108/132)가 이미 보장한다(분석 발견 ⑤ 미조우 0%).
2. **동시개체 폭주 미확인.** 07-12 시뮬 피크 ~22, 폭주 근거 없음. 폭주는 발견 D(×1.44 재측정)에서 확인될 때만 실재 → 선제 재구축은 근거 부족.
3. **구현 대비 이득 과소.** 고정 스케줄은 별도 정본 데이터(`stage3PrebossSchedule`) + 스케줄러 분기 + 상한/우선순위 로직(문서 82~87행)까지 상당한 신규 구현. 반면 이번 재설계는 오프닝 프론트로드 1줄 + 형태 2개 교체 + 곡선 재계산으로 동일한 "초반 리듬·밀도" 목표를 달성한다(YAGNI).

**재검토 조건**: balanceqa의 ×1.44 동시개체 재측정(발견 D)에서 108~150 스택 과부하가 확인되면, 그때 랜덤+버스트 중첩을 25초 고정 쿼터로 대체하는 안을 재개한다. 그전까지는 랜덤 웨이브 + 결정론 버스트 구조를 유지한다.
