## 1. 목표

스테이지 내 성장(임시)과 계정 영구 성장(반복 플레이 보상)을 시각·기능적으로 분리해서:

- **레벨업 동기**(스테이지 안)
- **반복 플레이 동기**(스테이지 클리어 후)

가 서로 침해하지 않도록 한다.

---

## 2. 두 재화 정의

| 재화 | 키 | 역할 | 보존 범위 | 그래픽 |
|---|---|---|---|---|
| 교과서 (구 도라야키) | `XpTextbook` 컴포넌트 → `gainXp` | 스테이지 내 XP / 레벨업 | 스테이지 종료 시 소멸 | 파란 책 (표지+책등+크림 책장) |
| 황금 코인 | `GoldCoin` 컴포넌트 → `gainGold` | 계정 영구 패시브 재화 | 영구 (계정 단위, `localStorage`) | 기존 GoldCoin 그래픽 |

> 명칭 변경 이유: "도라야키"는 학교 생존 컨셉과 어울리지 않아 구현 단계에서 **교과서**로 재매핑.
> 게임플레이 의미는 동일 — 적 처치 시 일정 확률로 드랍되는 즉시 흡수형 XP 오브.

---

## 3. 드랍 규칙

### 3-1. 교과서 (XP 재화) — 킬당 확률 드랍

- **드랍률: 30%** — 몬스터 처치 시 30% 확률로 1개 드랍. (`Enemies.jsx` `TEXTBOOK_DROP_RATE = 0.30`)
- 보스 / 엘리트는 별도 정책 (아래 3-3).

### 3-2. 황금 코인 — 시간 기반 시계 드랍

핵심 의도: **"5분 동안 약 10코인"** — 킬 수 무관, 시간으로 보장.

- **목표 평균 간격: 30초 ± 5초** (균등 무작위 25–35s). 코드: `GOLD_INTERVAL_MIN_MS = 25_000`, `GOLD_INTERVAL_MAX_MS = 35_000`.
- **누적 목표**: 300초 / 30초 = **약 10개 / 5분**.
- 트리거: `useFrame`에서 `goldTimerRef`가 만료되면 즉시 1개 출현, 다음 간격을 25–35s 무작위로 재설정.

#### 3-2-1. 출현 위치 정책 (구현됨)

타이머 만료 시점에:
1. **플레이어 반경 10 units (`GOLD_VISIBLE_RADIUS = 10`) 안 살아있는 적이 있으면** 그 중 무작위 1체 위치에 드랍.
2. 적이 없으면 **플레이어 주변 3.0–6.0 units 무작위 링** 위치에 드랍 (`pickGoldDropPos` 폴백).

> 사용자 의도 ("몇 킬을 하든 시간 기준") 충족.

### 3-3. 보스 / 엘리트 보너스 (구현됨)

`Enemies.jsx` `ELITE_BONUS` 상수.

| 적 | 교과서 | 황금 코인 | 비고 |
|---|---|---|---|
| E06 (거대 엘리트) | 100% (1개, XP = 적 기본 xp값 = 56) | 시계 타이머 무관 100% (1개) | 큰 보상감 |
| B01 (보스) | 100% (3개, **개당 XP 40 고정**) | 100% (5개) | 클리어 보너스. `B01.xp = 0`이므로 보너스용 고정값을 따로 사용 |

> 보스 교과서는 `getEliteBonusTextbookXp(type, fallbackXp)` 헬퍼로 `bonus.textbookXp ?? fallbackXp` 처리.
> B01만 `textbookXp: 40`을 명시한다.

## 4. 보존·소비 정책 (구현됨)

| 재화 | 인게임 사용 | 스테이지 종료 처리 |
|---|---|---|
| 교과서 | 자석 흡수 → `gainXp(value)` → 누적/레벨업 | 결과창 표시 없음 (XP/Lv는 HUD에 상시 노출) |

- `resetGame`은 `goldSession`/`survivalMilestonesHit`만 초기화, `goldTotal`은 유지.

## 5. 그래픽 사양

### 5-1. 교과서 (신규 — 구 도라야키)

탑다운에서 보이는 단순화 책 형태. (`XpTextbook.jsx`)

| 항목 | 값 |
|---|---|
| 외형 | 표지(상) + 책장(하) + 책등(왼쪽 짧은 변) + 제목 띠 |
| 표지 색 (`coverMat`) | `#2e5fb2` 파랑 toon |
| 책등 색 (`spineMat`) | `#1c3f7e` 짙은 파랑 |
| 책장 색 (`pageMat`) | `#f2eddc` 크림 |
| 제목 띠 (`titleMat`) | `#fff7e0` 아이보리 + 내부 짙은 줄 `#1a2e58` |
| 외곽선 | `outlineMat(0.96)` (기존 toon 일관) |
| 크기 | `boxGeometry(0.36, 0.04, 0.26)` 표지 + 책장, 그룹 `scale 0.8` |
| 모션 | 둥둥 플로팅 (Y/Z 사인 합성) + Y축 회전 (0.6 rad/s) |
| 위치 Y | `FLOOR_Y = 0.13` |

> 기획 단계 안(단팥빵 갈색 디스크)은 폐기. 학교 생존 컨셉 일관성을 위해 교과서로 교체.

### 5-2. 황금 코인 (기존 재사용)

- `GoldCoin.jsx` 모델 그대로 사용 — 파티클 6개 스폰 + 포물선 낙하 + 바운스 + 회전 + 자석 흡수.
- 흡수 시 `gainGold(value)` (기본 `value=1`).
- **변경 사항(완료)**: 의미가 "XP 오브" → "영구 골드"로 바뀜.

## 6. 5분 플레이 평균 시나리오

가정: 평균 적 처치율 약 50–80 마리/분.

| 항목 | 값 | 비고 |
|---|---|---|
| 5분간 처치 | 250–400 마리 | `Enemies.jsx` 스폰 곡선 기준 |
| 교과서 드랍 | 75–120개 (30%) | 개별 XP 값은 §7 |
| 황금 코인 (시계) | 약 10개 | 25–35s × 10회 |
| 황금 코인 (마일스톤) | 16개 | 1분 1 + 3분 3 + 보스 4 + 탈출 8 |
| 황금 코인 (엘리트/보스) | 6개 | E06 1 + B01 5 |
| **5분 총 골드** | **약 32개** | 시계 + 마일스톤 + 엘리트/보스 |

> 마일스톤 보너스는 본 기획 1차 안 이후 추가된 항목. `useGameStore.SURVIVAL_MILESTONES`.

## 7. XP 균형 보정

### 7-1. 문제

기존 시스템은 모든 적이 100% XP 드랍 → 5분간 누적 XP로 18–22 레벨업이 설계됨. 교과서 30% 드랍률을 그대로 적용하면 누적 XP가 약 1/3로 감소 → 5분 6–8 레벨업으로 추락 → Bang_Rules `7. XP & 레벨업` 정책 위반.

### 7-2. 해결 — 개별 XP 값 보정 (구현됨)

`Enemy.jsx`의 `ENEMY_STATS`. 30% 드랍률 보정 + Stage 1 재플랜 추가 상향 결합.

| 적 | 기획안 XP | **현재 구현 XP** | 비고 |
|---|---|---|---|
| E01 잡몹 | 3 | **6** | 기획 대비 2배 추가 상향 |
| E02 탱커 | 10 | **15** | 추가 상향 |
| E03 러너 | 3 | **5** | 추가 상향 |
| E04 원거리 | 7 | **10** | (1스테이지 미사용) |
| E05 돌진 | 10 | **15** | 추가 상향 |
| E06 거대 | 40 | **56** | 추가 상향 |
| B01 보스 | 0 | **0** | 보스 본체 처치 XP 없음 |

> 기획 안보다 추가 상향된 이유: 5분 풀플레이 QA에서 Lv 도달이 부족해 후속 튜닝 반영 (정확 시점은 `../Stage1_Balance/stage1_replan_2026-05-06.md` 및 후속 커밋 참조).
> B01은 본체 xp=0이지만 §3-3 보너스로 교과서 3개 × 40XP = 120XP 회수.

### 7-3. 검증 시뮬레이션 (현재값 기준)

- 일반 적 약 300 마리 처치 × 30% × 평균 XP ≈ **800–1200 XP** 회수.
- E06 1–2회 + B01 1회 보너스로 약 200–280 XP 추가.
- 합산 약 1000–1500 XP → 1스테이지 목표 Lv (Bang_Rules §7) 도달.
- 실측 미세조정은 `playtestLogger.js` 및 결과창 "📋 로그 복사" 데이터로 추적.
