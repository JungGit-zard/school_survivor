# Stage 1 세로형 교실 맵 크기 조절 계획

작성일: 2026-06-18

## 결론

Stage 1은 이미 코드상 가로보다 세로가 긴 맵이다. 현재 정본 수치는 `mapHalfX = 34`, `mapHalfZ = 54`이며, 실제 플레이 경계는 `68 x 108 units`이다.

이번 조절의 핵심은 맵을 무작정 더 길게 늘리는 것이 아니라, 현재 세로형 구조가 모바일 세로 화면에서 "긴 교실"로 읽히도록 문서, 오브젝트 배치, 시각 기준, 스폰 검증을 맞추는 것이다.

조건부 승인 방향:

- 1차 구현은 `mapHalfX = 34`, `mapHalfZ = 54`를 유지한다.
- 더 길게 조정하더라도 `mapHalfZ = 60`을 상한 후보로 둔다.
- `mapHalfX`는 적 스폰과 카메라 안전성을 위해 `32` 미만으로 줄이지 않는다.
- 중앙 생존 공간 `|x| < 12 && |z| < 12`는 계속 비운다.
- 책상/의자는 충돌 장애물, 쓰러진 학생은 비충돌 배경 오브젝트로 명시한다.

## 검수에 사용한 관점

관련 에이전트 관점을 동원해 다음 관점으로 나누어 검토했다.

- 문서 정합성: 기획 문서와 현재 코드 수치가 서로 맞는지 검토.
- 구현 가능성: 맵 경계, 카메라, 플레이어 제한, 적 스폰, 바닥 크기 영향 검토.
- 그래픽/모바일 UX: 세로 화면에서 교실이 길게 읽히는지 검토.
- 제품 관점: Stage 1 초반 학습, 4분 생존, Stage 2와의 차별성 검토.
- 적대적 검증: 세로 맵이 도망만 유리한 구조가 되는지, XP/골드 회수가 무너지는지 검토.
- 코드 매핑: 실제 변경 파일과 테스트 범위 확인.

## 정본 우선순위

기획 문서 사이에 오래된 5분/300초 기준과 현재 4분/240초 기준이 함께 남아 있다. 앞으로 Stage 1을 조정할 때는 아래 순서를 따른다.

1. `project_develop_policy.md`
2. 현재 코드의 스테이지 설정
3. `Planner/B. GAME_DESIGN/Stage_balance_summary.md`
4. 이 문서
5. 과거 Stage 1 세부 문서

과거 문서에 남은 `5분`, `300초`, `270초 이벤트` 수치는 역사 기록으로만 취급하고, 신규 구현 기준으로 직접 사용하지 않는다.

## 세로형 교실 레이아웃 기준

맵을 세로로 길게 느끼게 하려면 전체를 아래-중앙-위쪽 구역으로 읽히게 만든다.

| 구역 | 좌표 기준 | 목적 |
| --- | --- | --- |
| 아래 입구/혼란 시작 | `z = -45~-26` | 교실에 들어온 느낌, 무너진 책상/의자 일부 |
| 중앙 생존 구역 | `z = -18~18` | 초반 학습, 회피, XP 수집. 중앙 `|x| < 12 && |z| < 12` 비움 |
| 위쪽 교실 끝 | `z = 26~45` | 칠판/교탁/창가 방향 신호, 긴 교실의 끝점 |
| 좌우 외곽 | `|x| = 18~30` | 책상/의자/쓰러진 학생 배치, 중앙 통로 침범 금지 |

현재 오브젝트는 대부분 `x/z = ±18` 근처에 몰려 있어, 실제 `z = ±54` 맵의 위/아래가 비어 보일 위험이 있다. 구현 단계에서는 기존 시작 시야 주변 배치를 일부 유지하되, `z = ±26`, `z = ±38` 부근에 세로 리듬을 만드는 오브젝트를 나누어 배치한다.

## 오브젝트 배치 계획

- 책상/의자는 "무너진 교실 줄"로 보이게 배치한다. 완전 랜덤 배치는 피한다.
- 중앙 생존 구역에는 큰 충돌 오브젝트를 넣지 않는다.
- 책상/의자는 좌우 외곽과 상하 구역에 배치해 세로축을 만든다.
- 쓰러진 학생은 현재 10명 정본을 유지하되, 모바일 QA에서 적/XP/아이템과 섞이면 색 대비를 낮추거나 외곽으로 더 밀어낸다.
- 한 화면마다 빈 호흡 구간을 둔다. 세로맵이어도 매 순간 장애물이 꽉 차 보이면 교실이 아니라 막힌 방처럼 느껴진다.
- 장애물 점유율은 기존 그래픽 기준처럼 전체 맵의 15% 이하를 목표로 한다.

## 맵 크기 조절안

1차안: 현재 수치 유지

- `mapHalfX = 34`
- `mapHalfZ = 54`
- 이유: 이미 세로비 `108 / 68 = 1.59`이며, 카메라/스폰/바닥/플레이어 제한이 현재 안정 범위 안에 있다.

2차 후보: 세로감 약간 강화

- `mapHalfX = 34`
- `mapHalfZ = 60`
- 적용 조건: 모바일 QA에서 긴 교실이 충분히 읽히지 않고, 0~4분 성장/압박 지표가 유지될 때만 적용한다.

금지 후보:

- `mapHalfX < 32`: 벽 근처 스폰이 플레이어에게 너무 가까워질 위험이 커진다.
- `mapHalfZ > 60`: 긴 도망길이 생겨 적 압박과 XP 회수가 무너질 수 있다. 이 경우 스폰/드랍/카메라/바닥까지 별도 리밸런싱이 필요하다.

## 구현 순서

1. Stage 1 세로형 정본 테스트를 추가한다.
2. 오브젝트 배치 테스트에 "중앙 비움", "맵 경계 안 배치", "세로 구간별 최소 배치"를 추가한다.
3. 책상/의자/학생 배치를 `z = ±26`, `z = ±38` 구간까지 확장한다.
4. 시각 바닥에 Stage 1 경계감이나 교실 방향감을 주는 요소를 추가한다.
5. 적 스폰이 경계 근처에서 너무 가까워지지 않는지 검증한다.
6. 모바일 세로 화면에서 0~4분 플레이 QA를 진행한다.

## 통과 기준

- Stage 1이 모바일 9:16 화면에서 가로보다 세로로 긴 교실로 읽힌다.
- 시작 후 0~60초에 이동, 자동 공격, XP 수집, 레벨업 학습이 방해받지 않는다.
- 3:00~4:00 구간에서도 플레이어, 적, E05 돌진 경고, XP/골드/회복 아이템이 묻히지 않는다.
- 세로로 도망만 가는 플레이가 정답이 되지 않는다.
- 플레이어와 좀비가 책상/의자에 끼지 않는다.
- 맵 상하좌우 끝과 네 모서리에서 카메라가 빈 배경이나 읽기 어려운 스폰을 만들지 않는다.
