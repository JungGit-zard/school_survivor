# 몬스터 드랍 재화 이원화 기획안

> 작성일: 2026-05-08
> 근거: 사용자 요청 (2026-05-08), `Bang_Rules.md` 2026-05-06 Stage 1 Re-balance Addendum
> 목적: 인게임 단기 성장(레벨업) 재화와 계정 영구 성장(패시브) 재화를 분리한다.

---

## 1. 목표

스테이지 내 성장(임시)과 계정 영구 성장(반복 플레이 보상)을 시각·기능적으로 분리해서:

- **레벨업 동기**(스테이지 안)
- **반복 플레이 동기**(스테이지 클리어 후)

가 서로 침해하지 않도록 한다.

---

## 2. 두 재화 정의

| 재화 | 키 | 역할 | 보존 범위 | 그래픽 |
|---|---|---|---|---|
| 도라야키 | `xpDorayaki` | 스테이지 내 XP / 레벨업 | 스테이지 종료 시 소멸 | 갈색 단팥빵 디스크 (신규 제작) |
| 황금 코인 | `goldCoin` | 계정 영구 패시브 재화 | 영구 (계정 단위) | 기존 GoldCoin 그래픽 재사용 |

> 현재 구현은 `GoldCoin` 컴포넌트가 XP 드랍에 그대로 쓰이고 있다. 본 기획에서:
> - `GoldCoin` 컴포넌트는 그래픽을 그대로 두되 **새 황금코인 재화**로 의미만 옮긴다.
> - XP 드랍은 신규 `XpDorayaki` 컴포넌트로 교체한다 (기존 `XpOrb.jsx`는 미사용 상태였으므로 정리 또는 재활용 검토).

---

## 3. 드랍 규칙

### 3-1. 도라야키 (XP 재화) — 킬당 확률 드랍

- **드랍률: 30%** — 몬스터 처치 시 30% 확률로 1개 드랍.
- 보스 / 엘리트는 별도 정책 (아래 3-3).

### 3-2. 황금 코인 — 시간 기반 시계 드랍

핵심 의도: **"5분 동안 약 10코인"** — 킬 수 무관, 시간으로 보장.

- **목표 평균 간격: 30초 ± 5초** (균등 무작위 25–35s).
- **누적 목표**: 300초 / 30초 = **약 10개 / 5분**.
- 트리거: `elapsedMs` 기준 타이머가 만료되면 즉시 1개 출현, 다음 간격을 25–35s 무작위로 재설정.

#### 3-2-1. 출현 위치 정책

| 옵션 | 설명 | 장단점 |
|---|---|---|
| **A. 시계방식** (추천) | 타이머 만료 시점에 화면 안 무작위 적의 위치, 적이 없으면 플레이어 주변 3.0–6.0 units 링 | 시간 정확 / 킬 무관 |
| B. 킬 태그 방식 | 타이머 만료 후 다음 킬 시 드랍 (5초 안 미발생 시 강제 출현) | 자연스러움 / 시간 변동 큼 |

→ **사용자 의도("몇킬을 하든 시간기준")에 따라 옵션 A 채택.**

### 3-3. 보스 / 엘리트 보너스

| 적 | 도라야키 | 황금 코인 |
|---|---|---|
| E06 (거대 엘리트) | 100% (1개) | 시계 타이머 무관 100% (1개) — 큰 보상감 |
| B01 (보스) | 100% (3개) | 100% (5개) — 클리어 보너스 |

> 이 항목은 1차 권장. 사용자 확정 시 코드 적용.

## 4. 보존·소비 정책

| 재화 | 인게임 사용 | 스테이지 종료 처리 |
|---|---|---|
| 도라야키 | 즉시 흡수 → XP 누적 → 레벨업 | 결과창 표시 후 폐기 |
| 황금 코인 | 즉시 흡수 → 결과창에 합산 표시 | 계정 잔고에 영구 가산 (1차 서비스: localStorage 등 단순 저장) |

> 계정 잔고 영구 저장은 본 기획 범위. 패시브 능력 강화 UI는 차후 별도 기획.

## 5. 그래픽 사양

### 5-1. 도라야키 (신규)

탑다운 시점에서 보이는 형태로 단순화:

| 항목 | 값 |
|---|---|
| 외형 | 위에서 본 둥근 디스크 (도라야키 윗면) |
| 본체 색 | 단팥빵 갈색 `#A0531C` 톤 (단호박-팥 중간) |
| 외곽선 | three.js outline 0.96 (기존 toon 스타일과 일관) |
| 윗면 하이라이트 | 약간 밝은 갈색 `#C8742A` 작은 원 (구운 흔적 느낌) |
| 가장자리 띠 | 진한 갈색 `#5B2D10` 얇은 링 (빵 단면) |
| 직경 | 0.36 units (현재 GoldCoin과 비슷한 크기) |
| 높이 | 0.10 units (디스크형) |

3D 모델: `CylinderGeometry(r=0.18, r=0.18, h=0.10)` + 윗면용 작은 disc 메시.

### 5-2. 황금 코인 (재사용)

- 기존 `GoldCoin.jsx` 그래픽을 그대로 사용.
- **변경 사항**: 의미가 "XP 오브" → "영구 골드"로 바뀌므로, 흡수 시 store 액션이 `gainXp` 가 아닌 `gainGold` 로 분기되어야 한다.

## 6. 5분 플레이 평균 시나리오 (검증 시뮬레이션)

가정: 평균 적 처치율 약 50–80 마리/분.

| 항목 | 값 | 비고 |
|---|---|---|
| 5분간 처치 | 250–400 마리 | Bang_Rules 스폰 곡선 기준 |
| 도라야키 드랍 | 75–120개 (30%) | 개별 XP 값은 §7에서 보정 |
| 황금 코인 드랍 | 약 10개 (시계) | 25–35s × 10회 |

---

## 7. XP 균형 보정 (필수)

### 7-1. 문제

기존 시스템은 모든 적이 100% XP 드랍 → 5분간 누적 XP로 18–22 레벨업이 설계됨.

도라야키 30% 드랍률을 그대로 적용하면 누적 XP가 약 1/3로 감소 → 5분 6–8 레벨업으로 추락 → Bang_Rules `7. XP & 레벨업` 정책 위반.

### 7-2. 해결안 — 개별 XP 값 보정

도라야키 1개당 XP를 약 3.3배 (≈ 1/0.30) 올린다.

| 적 | 기존 XP | 신규 XP (도라야키 1개당) |
|---|---|---|
| E01 잡몹 | 1 | 3 |
| E02 탱커 | 3 | 10 |
| E03 러너 | 1 | 3 |
| E04 원거리 | 2 | 7 |
| E05 돌진 | 3 | 10 |
| E06 거대 | 12 | 40 |
| B01 보스 | 0 | 0 |

> 추가로 보스/엘리트는 §3-3에 따라 도라야키 여러 개 드랍 → 큰 보상감.

### 7-3. 검증 시뮬레이션

300 마리 처치 × 30% × 평균 4 XP ≈ 360 XP → Lv.20 누적 요구치 (`6 → 9 → 14 → 19 → 26 → 35 ...` 누적 ≈ 720 XP)의 절반.

→ 보너스 드랍(엘리트/보스 §3-3)으로 추가 100–200 XP 보충 시 5분 Lv.18–22 도달 가능.

> 1회 풀플레이 QA 후 미세조정 필요.

## 9. 검증 체크리스트 (Karpathy #4 / Goal-Driven)

- [ ] 도라야키와 황금코인이 시각적으로 즉시 구분된다 (스크린샷 비교).
- [ ] 5분 플레이에서 황금코인 누적 9–12개 (10 ± 1 허용).
- [ ] 5분 플레이에서 레벨업 18–22회 (Bang_Rules 7-기준 유지).
- [ ] 도라야키 드랍 빈도가 체감상 "킬당 약 1/3" 패턴으로 보인다.
- [ ] 게임 재시작 시 인게임 골드 카운터 초기화, 영구 누적치는 보존.
- [ ] 보스 처치 시 다량 황금코인 + 다량 도라야키 (큰 보상감 체감).

## 10. 구현 순서 (Karpathy #2 / Simplicity First)

1. 그래픽 신규: `XpDorayaki.jsx` 모델 (단순 cylinder + outline) — 비주얼 1차 OK까지.
2. Enemies.jsx 드랍 분기:
   - 사망 시 30% 도라야키
   - 시계 타이머 25–35s 황금코인 (적 위치 / 폴백 플레이어 링)
3. Store: `gainGold`, `goldBalance` 추가, GoldCoin 흡수 액션 분기.
4. ENEMY_STATS XP 값 §7-2 보정.
5. HUD 골드 카운터 + 결과창 표시.
6. localStorage 영구 누적 (단순 1줄).
7. Bang_Rules 부록 추가.
8. 5분 풀플레이 QA → 미세조정.

각 단계 끝마다 빌드 검증 (`npm run build`) + 브라우저 즉시 확인.
