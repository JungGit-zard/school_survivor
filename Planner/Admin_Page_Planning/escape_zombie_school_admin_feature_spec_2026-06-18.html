Escape! zombie school
게임 어드민 기능기획서

현재 프로젝트는 React + Three.js/R3F 기반 모바일 세로형 생존 액션 게임이다. Stage 1은 240초 교실 생존, Stage 2는 복도 투사체 시험으로 구성되어 있고, 무기/패시브/스테이지/웨이브/보상/플레이어 기록이 로컬 저장 및 코드 카탈로그 중심으로 관리된다. 본 어드민은 운영자가 이 데이터를 코드 수정 없이 점검·튜닝·배포 승인할 수 있게 하는 내부 관리 페이지를 목표로 한다.

Game Read

게임 파악 요약

  • 타이틀 → 스테이지 선택 → 게임 → HUD/레벨업/게임오버 → 코인상점 루프.
  • Stage 1: 감염된 교실에서 240초 생존, 192초 보스 조우.
  • Stage 2: Stage 1 클리어 또는 180초 생존 3회 후 해금되는 복도 탄환 생존.
  • 무기 14종과 코인상점 패시브 성장 구조.
Admin Goal

어드민 목표

  • 게임 밸런스 변경사항을 한 화면에서 확인하고 승인.
  • 무기/적/웨이브/보상 수치를 테이블 기반으로 관리.
  • 운영 공지, 이벤트, 코인상점 상태를 빠르게 점검.
  • QA 스크린샷/브라우저 플레이테스트 결과를 연결.
MVP Scope

1차 범위

  • 읽기 중심 대시보드 + 안전한 임시 편집.

  • 실제 게임 반영 전 리뷰/승인 플로우 필수.
  • 모바일 세로 게임 특성상 390×844 프리뷰 우선.

주요 기능 상세

1. 운영 대시보드

  • 현재 빌드 상태, 최근 테스트 통과 여부, Stage별 주요 지표 표시.
  • Stage 1/2 카드: 지속시간, 해금 조건, 보스 시간, 맵 크기.
  • 최근 변경안: 누가 어떤 수치를 바꿨는지 diff 형태로 표시.
  • 긴급 버튼: 테스트 데이터 리셋, 빌드 확인, QA 리포트 열기.

2. 스테이지/웨이브 관리

  • Stage config 편집: durationSec, bossWarningSec, mapHalfX/Z, unlock condition.
  • Wave timeline 시각화: 0–240초 구간을 막대그래프로 표시.
  • Enemy weights 편집: E01/E02/E03/E04/E05/E06/B01 비율 합계 검증.
  • Burst event: 특정 초에 등장하는 적 타입/수량 관리.

3. 무기/업그레이드 관리

  • 무기별 base stat: damage, cooldown, range, radius, count, speed 등.
  • Starter/Unlock 조건 구분, OR 조건 시각화.
  • 레벨업 카드 출현 minLevel 및 설명문 관리.
  • 상향/하향 변경 시 예상 영향 태그: 초반 안정성, 보스전, 광역 처리, 단일딜.

4. 경제/상점 관리

  • 생존 마일스톤 코인 보상 48/144/192/240초 관리.
  • 코인상점 패시브 레벨, 가격, 효과, maxLevel 테이블.
  • 테스트 계정 goldTotal 부여/차감/초기화.
  • 구매 가능/코인 부족/최대 레벨 상태 미리보기.

5. 플레이어 기록/해금 관리

  • totalRuns, totalKills, totalGold, totalSurvivalSeconds, bestSurvivalSeconds.
  • 무기 해금 상태 확인 및 테스트용 일괄 해금.
  • Stage 2 해금 조건 상태: Stage 1 클리어 또는 180초 생존 3회.
  • Google 계정 연동 이후 유저 검색/세그먼트 기능 확장.

6. QA Lab

  • 타이틀/코인상점/게임시작/게임오버 스크린샷 목록화.
  • 브라우저 플레이테스트 결과와 이슈 severity 표시.
  • 모바일 390×844 기준 회귀 체크리스트.
  • 승인 전 필수 체크: 빌드, 테스트, 플레이 1회, 상점 이동.

변경 승인 플로우

1 Draft기획자/운영자가 수치를 수정하지만 라이브에는 반영하지 않는다.
2 Validate합계, 범위, 음수, 과도한 cooldown 등 자동 검증.
3 Preview390×844 게임 프리뷰와 diff 비교.
4 QA브라우저 플레이테스트 및 스크린샷 확인.
5 ApproveOwner 승인 후 적용/배포 요청.
Data Safety

안전장치

  • 라이브 반영 전 모든 변경은 draft와 diff로 보관.
  • Stage duration, boss timing, unlock condition 변경은 위험 변경으로 표시.
  • 삭제 대신 비활성화 우선. 웨이브/무기 항목 삭제는 2단계 확인.
  • 수치 범위 guard: cooldown 0 이하 금지, weight 합계 1.0 자동 보정/경고, mapHalf 최소값 제한.
  • 운영 어드민은 게임 클라이언트 번들에 포함하지 않고 별도 내부 페이지로 분리.
1차 구현은 실제 데이터 저장보다 “게임 상태를 정확히 읽고, 안전하게 조정안을 만드는 도구”에 집중하는 것이 좋다. 바로 라이브 수정 기능부터 붙이면 밸런스 사고 위험이 커진다.
