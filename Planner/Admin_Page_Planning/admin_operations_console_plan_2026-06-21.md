# Escape! zombie school Admin Operations Console Plan

> 작성일: 2026-06-21  
> 상태: 구현 기준 기획서  
> 범위: 별도 운영 페이지, 게임 밸런스 조작, 랭킹 보상/시즌 진행 조작

## 1. 목표

운영자가 코드 파일을 직접 열지 않아도 게임 운영에 필요한 핵심 값을 확인하고 조정할 수 있는 별도 어드민 페이지를 만든다. 1차 구현은 로컬 운영 설정을 저장하고, 게임 클라이언트가 그 설정을 읽어 실제 플레이와 랭킹 화면에 반영되는 구조로 한다.

## 2. 큰 흐름

### A. 게임 플레이 밸런스 탭

- Stage 1, Stage 2 생존 시간 조정.
- 플레이어 기본 최대 HP 보너스 조정.
- 플레이어 이동 속도 배율 조정.
- 생존 마일스톤 골드 보상 배율 조정.
- 변경 전/후 효과를 미리보기 카드로 표시.

### B. 랭킹 보상과 시즌 진행 탭

- 시즌 ID, 시즌명, 상태(draft/live/closed) 조정.
- 시즌 시작일, 종료일 조정.
- Stage 2 보너스 점수와 클리어 보너스 점수 조정.
- 1위, TOP 10, TOP 100 보상 골드와 배지명 조정.
- 랭킹 페이지에서 현재 시즌명과 보상 요약을 노출.

## 3. 안전장치

- 저장은 `localStorage`의 버전 있는 설정 객체로 시작한다.
- 모든 숫자 입력은 최소/최대 범위로 보정한다.
- 기본값 복원 버튼을 제공한다.
- 저장 전 운영자가 변경 효과를 읽을 수 있도록 미리보기를 제공한다.
- 현재 단계에서는 서버 권한/공식 랭킹 검증을 구현하지 않는다. 공식 경쟁 랭킹은 서버 검증 후 별도 확장한다.

## 4. 게임 연동 기준

- `stageConfig`가 어드민 설정을 읽어 스테이지 지속 시간과 마일스톤 골드 보상에 반영한다.
- `useGameStore`가 어드민 설정을 읽어 새 게임 시작 시 플레이어 기본 HP와 이동 속도에 반영한다.
- `rankingScorePolicy`가 어드민 설정을 읽어 Stage 2 보너스와 클리어 보너스에 반영한다.
- `UserRanking`이 어드민 설정을 읽어 시즌명과 보상 요약을 표시한다.

## 5. 1차 구현 화면

- 경로: `/admin`
- 구성:
  - 상단: 운영 콘솔 제목, 게임으로 돌아가기, 저장 상태.
  - 좌측 또는 상단 탭: `게임 밸런스`, `랭킹/시즌`.
  - 중앙: 탭별 입력 폼.
  - 우측: 적용 미리보기, 저장/기본값 복원.

## 6. 제외 범위

- 관리자 로그인/권한 분리.
- Firebase 서버 저장.
- 실제 공식 시즌 정산 지급.
- 유저별 보상 수령 처리.
- 실시간 운영 로그 서버 전송.

이 제외 범위는 기능을 줄이기 위한 것이 아니라, 현재 클라이언트 중심 프로토타입에서 밸런스 사고와 보안 사고를 막기 위한 1차 경계다.
