# Stage 2 복도형 원거리 투사체 스테이지 기획

- 작성일: 2026-06-03
- 작성 방식: Superpowers brainstorming + Agent Room 서브에이전트 논의
- 논의 참여: code-mapper, game-developer, product-manager, reviewer
- 문서 위치: Planner
- 상태: 1차 기획안. 구현 전 수치 검증과 사용자 승인 필요.

## 1. 사용자 확정 사항

이번 Stage 2는 사용자가 선택한 방향을 기준으로 한다.

| 항목 | 확정값 |
|---|---|
| Stage 2 방향 | A안: 복도형 원거리 투사체 스테이지 |
| 목표 난이도 | Stage 1을 몇 번 플레이해 무기와 패시브를 조금 해금한 뒤 도전 |
| 클리어 조건 | Stage 1처럼 300초 생존 |

이 문서에서 "투사체"는 적이 플레이어에게 날리는 탄환을 뜻한다. "복도형"은 단순 배경 테마가 아니라, 플레이어가 좌우 또는 앞뒤 레인을 읽으며 피해야 하는 스테이지 구조를 뜻한다.

## 2. 현재 Stage 1 구현 점검

Stage 2 기획은 오래된 문서가 아니라 현재 구현된 Stage 1 코드를 기준으로 세운다.

### 2.1 클리어와 시간 구조

- 현재 스테이지 길이는 `STAGE_DURATION_SEC = 300`이다.
- 게임은 300초가 지나면 생존 클리어로 처리된다.
- 보스를 반드시 처치해야 클리어되는 구조가 아니다.
- 생존 보너스는 60초, 180초, 240초, 300초 구간으로 지급된다.
- 현재 클리어 기록은 `stage1Clears` 중심으로 저장된다.

Stage 2도 300초 생존 클리어를 유지하되, 기록은 Stage 1과 분리하는 것이 안전하다. 구현 시에는 `stage2Clears` 같은 별도 값을 두는 방안을 우선 검토한다.

### 2.2 현재 Stage 1 적 구성

현재 Stage 1에서 실제로 쓰이는 적은 다음 계열이다.

| 적 | 현재 역할 |
|---|---|
| E01 | 기본 추격 좀비 |
| E02 | 체력이 높은 압박형 적 |
| E03 | 수가 늘어나며 압박하는 빠른 적 |
| E05 | 돌진형 적 |
| E06 | 후반 엘리트 벽 역할 |
| B01 | 240초 이후 등장하는 보스, 현재는 접근/돌진 중심 |

E04는 코드에 이미 원거리 적으로 존재하지만, Stage 1 스폰표에서는 제외되어 있다. 따라서 Stage 2에서 E04를 다시 쓰더라도 반드시 "Stage 2 전용"으로 분리해야 한다.

### 2.3 E04 현재 코드 기준값

E04의 현재 구현 기준값은 아래와 같다.

| 항목 | 현재 코드 기준 |
|---|---|
| 체력 | 32 |
| 이동 속도 | 0.45 |
| 경험치 | 10 |
| 원거리 여부 | true |
| 발사 쿨다운 | 2200ms |
| 탄환 피해 | 8 |
| 탄속 | 1.9 |
| 선호 거리 | 5.5 |
| 최소 거리 | 3.5 |

과거 문서에는 E04 경험치가 2 또는 7로 적힌 곳이 있으나, 현재 코드는 10이다. Stage 2 1차 기획에서는 현재 코드값을 기준으로 삼되, 모바일 플레이 검증 후 탄속, 피해, 쿨다운은 조정 가능 수치로 둔다.

### 2.4 성장과 해금 상태

현재 무기 카탈로그는 12종이다. 한 런에서 보유 가능한 무기 수는 8개이며, 무기 최대 레벨은 5다.

중요 해금 조건은 다음과 같다.

| 항목 | 현재 해금 기준 |
|---|---|
| guidedMissile | 총 5런 |
| starlink | 총 10런 또는 총 5000킬 |
| compassBlade | 한 런 80킬 또는 총 200킬 |
| umbrellaGuard | 한 런 90초 생존 또는 총 300초 생존 |
| eraserBomb | 한 런 골드 80 또는 총 골드 200 |

Stage 2는 guidedMissile이나 starlink가 반드시 있어야 하는 난이도로 만들지 않는다. 기준 플레이어는 Stage 1을 3~5회 플레이했고, 일부 패시브 1레벨과 몇 개의 해금 무기를 접한 상태로 본다.

### 2.5 문서와 코드의 차이

Stage 2 설계에서 주의해야 할 차이는 다음과 같다.

| 항목 | 구문서 | 현재 코드/상태 | Stage 2 적용 |
|---|---|---|---|
| E04 경험치 | 2 또는 7 | 10 | 현재 코드값 기준, 조정 가능 |
| Stage 1 초반 버스트 | 0초 x8 / 30초 x6 | 0초 x16 / 30초 x12 | Stage 2는 새 스폰표로 분리 |
| XP 처리 | 1회 1레벨업처럼 설명 | `while`로 연속 레벨업 가능 | 현재 코드 기준 |
| 구현 무기 수 | 7종 설명 문서 존재 | 12종 카탈로그 | 현재 코드 기준 |
| 클리어 기록 | Stage 1 중심 | `stage1Clears` | Stage 2 별도 기록 필요 |

## 3. Stage 2 핵심 의도

Stage 2의 목표는 "Stage 1보다 더 많은 적"이 아니다. Stage 1에서 익힌 이동, 성장, 무기 선택에 더해 원거리 탄환을 읽고 위치를 바꾸는 법을 가르치는 두 번째 생존 시험이다.

플레이어가 배워야 할 것은 세 가지다.

1. 복도에서 한 방향으로만 도망치면 위험해진다.
2. E04 탄환은 보고 피할 수 있어야 한다.
3. 돌진형 적과 원거리 적이 동시에 있을 때도 안전 레인을 찾아야 한다.

Stage 2는 공정해야 한다. 공정하다는 뜻은 어렵지 않다는 뜻이 아니라, 맞기 전에 위험을 알아볼 수 있고, 회피할 공간이 보이며, 실패했을 때 이유를 이해할 수 있다는 뜻이다.

## 4. 입장 조건과 권장 성장 상태

### 4.1 입장 조건 제안

Stage 2 입장 조건은 둘 중 하나를 만족하면 열린다.

| 조건 | 이유 |
|---|---|
| Stage 1 1회 클리어 | 가장 명확한 진행 기준 |
| Stage 1 180초 이상 생존 3회 | 클리어 전 플레이어도 다음 목표를 볼 수 있게 함 |

이 조건은 "몇 번 플레이한 뒤 도전"이라는 사용자 선택과 맞다. 단, 최종 구현 전에는 UI에서 조건을 어떻게 보여줄지 별도 확인이 필요하다.

### 4.2 권장 성장 상태

Stage 2 1차 밸런스의 기준 플레이어는 다음 상태로 본다.

| 항목 | 기준 |
|---|---|
| Stage 1 플레이 횟수 | 3~5회 |
| 패시브 | 이동 속도, 최대 체력, 자석, 공격력 계열 중 일부 1레벨 |
| 무기 경험 | 기본 무기 여러 개와 scienceFlask 또는 bell 경험 |
| 고급 해금 | guidedMissile, starlink는 있으면 편하지만 필수 아님 |

무해금 계정도 플레이는 가능해야 하지만, 첫 클리어 기준은 "조금 성장한 계정"으로 잡는다.

## 5. Stage 2 공간 규칙

Stage 2는 실제 플레이 규칙에서 복도형을 드러내야 한다.

| 항목 | 1차 규칙 |
|---|---|
| 기본 구조 | 긴 학교 복도 형태 |
| 플레이 감각 | 좌우 레인 이동과 전후 이동을 모두 사용 |
| 스폰 방식 | Stage 1의 단순 원형 스폰을 그대로 쓰지 않음 |
| 안전 공간 | 항상 최소 1개 이상의 회피 레인이 보여야 함 |
| 측후면 스폰 | 허용하되, 플레이어 바로 근처 즉시 피격 스폰 금지 |
| 벽/경계 | 플레이어가 끼이거나 탄환이 보이지 않는 위치에서 맞지 않게 처리 |

구현 전 결정이 필요한 부분은 "복도형을 실제 맵 충돌까지 바꿀지"이다. 최소 구현안은 배경과 스폰 레인으로 복도감을 먼저 만들고, 확장 구현안은 벽 충돌과 카메라 규칙까지 포함한다.

## 6. 300초 웨이브 기획

Stage 2는 300초 생존을 유지한다. 시간표는 Stage 1과 별도로 둔다.

| 시간 | 적 구성 | E04 상한 | 주요 사건 | 의도 |
|---|---:|---:|---|---|
| 0:00~0:30 | E01 | 0 | 복도 이동 적응 | 첫 30초는 탄환 없이 방향 감각 학습 |
| 0:30~1:00 | E01 + E03 | 0 | 빠른 압박 추가 | 직선 도주만으로는 불리하다는 점 학습 |
| 1:00~1:30 | E01 + E02 + E03 | 0 | E02 첫 투입 | 좁은 길에서 체력 높은 적이 벽이 됨 |
| 1:30~2:00 | E01 + E03 + E04 | 1~2 | E04 첫 등장 | 단일 직선 탄환을 보고 피하는 튜토리얼 |
| 2:00~2:30 | E02 + E04 | 2~3 | 탱커와 사수 조합 | 막는 적 뒤에서 탄환이 오는 구조 학습 |
| 2:30~3:00 | E01 + E03 + E05 | 0~1 | E05 첫 강조 | 돌진과 탄환을 동시에 과하게 섞지 않음 |
| 3:00~3:30 | E03 + E04 + E05 | 2~3 | 레인 변경 요구 | 한 레인에 오래 머물면 위험 |
| 3:30~4:00 | E02 + E04 + E06 | 3~4 | E06 엘리트 벽 | 느린 큰 압박과 탄환 회피 결합 |
| 4:00~4:30 | B01 + 소량 일반 적 + 제한 E04 | 2~3 | B01 등장 | 보스 인지 우선, 탄환은 보조 압박 |
| 4:30~5:00 | B01 + E04 + E05 + E03 | 3~4 | 최종 생존 압박 | 돌진, 탄환, 무리 압박을 종합 테스트 |

초반 90초 안에는 E04를 넣지 않는다. 원거리 탄환은 90초 이후부터 서서히 도입한다.

## 7. E04 투사체 규칙

E04는 Stage 2의 대표 적이지만, 억울한 피격을 만들면 안 된다.

| 규칙 | 1차 기준 |
|---|---|
| 첫 등장 | 90초 이후 |
| 첫 발사 지연 | 등장 후 0.8~1.0초 이상 |
| 발사 형태 | 직선 탄환만 사용 |
| 유도/반사/예측 사격 | 1차 구현에서는 금지 |
| 초반 탄환 라인 | 1라인 |
| 중반 탄환 라인 | 최대 2라인 |
| 후반 탄환 라인 | 최대 3라인 |
| 화면 내 적 탄환 총량 | 1차 상한 6발 |
| 오프스크린 즉시 발사 | 금지 |
| 플레이어 근접 즉시 발사 | 금지 |
| E05/B01 경고 중 발사 | 줄이거나 잠시 억제 |

탄환 색, 예고선, 피격 판정 크기는 모바일에서 가장 먼저 검증해야 한다. 손가락에 가려지는 영역에서 탄환이 갑자기 나오면 실패 원인을 알기 어렵다.

## 8. 엘리트와 보스 규칙

### 8.1 E06

E06은 Stage 2에서도 "복도 벽" 역할로 쓴다. 빠르게 죽이는 적이 아니라, 플레이어의 이동 경로를 제한해 E04 탄환 회피를 어렵게 만드는 적이다.

단, E06과 E04를 함께 쓸 때는 안전 레인을 남겨야 한다.

### 8.2 B01

B01은 240초 전후 등장시키는 방향을 우선 기획한다. 이는 현재 Stage 1의 240초 보스 경고 구조와도 맞는다.

1차 Stage 2에서 B01은 자체 탄환을 다시 넣지 않는다. B01은 접근/돌진 압박을 담당하고, 원거리 압박은 E04가 담당한다. 보스 탄환까지 동시에 넣으면 Stage 2의 학습 목표가 흐려지고 모바일 회피가 불공정해질 가능성이 크다.

| 항목 | 기준 |
|---|---|
| B01 등장 | 240초 전후 |
| B01 역할 | 큰 추격/돌진 압박 |
| B01 자체 탄환 | 1차 구현 금지 |
| B01 구간 E04 | 수를 제한하고 발사 타이밍을 조절 |
| 클리어 조건 | B01 처치가 아니라 300초 생존 |

## 9. 보상과 메타 진행

Stage 2는 Stage 1보다 조금 높은 보상을 줘야 한다. 그래야 플레이어가 위험한 스테이지에 도전할 이유가 생긴다.

| 보상 항목 | 1차 기획 |
|---|---|
| 기본 클리어 보상 | Stage 1보다 약 15~20% 높은 총 골드 기대값 |
| 300초 생존 보상 | Stage 2 전용 클리어 보상 지급 |
| 첫 클리어 보상 | 신규 패시브 또는 상점 해금 후보 |
| 반복 플레이 보상 | Stage 1보다 높은 골드 효율, 단 과도한 파밍처는 금지 |
| 기록 | Stage 2 최고 생존 시간과 클리어 횟수 분리 기록 |

첫 클리어 해금 후보는 `armor` 계열 패시브가 적합하다. 이유는 Stage 2가 탄환과 돌진을 배우는 스테이지이기 때문이다. 다만 현재 패시브 카탈로그에 바로 구현되어 있는지 확인한 뒤 확정해야 한다.

두 번째 후보는 `cooldown` 계열 패시브다. 이 보상은 강력하므로 Stage 2 첫 클리어가 아니라 Stage 2 보스 처치, 누적 클리어, 또는 이후 Stage 3 준비 보상으로 미룰 수 있다.

## 10. UI와 안내

Stage 2에서 플레이어에게 필요한 정보는 간단해야 한다.

| 위치 | 안내 내용 |
|---|---|
| 스테이지 선택 | "복도에서 탄환을 피하며 300초 생존" |
| 입장 조건 미달 | Stage 1 클리어 또는 180초 생존 3회 필요 |
| 첫 E04 등장 | 짧은 경고 문구 또는 시각 신호 |
| 결과창 | 사망 원인 힌트: 이동 속도, 최대 체력, 광역 무기 추천 |

화면 안의 설명문은 길게 쓰지 않는다. 실제 게임 화면에서는 문구보다 탄환 예고선, 적 실루엣, 안전 레인이 더 중요하다.

## 11. Stage 1 회귀 금지 규칙

Stage 2 구현 시 반드시 지킬 회귀 금지 규칙이다. "회귀"는 이미 고친 기능이나 기존 동작이 다시 나빠지는 일을 뜻한다.

1. Stage 1에는 E04가 스폰되지 않는다.
2. Stage 1 B01에는 1차 Stage 2 작업 때문에 탄환 패턴이 다시 붙지 않는다.
3. Stage 1 300초 생존 클리어 구조는 유지한다.
4. Stage 1 생존 보너스와 기존 무기 해금 조건은 임의로 바꾸지 않는다.
5. Stage 2 기록 추가가 `stage1Clears` 값을 오염시키지 않는다.

| 투사체 공정성 | 오프스크린 즉시 발사, 근접 즉시 발사, 회피 불가 교차탄이 없는지 |
| 모바일 가독성 | 조이스틱, HUD, 탄환 예고, 적 실루엣이 서로 가리지 않는지 |
| 난이도 | 무해금, 조금 성장, guidedMissile/starlink 보유 계정으로 각각 플레이 감각 기록 |
| 클리어 | 300초 생존 시 결과창과 보상이 정상인지 |
| 기록 | Stage 2 기록이 Stage 1 기록과 분리되는지 |
| 보스 | 240초 경고와 B01 등장/보상이 의도와 맞는지 |
| 성능 | 탄환 증가 시 모바일 입력 지연이나 프레임 저하가 심하지 않은지 |
| 자동 테스트 후보 | Stage 1 회귀, Stage 2 스폰표, Stage 2 클리어 기록, E04 탄환 상한 |

## 13. 구현 전 사용자 승인 필요 항목

아래 항목은 구현 전에 다시 확정해야 한다.

1. 복도형을 실제 벽/충돌/카메라까지 바꾸는지, 아니면 1차는 스폰 레인과 배경 연출로 시작하는지.
2. Stage 2 B01 등장 시점을 240초로 확정할지.
3. B01 자체 탄환을 1차 구현에서 완전히 금지할지.
4. Stage 2 기록 키를 `stage2Clears`, `stage2BestSurvivalSec`처럼 분리할지.
5. Stage 2 첫 클리어 보상을 `armor` 패시브 후보로 둘지, 골드 보상만 먼저 둘지.
6. E04 화면 내 탄환 상한 6발과 첫 발사 지연 0.8~1.0초를 1차 기준으로 승인할지.

## 14. 1차 결론

Stage 2는 Stage 1의 후속 난이도이지만, 단순히 적 수를 늘리는 스테이지가 아니다. 핵심은 복도라는 제한된 공간에서 E04 원거리 탄환을 읽고, E05/B01의 돌진 압박과 함께 안전 레인을 찾는 것이다.

가장 안전한 1차 구현 방향은 다음과 같다.

1. Stage 1은 그대로 보존한다.
2. Stage 2는 300초 생존 클리어로 분리한다.
3. E04는 Stage 2 전용으로 90초 이후부터 천천히 도입한다.
4. B01은 240초 전후 등장하되 자체 탄환은 넣지 않는다.
5. 복도형은 최소한 스폰 레인과 이동 압박 규칙으로 실제 플레이에 반영한다.
6. 모바일에서 탄환 가독성과 회피 가능성을 검증한 뒤 수치를 확정한다.
