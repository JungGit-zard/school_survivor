## 6. VFXLayer 성능 최적화 — setState 디바운스

### 발견
[`VFXLayer.jsx`](../../Developer/r3f_prototype/src/components/VFXLayer.jsx) emit 콜백과 onDone 콜백이 매 호출마다 직접 `setEvents`. 같은 마이크로태스크 내 N개 emit + M개 done이 발생하면 N+M회 React reconciliation.

### 적용
`queueMicrotask` 기반 디바운스 — 같은 마이크로태스크 내 다중 호출을 setState **1회**로 합침.

```js
const flushPending = useCallback(() => {
  if (pendingRef.current) return
  pendingRef.current = true
  queueMicrotask(() => {
    pendingRef.current = false
    setEvents([...eventsRef.current])
  })
}, [])
```

emit 콜백 / onDone 둘 다 `setEvents` 직접 호출 제거 → `flushPending()`만 호출.

### 검증
- `npm run build` ✓ 594ms
- `npm run test` ✓ 19/19

### 실측은 TODO
실제 프레임 드랍 측정은 DevTools Performance 캡처 필요. 이 변경은 이론적 이득(reconciliation 횟수 감소), 위험 거의 없는 안전한 개선.
