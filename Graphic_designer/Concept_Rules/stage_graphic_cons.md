# Escape! zombie school Stage Graphic Concept

## 1. Graphic Goal

이 문서는 `Planner/escape_zombie_school_main_contents_plan.md`의 5분 스테이지 기획을 기준으로, "좀비에 감염된 학교를 빠져나가는 주인공의 모험"을 실제 게임 화면으로 구현하기 위한 그래픽 컨셉안이다.

첫 버전의 목표는 화려한 장식보다 즉시 플레이 가능한 화면 명확성이다.

- 장르: 모바일 세로형 서바이버라이크 핵앤슬래시
- 무대: 감염 사태가 발생한 학교 부지
- 감정: 익숙한 학교 공간이 위험한 탈출 장소로 변한 느낌
- 화면 우선순위: 플레이어 위치, 적 접근 방향, 공격 범위, 위험 장판, 드랍 아이템
- 구현 엔진: Phaser 4
- 캐릭터 렌더링: Three.js 기반 3D 캐릭터
- 캐릭터 스타일: Three.js 카툰 렌더링 완전 도입

## 2. Final Screen Concept

게임 화면은 2D 탑다운 서바이버라이크 구조를 기본으로 하되, 플레이어와 몬스터는 Three.js 3D 카툰 캐릭터를 Phaser 화면 위에 합성한다.

카메라는 플레이어를 따라가는 세로형 화면이다. 배경은 Phaser 타일맵으로 구성하고, 캐릭터만 Three.js 씬에서 렌더링한다. 개발자는 Phaser를 게임 로직, 충돌, UI, 타일맵, 이펙트의 중심으로 두고 Three.js는 캐릭터 시각 표현 전용으로 사용한다.

권장 화면 기준:

- 기본 기준: 9:16 세로 화면
- 기준 논리 해상도: 720 x 1280
- 대표 고해상도 검수 기준: 1080 x 1920
- 실제 렌더링: 기기 해상도와 `devicePixelRatio`에 맞춰 반응형 스케일링
- UI 안전 영역: 상단 노치, 하단 제스처 바, 기기별 긴 화면 비율을 고려
- 게임 월드: 화면 가로 2.5배, 세로 3.5배 수준의 가짜 무한 학교 부지
- 카메라: 플레이어 중심 추적
- 캐릭터 시점: 살짝 위에서 내려보는 3/4 탑다운

## 3. Visual Identity

그래픽 스타일은 "어두운 공포"가 아니라 "카툰풍 감염 학교 액션"이다. 너무 사실적인 공포 표현은 피하고, 초보 플레이어가 위험 요소를 즉시 읽을 수 있게 색과 외곽선을 분명히 한다.

핵심 키워드:

- 감염된 학교
- 카툰 렌더링
- 굵은 외곽선
- 높은 가독성
- 노란 학교 조명과 초록 감염 오염
- 붉은 위험 신호
- 모바일 화면에서도 구분되는 큰 실루엣

색상 방향:

- 바닥 기본색: 어두운 회청색, 탁한 콘크리트색
- 학교 포인트색: 노란 안전선, 흰 운동장 라인, 파란 교실 표지판
- 감염 포인트색: 형광 녹색, 탁한 보라색
- 위험 포인트색: 붉은 경고색, 주황색 예고선
- 플레이어 포인트색: 밝은 하늘색 또는 노란색
- XP 오브: 청록색
- 회복 오브: 선명한 분홍색 또는 녹색

## 4. Stage Concept

1스테이지는 "감염된 학교 운동장과 본관 주변"이다.

플레이어는 학교 정문 또는 운동장 중앙 근처에서 시작한다. 맵은 무한히 이어지는 것처럼 보이지만 실제 구현은 반복 타일 재배치 방식이다. 중앙부는 회피 공간을 확보하고, 장애물은 가장자리 또는 외곽 장식 중심으로 배치한다.

필수 배경 요소:

- 운동장 바닥
- 흰색 트랙 라인
- 학교 본관 실루엣 일부
- 쓰러진 책상과 의자
- 찢어진 시험지
- 오염된 사물함
- 안전 표지판
- 감염 경고 테이프
- 깨진 창문 그림자
- 오염 웅덩이

장애물 배치 규칙:

- 완전 막힘 구역은 만들지 않는다.
- 이동 방해는 전체 맵의 15퍼센트 이하로 제한한다.
- 중앙부는 항상 넓게 비워둔다.
- 원거리 적 탄과 장애물이 겹쳐 억울한 피격이 발생하지 않게 한다.

## 5. Phaser Layer Structure

Phaser는 게임의 주 화면과 2D 요소를 담당한다.

권장 레이어 순서:

1. `BackgroundTileLayer`
   - 운동장 바닥, 복도 바닥, 반복 타일
2. `GroundDetailLayer`
   - 종이, 균열, 혈흔이 아닌 오염 자국, 작은 쓰레기
3. `ObstacleLayer`
   - 책상, 사물함, 표지판, 콘, 바리케이드
4. `ShadowLayer`
   - 캐릭터 그림자, 보스 그림자, 큰 장애물 그림자
5. `ThreeCharacterCanvas`
   - Three.js 플레이어와 몬스터 렌더링 결과
6. `ProjectileLayer`
   - 연필 투척, 비상벨 충격파, 오염 탄
7. `VFXLayer`
   - 타격, 레벨업, 돌진 예고, 장판 경고, 사망 소멸
8. `DropLayer`
   - XP 오브, 회복 오브, 자석 아이템
9. `UILayer`
   - 체력, 시간, 레벨, 무기 선택, 보스 등장 경고

주의:

- Three.js 캐릭터는 실제 충돌 판정의 기준이 아니다.
- 충돌 판정은 Phaser의 원형 또는 캡슐형 히트박스로 처리한다.
- Three.js 모델 위치는 Phaser 월드 좌표를 따라 시각적으로만 동기화한다.

## 6. Three.js Toon Rendering Direction

플레이어와 몬스터는 Three.js에서 카툰 렌더링을 완전히 적용한다.

필수 구성:

- `MeshToonMaterial` 기반 재질 사용
- 모델별 굵은 외곽선 적용
- 그림자는 부드러운 실사 그림자보다 단순한 둥근 바닥 그림자 우선
- 광원은 복잡하게 늘리지 않고 고정 방향광 + 약한 환경광 사용
- 색 단계는 2~4단계로 제한하여 카툰 느낌을 유지

외곽선 구현 권장:

- 모델을 복제한다.
- 복제 모델의 법선 방향을 살짝 확장한다.
- 검정 또는 짙은 남색 단색 재질을 입힌다.
- 원본 모델보다 먼저 또는 뒤집힌 면 기준으로 렌더링해 외곽선처럼 보이게 한다.

렌더링 톤:

- 실사 금지
- 과한 피 표현 금지
- 캐릭터 실루엣과 색 구분 우선
- 감염체는 우스꽝스럽지만 위협적인 카툰 비율 사용

## 7. Character Visual Concept

### 7-1. Player

주인공은 감염된 학교에서 탈출하려는 학생이다.

실루엣:

- 작은 체형
- 큰 머리, 짧은 몸통
- 등에 가방
- 한 손에 연필 또는 임시 무기
- 이동 방향에 따라 몸이 살짝 기울어짐

색상:

- 교복 기본색: 남색 또는 짙은 회색
- 포인트: 노란 명찰, 하늘색 가방 장식
- 체력 낮음 상태: 캐릭터 주변 붉은 가장자리 점멸

필수 애니메이션:

- `idle`: 제자리 호흡
- `run`: 빠른 도망 느낌의 짧은 보폭
- `hit`: 0.12초 흰색 플래시 + 뒤로 살짝 흔들림
- `levelUp`: 위쪽으로 빛나는 링 1회
- `death`: 넘어짐보다 화면 밖으로 쓰러지는 단순 연출

### 7-2. Common Zombie: Infected Student

기획 대응: 감염된 학생

- 빠르고 약한 기본 적
- 교복이 흐트러진 학생
- 형광 녹색 눈
- 팔을 앞으로 뻗고 빠르게 접근
- 몸 크기: 플레이어와 비슷하거나 0.95배

색상:

- 탁한 교복색
- 녹색 감염 포인트
- 외곽선 검정

### 7-3. Heavy Zombie: Infected PE Teacher

기획 대응: 감염된 체육교사

- 느리지만 단단한 중형 적
- 체육복, 호루라기, 큰 어깨
- 넉백 저항이 시각적으로 느껴지도록 낮은 자세
- 몸 크기: 일반 적 1.35배

시각 신호:

- 맞을 때 밀림이 작다.
- 접근 시 바닥에 무거운 원형 그림자.

### 7-4. Swarm Zombie: Small Infected Students

기획 대응: 감염된 소형 학생 떼

- 작고 빠른 무리
- 동일 모델을 2~3가지 색 변형으로 사용
- 몸 크기: 일반 적 0.7배
- 한 번에 여러 마리 등장하므로 디테일보다 가독성 우선

시각 신호:

- 무리 단위로 약한 먼지 이펙트
- 체력이 낮으므로 피격 시 강한 플래시 후 빠르게 소멸

### 7-5. Ranged Zombie: Pollution Spitter

기획 대응: 오염 침뱉개

- 원거리 오염 탄을 발사하는 적
- 구부정한 자세
- 입 또는 손에 녹색 오염 덩어리
- 몸 크기: 일반 적 1.05배

시각 신호:

- 발사 0.35초 전 입 주변 녹색 빛 충전
- 오염 탄은 둥근 초록 투사체
- 탄 경로가 너무 어둡지 않게 밝은 테두리 적용

### 7-6. Charger Zombie: Frenzy Runner

기획 대응: 광폭 질주 감염체

- 돌진으로 회피 실수를 유도하는 적
- 운동부 학생처럼 보이게 운동복 또는 번호표 사용
- 몸을 낮추고 직선으로 달리는 실루엣
- 몸 크기: 일반 적 1.15배

시각 신호:

- 돌진 0.7초 전 주황색 직선 예고선 표시
- 예고 중 캐릭터가 떨림
- 돌진 실패 후 1초 동안 별 모양 또는 빙글 이펙트

### 7-7. Elite Zombie: Special Infected Teacher

기획 대응: 특수 감염 교사

- 중간 난이도 봉우리
- 교사 실루엣 + 감염된 팔 또는 큰 분필 막대
- 몸 크기: 일반 적 1.7배

시각 신호:

- 등장 시 화면 상단에 작은 경고 텍스트
- 발밑에 보라색 링
- 충격파 공격 전 원형 경고 범위 표시
- 처치 시 대형 XP 오브와 회복 오브가 눈에 띄게 튀어나옴

### 7-8. Boss: Infected Principal

기획 대응: 5분 보스 교장 감염체

- 스테이지 종료 목표
- 교장 정장, 찢어진 넥타이, 큰 그림자
- 감염된 팔 또는 등에 오염 덩어리
- 몸 크기: 일반 적 2.5배

필수 패턴 시각화:

- 근접 내려찍기: 부채꼴 또는 원형 붉은 예고 영역
- 바닥 오염지대: 초록색 끈적한 웅덩이 3개
- 전방 오염 분사: 넓은 부채꼴 녹색 경고 범위
- HP 50퍼센트 이하: 몸의 감염 발광이 강해지고 장판 생성 이펙트 증가

## 8. Weapon Visual Concept

### 8-1. Pencil Throw

기획 대응: 연필 투척

- Phaser 투사체로 구현한다.
- 노란 연필 오브젝트가 가장 가까운 적 방향으로 날아간다.
- 레벨 3 이상부터 양옆 각도로 추가 연필을 발사한다.
- 관통 발생 시 연필 뒤에 짧은 흰색 속도선을 남긴다.

### 8-2. School Bag Swing

기획 대응: 책가방 휘두르기

- 플레이어 주변을 도는 2D 또는 간단한 3D 가방 오브젝트.
- 레벨에 따라 회전 가방 수 증가.
- 타격 순간 작은 충격 원과 노란 별 이펙트.
- 방어형 무기이므로 플레이어 주변 안전지대가 보이게 한다.

### 8-3. Emergency Bell Shockwave

기획 대응: 비상벨 충격파

- 상하좌우 또는 8방향으로 퍼지는 붉은 링/파형.
- 사운드가 없어도 화면에서 "비상벨" 느낌이 나도록 빨간 원형 파동 사용.
- 레벨 3부터 대각선 방향 추가.

### 8-4. Electric Stun Gun

기획 대응: 전기 스턴건

- 가까운 적에게 노란 전기 줄 연결.
- 연쇄 대상은 얇은 전기선으로 이어진다.
- 엘리트 추가 피해 시 보라색 스파크를 섞는다.

## 9. VFX Rules

모든 이펙트는 짧고 명확해야 한다. 적이 많아지는 3분 이후에도 화면을 덮어서는 안 된다.

필수 이펙트:

- 피격: 흰색 플래시 0.08~0.12초
- 사망: 작게 터지는 감염 연기 + XP 오브 생성
- 레벨업: 플레이어 중심 빛 링 1회
- 보스 등장: 화면 가장자리 붉은 경고 1초
- 오염 장판: 가장자리 초록 테두리 + 내부 반투명
- 자석 아이템: 화면의 XP 오브가 플레이어에게 빨려 들어가는 곡선 이동

금지:

- 적 피격마다 큰 파티클 폭발
- 어두운 검정 연기 남발
- 플레이어와 적을 가리는 장시간 광원 효과

## 10. Drop Item Visuals

### XP Orb

- 소형: 청록색 작은 구슬
- 중형: 파란빛이 섞인 청록 구슬
- 대형: 밝은 청록색 다발
- 바닥에 떨어진 뒤 0.2초 튀고 정지

### Heal Orb

- 분홍색 또는 밝은 초록색 십자 아이콘 구슬
- 특수 감염체 처치 시 50퍼센트 확률 드랍
- XP 오브와 혼동되지 않게 십자 마크를 반드시 넣는다.

### Magnet Item

- 자석 모양 아이콘
- 보스 직전 역전 아이템이므로 화면에서 강하게 구분
- 색상: 보라색 + 흰색 테두리

## 11. UI Graphic Direction

UI는 장식보다 정보 전달이 우선이다.

필수 UI:

- 상단 중앙: 남은 시간 또는 경과 시간
- 상단 좌측: 플레이어 체력바
- 상단 우측: 현재 레벨
- 하단: 현재 장착 무기 아이콘 4칸
- 레벨업 시: 3개 선택 카드
- 보스 등장 시: 경고 배너

UI 스타일:

- 학교 방송실/안내판 느낌
- 카드 배경은 어두운 회색
- 선택 가능한 항목은 노란 테두리
- 위험 메시지는 붉은색
- 글자는 모바일에서 읽히도록 크게

## 12. Technical Implementation Notes

### Phaser Responsibilities

- 씬 전환
- 타일맵
- 카메라
- 입력
- 충돌
- 스폰
- 투사체

- UI
- 2D 이펙트

### Three.js Responsibilities

- 플레이어 3D 모델 렌더링

- 카툰 재질
- 외곽선
- 캐릭터 애니메이션

### Coordinate Sync

Phaser 월드 좌표를 기준으로 Three.js 캐릭터 위치를 갱신한다.

권장 방식:

- Phaser의 플레이어, 몬스터 오브젝트는 논리 객체로 유지한다.
- 각 논리 객체에 `visualId`를 둔다.
- Three.js 씬은 `visualId`에 대응하는 모델을 가진다.
- 매 프레임 Phaser 좌표를 Three.js 좌표로 변환한다.
- Phaser 카메라 이동량을 Three.js 카메라 또는 렌더 그룹에 반영한다.

좌표 변환 기준:

- Phaser x -> Three x
- Phaser y -> Three z
- Three y -> 캐릭터 높이

### Performance Rules

- 화면 내 몬스터 100마리 이상을 목표로 잡는다.
- 일반 좀비는 모델 디테일을 낮추고 색상 변형으로 다양성을 만든다.
- 같은 타입 몬스터는 가능하면 인스턴싱 또는 모델 풀링을 사용한다.
- 사망한 몬스터 모델은 제거하지 말고 풀에 반환한다.
- 외곽선은 모든 작은 몬스터에 과하게 두껍게 적용하지 않는다.
- 저사양 모드에서는 먼 몬스터 외곽선 또는 그림자를 줄인다.

# 16. Actual Stage Graphic Implementation Plan

## 16-1. 구현 목표

이 계획서는 1스테이지 "감염된 학교 운동장과 본관 주변"을 실제 게임 화면으로 구현하기 위한 작업 지시서이다.

목표는 아래 3가지를 동시에 만족하는 것이다.

- 플레이어가 즉시 읽을 수 있는 모바일 세로형 화면
- Phaser 4 기반 2D 스테이지와 UI
- Three.js 기반 카툰 렌더링 플레이어/감염체 캐릭터

첫 구현은 완성된 아트보다 "읽히는 플레이 화면"을 우선한다.

## 16-2. 최종 화면 구성

기준 화면:

- 화면 비율: 9:16
- 기준 논리 해상도: 720 x 1280
- 대표 고해상도 검수 해상도: 1080 x 1920
- 실제 렌더링 해상도: 기기 해상도와 `devicePixelRatio`에 맞춰 자동 스케일
- UI 안전 영역: 상단 노치와 하단 제스처 영역을 침범하지 않게 배치
- 카메라: 플레이어 중심 추적
- 시점: 3/4 탑다운
- 스테이지: 가짜 무한 학교 부지

화면 구성:

- 중앙: 학생 생존자
- 주변: 감염된 학생과 감염체 웨이브
- 바닥: 운동장 트랙과 콘크리트 학교 부지 타일
- 외곽: 본관 그림자, 사물함, 책상, 경고 테이프
- 상단 UI: 생존 시간, 체력, 레벨
- 하단 UI: 장착 무기 4칸
- 전투 중: XP 오브, 회복 오브, 오염 장판, 투사체, 경고선

## 16-3. Phaser 레이어 구현 계획

Phaser 레이어는 아래 순서로 만든다.

### 1. `BackgroundTileLayer`

담당:

- 운동장 기본 바닥
- 콘크리트 바닥
- 복도 느낌의 타일
- 반복 배치되는 가짜 무한 맵 바닥

구현 방법:

- 256 x 256 또는 512 x 512 크기의 반복 타일을 사용한다.
- 카메라 이동에 따라 타일을 재배치해서 무한 맵처럼 보이게 한다.
- 첫 버전에서는 3종 타일만 사용한다.

필수 타일:

- `tile_school_interior_floor`

### 2. `GroundDetailLayer`

담당:

- 찢어진 시험지
- 작은 오염 자국
- 바닥 균열
- 작은 쓰레기
- 운동장 선 부분 장식

구현 방법:

- 충돌 없는 장식 스프라이트로 배치한다.
- 화면을 복잡하게 만들지 않도록 밀도를 낮게 유지한다.
- 플레이어 주변 1화면 안에는 과도하게 배치하지 않는다.

배치 기준:

- 화면 1개 기준 장식 8~15개 이하
- 바닥색보다 약간 밝거나 어둡게 처리
- XP 오브 색상과 겹치지 않게 청록색 사용 금지

### 3. `ObstacleLayer`

담당:

- 쓰러진 책상
- 의자 더미
- 오염된 사물함
- 안전 콘
- 감염 경고 테이프
- 임시 바리케이드

구현 방법:

- 충돌 오브젝트와 장식 오브젝트를 구분한다.
- 충돌 오브젝트는 단순한 사각형 또는 원형 충돌 박스를 사용한다.
- 완전 막힘 구역을 만들지 않는다.

배치 기준:

- 실제 이동 방해는 전체 맵의 15퍼센트 이하
- 중앙부 회피 공간은 비운다.
- 원거리 오염 탄이 장애물 뒤에서 갑자기 맞는 상황을 만들지 않는다.

### 4. `ShadowLayer`

담당:

- 플레이어 바닥 그림자
- 감염체 바닥 그림자
- 보스 큰 그림자

구현 방법:

- Phaser의 반투명 타원 스프라이트 또는 그래픽 객체로 처리한다.
- Three.js 실시간 그림자보다 저렴한 고정 그림자를 우선한다.
- 캐릭터 크기에 맞춰 그림자 스케일을 조정한다.

그림자 기준:

- 플레이어: 작은 타원
- 일반 감염체: 플레이어와 비슷한 타원
- 체육교사: 1.35배 타원
- 보스: 2.5배 이상 넓은 타원

### 5. `ThreeCharacterCanvas`

담당:

- 학생 생존자 모델
- 감염체 모델 전체
- Three.js 카툰 렌더링 결과

구현 방법:

- Phaser 게임 캔버스 위 또는 아래에 Three.js 캔버스를 합성한다.
- Phaser 월드 좌표를 Three.js 좌표로 변환한다.
- 캐릭터 모델은 Phaser 논리 객체의 위치를 따라간다.

주의:

- Three.js 캐릭터는 시각 표현 전용이다.
- 충돌 판정은 Phaser 논리 객체가 담당한다.
- UI보다 아래, 배경보다 위에 보이게 렌더 순서를 조정한다.

### 6. `ProjectileLayer`

담당:

- 연필 투척
- 오염 탄
- 비상벨 충격파
- 전기 스턴건 연결선

구현 방법:

- 빠른 판정과 수량 관리를 위해 Phaser 2D 오브젝트로 구현한다.
- 투사체는 화면 가독성을 위해 밝은 테두리를 둔다.
- 발사체 상한을 둔다.

초기 상한:

- 플레이어 투사체: 80개
- 적 투사체: 40개
- 전기 연결선: 20개

### 7. `VFXLayer`

담당:

- 피격 플래시
- 사망 감염 연기
- 레벨업 링
- 돌진 예고선
- 충격파 경고 원
- 보스 장판
- 보스 부채꼴 분사 경고

구현 방법:

- Phaser Graphics 또는 짧은 스프라이트 애니메이션으로 구현한다.
- 지속 시간이 긴 이펙트는 반투명으로 처리한다.
- 플레이어를 가리는 불투명 이펙트는 금지한다.

### 8. `DropLayer`

담당:

- XP 오브
- 회복 오브
- 자석 아이템

구현 방법:

- 작은 스프라이트와 약한 반짝임 애니메이션 사용
- XP 오브는 청록색
- 회복 오브는 십자 마크 필수
- 자석 아이템은 보라색과 흰색 테두리 사용

### 9. `UILayer`

담당:

- 생존 시간
- 체력바
- 현재 레벨
- 무기 슬롯 4칸
- 레벨업 선택 카드
- 보스 경고 배너

구현 방법:

- Phaser UI로 구현한다.

- 예고와 실제 피해 영역이 일치해야 한다.

## 16-8. UI 그래픽 구현 계획

### 생존 시간

- 위치: 상단 중앙
- 표현: `04:32` 형식
- 보스 직전 10초부터 색상 강조

- 위치: 상단 좌측
- 색상: 정상 초록, 위험 빨강

- 표현: `Lv. 4`

- 획득하지 않은 슬롯은 어둡게 처리

### 레벨업 선택 카드

- 개수: 3개
- 카드 톤: 어두운 회색
- 선택 가능 테두리: 노란색
- 카드 안에는 아이콘, 이름, 효과를 표시

## 16-9. 구현 단계별 체크리스트

### 단계 1. 바닥과 카메라

완료 기준:

- 기준 논리 해상도 720 x 1280에서 운동장 바닥이 보인다.
- 대표 검수 해상도 1080 x 1920에서도 바닥 타일과 UI 비율이 깨지지 않는다.
- 플레이어 기준 카메라가 이동한다.
- 바닥 타일이 끊기지 않고 반복된다.

### 단계 2. 플레이어와 기본 감염체 표시

완료 기준:

- `StudentSurvivor`가 Three.js 카툰 모델로 보인다.
- `InfectedStudent`가 Three.js 카툰 모델로 보인다.
- 두 캐릭터의 외곽선이 보인다.
- Phaser 좌표와 Three.js 모델 위치가 어긋나지 않는다.

### 단계 3. 기본 전투 시각화

완료 기준:

- 연필 투척이 적 방향으로 날아간다.
- 피격 시 흰색 플래시가 나온다.
- 적 사망 시 XP 오브가 나온다.
- XP 오브가 청록색으로 구분된다.

### 단계 4. 적 타입 확장

완료 기준:

- 감염된 체육교사가 일반 감염 학생보다 크고 무겁게 보인다.
- 소형 감염 학생 떼가 작고 무리로 읽힌다.
- 오염 침뱉개는 발사 전 충전 표시가 있다.
- 광폭 질주 감염체는 돌진 전 주황 예고선이 있다.

### 단계 5. 엘리트와 보스 시각화

완료 기준:

- 특수 감염 교사는 보라색 링과 충격파 경고 원을 가진다.
- 교장 감염체는 일반 적과 확실히 다른 크기와 실루엣을 가진다.
- 보스 장판 3개가 명확히 보인다.
- 보스 부채꼴 분사 경고가 실제 피해 범위와 일치한다.

### 단계 6. 모바일 가독성 조정

완료 기준:

- 9:16 세로 화면에서 플레이어가 항상 먼저 보인다.
- 720 x 1280 논리 해상도와 1080 x 1920 검수 해상도 모두에서 UI가 안전 영역 안에 있다.
- 3분 이후 적 밀도에서도 투사체와 오브가 구분된다.
- UI가 게임 오브젝트를 과하게 가리지 않는다.
- 색상 대비가 충분하다.

## 16-10. 그래픽 QA 기준

그래픽 구현 후 QA는 아래 항목을 확인한다.

- 플레이어와 적이 배경에 묻히지 않는가
- 감염체 타입이 크기와 색상으로 구분되는가
- 오염 탄과 XP 오브가 헷갈리지 않는가
- 회복 오브는 십자 마크로 식별되는가
- 위험 장판은 피해 발생 전에 충분히 보이는가
- 보스 공격 예고와 실제 판정이 일치하는가
- 외곽선이 너무 두꺼워 화면이 지저분하지 않은가
- 저사양 모드에서 그림자와 외곽선 감소 정책이 있는가
- 모든 그래픽 요소가 Phaser/Three.js 역할 분리에 맞는가

## 16-11. 첫 프로토타입에서 만들지 않을 것

아래 항목은 첫 그래픽 프로토타입에서 제외한다.

- 정교한 교실 내부 맵
- 복잡한 실시간 조명
- 캐릭터별 고급 표정 애니메이션
- 실사형 피와 상처 표현
- 컷신
- 대화창 중심 스토리 연출
- 고해상도 3D 배경
- 물리 기반 파편 효과

첫 프로토타입은 생존 화면이 읽히는지 검증하는 것이 목적이다.
