## 1. 전략적 장애물의 시각적 조건

### 1-1. "막혀있다" 0.5초 판독 조건

플레이어가 빠른 이동 판단을 내리는 시간은 약 0.5초다. 이 시간 안에 장애물이 "통과 불가"임을 전달하려면 아래 세 가지 조건이 동시에 충족되어야 한다.

| 조건 | 기준 | 이유 |
|------|------|------|
| 실루엣 크기 | 화면 타일 1칸(약 64px @ 720p) 이상 | 시야각 안에서 즉시 덩어리감이 읽힘 |
| 수직 돌출감 | Y축 높이 0.6유닛 이상 (바닥면 기준) | 3/4 탑다운 시점에서 그림자와 높이가 겹쳐야 "서 있는 물체"로 인식 |
| 색-형 대비 | 바닥 `#a99a73` 대비 ΔL* ≥ 30 (CIELAB 기준) | 낮은 채도 바닥 위에서 pop |

### 1-2. 높이/부피감 기준

현재 ClassroomFloor.jsx의 props는 대부분 높이 0.5유닛 이하로 설정되어 있다. 3/4 탑다운 시점에서 높이가 낮으면 바닥 데칼처럼 보여 장애물 판독이 어렵다.

**권장 최소 높이 기준:**

```
장식용 바닥 레이어 (시험지, 그림자)  →  Y ≤ 0.05  (바닥 데칼)
슬로우/데미지 존 (웅덩이, 테이프)   →  Y ≤ 0.05  (지면 레이어)
소형 이동 방해 (안전 콘)             →  Y ≥ 0.50  (캐릭터 무릎 높이)
중형 장애물 (책상, 의자 더미)        →  Y ≥ 0.80  (캐릭터 허리 높이)
대형 장애물 (사물함, 바리케이드)     →  Y ≥ 1.40  (캐릭터 전신 높이)
```

### 1-3. 배경 바닥(`#a99a73`)과의 색상 대비

바닥 주색 `#a99a73`(황갈색 나무)은 중채도 따뜻한 톤이다. 장애물이 이 위에서 읽히려면 아래 기준을 따른다.

| 바닥 색 | 장애물 유형 | 권장 주색 | 대비 방향 |
|---------|------------|----------|----------|
| `#a99a73` | 완전 막힘 물체 | `#3d2a1a` ~ `#5c3d1a` | 명도 ↓↓ (어두운 갈색) |
| `#a99a73` | 감염 오염 물체 | `#2f7571` ~ `#41745a` | 색상 대비 (청록-녹) |
| `#a99a73` | 경고 신호 물체 | `#FF6600` ~ `#FFD600` | 보색 대비 (주황-노랑) |
| `#a99a73` | toon outline | `#1a1a1a` ~ `#2d2738` | 명도 최대 대비 |

**핵심 원칙:** 완전 막힘 장애물의 toon outline은 반드시 `#1a1a1a` 이상 어두운 색, 두께는 2px 이상(화면 기준)이어야 바닥과 분리된다.

## 2. 각 Prop별 비주얼 강화 제안

### PropFallenDesk (쓰러진 책상)

| 항목 | 현재 값 | 제안 값 | 비고 |
|------|--------|--------|------|
| 책상판 크기 | `[1.6, 0.08, 0.7]` | `[2.2, 0.12, 1.0]` | 넓이 +37%, 두께 +50% |
| 다리 길이 | 0.55유닛 (높이 0.22) | 0.75유닛 (높이 0.30) | 쓰러진 상태에서도 부피감 확보 |
| 전체 높이감 | ~0.35유닛 | ~0.45유닛 | 바닥 데칼 탈피 |
| 책상 색 | `0x5C3D1A` (갈색) | `0x5C3D1A` 유지 + roughness 강화 | 바닥과 명도차 확보 |
| 다리 색 | `0x3E2710` | `0x2a1a08` | 더 어둡게 — 명도 대비 향상 |
| toon outline 두께 | 미적용 (MeshLambert) | 2.5px 권장 | inverted hull 방식 |
| 충돌 판정 | 없음 | 사각형 박스 추가 제안 | 실제 이동 방해 |

**색상 강화 방향:** 현재 책상 갈색 `0x5C3D1A`는 바닥 `#a99a73`보다 명도가 낮아 구분은 되나 채도가 비슷하다. 책상판 상면에 `0x805947`(어두운 변형) 강조 레이어를 얇게 추가하면 깊이감이 생긴다.

| 항목 | 현재 값 | 제안 값 | 비고 |
|------|--------|--------|------|
| 본체 크기 | `[0.55, 1.4, 0.45]` | `[0.70, 1.6, 0.55]` | 폭 +27%, 깊이 +22% |
| 전체 높이 | 1.4유닛 | 1.6유닛 | 캐릭터 전신 가리는 높이 |
| 본체 색 | `0x4a5c4a` (탁한 초록) | `0x3d4e3d` | 더 어둡게 — 감염 계열 |
| 오염 얼룩 색 | `0x2d5c2d` | `0x2f7571` | Poison 팔레트 청록 — 독성 강화 |
| 오염 얼룩 수 | 2개 | 3~4개 | 상단+측면으로 확장 |
| 오염 얼룩 opacity | 0.75 | 0.85 | 더 진하게 |

**색상 강화 방향:** 사물함은 완전 막힘 장애물이므로 toon outline이 가장 중요하다. `#1a1a1a` 검정 outline 3px + 오염 흘러내림 `#2f7571`의 조합이 "감염된 장애물"을 0.5초 안에 전달한다.

## 3. "통과 불가" vs "슬로우/데미지 존" 시각 구분 가이드

| 타입 | 주색 | outline 색 | outline 두께 | 투명도 | 높이/레이어 | 기타 |
|------|------|-----------|-------------|--------|------------|------|
| 완전 막힘 | `#3d2a1a`~`#4a5c4a` (갈색/올리브) | `#1a1a1a` | 2.5~3.0px | 불투명 1.0 | 0.8~1.6유닛 수직 | 그림자 필수, inverted hull outline |
| 슬로우 존 | `#FFD600`/`#1A1A1A` (경고 테이프) | `#1a1a1a` | 1.5px 가장자리 | 0.88~0.95 | 바닥 데칼 Y≤0.05 | 사선 패턴으로 인식 |
| 데미지 존 (웅덩이) | `#2f7571`~`#41745a` (청록-감염녹) | `#41745a` | 3~4px 원형 | 0.55~0.75 | 바닥 데칼 Y≤0.05 | 펄스 없음, 보스 장판과 구분 |
| 보스 위험 장판 (참고) | `#95bf91` (선명 녹) | `#95bf91` | 5~6px | 0.40~0.60 | 바닥 데칼 Y≤0.05 | 펄스 애니메이션 필수, 가장 밝음 |

**판독 우선순위 체계:**
1. 높이 있는 3D 물체 → 완전 막힘
2. 바닥 데칼 + 사선/줄무늬 패턴 → 슬로우 구역
3. 바닥 데칼 + 녹청색 유기적 형태 → 데미지 구역
4. 바닥 데칼 + 선명 녹 + 펄스 → 보스 위험 (최고 위험)

## 4. 새로운 "바리케이드" Prop 제안

### PropBarricadeSchool (학교 임시 바리케이드)

**컨셉:** 좀비 사태 초기 누군가 급조한 방어선. 책상 2개 + 의자 1~2개 + 경고 테이프를 가로로 쌓아 막아 놓은 형태. 사물함보다 넓고 낮다.

**학교 설정 리얼리티:**
- 학생들이 대피 중 복도에 밀어놓은 교실 비품
- 넥타이나 체육복 벨트로 묶인 느낌 (텍스처 디테일)
- 경고 테이프가 X자로 감겨 있음

**크기 스펙:**

```
전체 폭:    3.0 ~ 4.0유닛  (복도/통로 폭 차단 가능)
전체 높이:  1.1 ~ 1.3유닛  (캐릭터 키와 비슷)
전체 깊이:  0.6 ~ 0.8유닛
```

**구성 요소:**

| 요소 | 크기 | 색 | 비고 |
|------|------|-----|------|
| 아래 책상 (수평) | [2.2, 0.12, 1.0] | `#5C3D1A` | 책상 뒤집어 쌓음 |
| 위 책상 (비스듬히) | [2.0, 0.12, 0.9] | `#4a2e1a` | rotation [0, 0, 0.15] |
| 의자 × 2 | 기존 PropChairPile 재사용 | `#3a2008` | 측면 지지대 역할 |
| 경고 테이프 | [4.0, 0.30] plane | `#FFD600`/`#1A1A1A` | X자 대각선 방향으로 2장 교차 |
| toon outline | 전체 | `#1a1a1a` | 3.0px |

**색상 강화 방향:**
- 바리케이드 전체 실루엣이 바닥 `#a99a73`에서 팝 되어야 한다
- 어두운 갈색 `#3a2008`~`#5C3D1A` 계열로 통일
- 경고 테이프 X자가 포인트 컬러 역할 (`#FFD600`)
- 바리케이드는 "완전 막힘" → outline 3px 필수

**배치 예시 (ClassroomFloor 기준):**
```jsx
// 좌우 출입구 차단 용도
<PropBarricadeSchool position={[-8, 0, -18]} rotation={[0, 0, 0]} />
<PropBarricadeSchool position={[ 8, 0,  18]} rotation={[0, Math.PI, 0]} />
```

## 5. 모바일 가독성 체크리스트 (390×844 세로 화면)

기준: 390×844 논리픽셀, devicePixelRatio=3 (실 렌더 1170×2532)  
3/4 탑다운 시점, 플레이어 중심 카메라

### 화면 내 prop 예상 픽셀 크기 (참고값)

논리 해상도 390px 폭 = 게임 월드 약 12~15유닛 표시.  
→ 1유닛 ≈ 26~32px (논리픽셀)

| Prop | 현재 폭 | 현재 화면 px | 제안 폭 | 제안 화면 px | 가독성 |
|------|---------|------------|--------|------------|--------|
| PropFallenDesk | 1.6유닛 | ~42~51px | 2.2유닛 | ~57~70px | OK |
| PropChairPile | 0.55유닛 | ~14~18px | 0.65유닛 | ~17~21px | 경계선 — outline 필수 |
| PropContamLocker | 0.55유닛 폭 | ~14~18px | 0.70유닛 | ~18~22px | 높이로 보완 |
| PropWarningTape | 3.5×0.22유닛 | 폭 충분, **높이 5~7px** | 4.5×0.35유닛 | **높이 9~11px** | 높이 미달 위험 |
| PropSafetyCone | 0.24유닛 폭 | ~6~8px | 0.56유닛 폭 | ~15~18px | 현재 너무 작음 |
| PropContamPuddle | 1.98~2.86유닛 | ~52~91px | 3.0~4.5유닛 | ~78~144px | OK |
| PropBarricadeSchool (신규) | 3.0~4.0유닛 | ~78~128px | — | — | OK |

### 체크리스트

**PropFallenDesk**
- [ ] 390px 폭 화면에서 책상판이 50px 이상인가
- [ ] 쓰러진 다리가 바닥에 묻히지 않고 silhouette로 읽히는가
- [ ] toon outline 추가 후 배경 타일과 구분되는가

**PropChairPile**
- [ ] 2단 적층 구조가 하나의 덩어리로 읽히는가
- [ ] 작은 다리(~14~18px)가 outline 없이 바닥에 묻히지 않는가
- [ ] 의자 더미가 플레이어(캐릭터)보다 확실히 구분되는 색인가

**PropContamLocker**
- [ ] 사물함 폭이 좁아서(~18~22px) 얇게 보이지 않는가 — 폭 확대 필수
- [ ] 오염 흘러내림이 390px 화면에서 읽히는가 (최소 5px 폭)
- [ ] 사물함이 플레이어보다 항상 앞/뒤로 명확히 분리 렌더링되는가

**PropWarningTape**
- [ ] 테이프 폭이 최소 9px(논리) 이상인가 — 현재 5~7px 미달
- [ ] 노란/검정 사선 패턴이 1칸 이상(최소 2주기) 보이는가
- [ ] 테이프 위를 걷는 캐릭터 발이 테이프에 묻히지 않는가

**PropSafetyCone**
- [ ] 현재 6~8px는 터치 조작 중 인식 불가 — 15px 이상 필수
- [ ] 주황 콘 실루엣이 배경과 즉시 구분되는가
- [ ] 콘 4개가 집중 배치될 때 서로 묻히지 않는가

**PropContamPuddle**
- [ ] 웅덩이 경계선(outline)이 390px 화면에서 3px 이상으로 보이는가
- [ ] 보스 장판과 색상으로 구분되는가 (청록 vs 선명 녹)
- [ ] 웅덩이 위 캐릭터가 웅덩이에 묻히지 않는가 (renderOrder 확인)

**PropBarricadeSchool (신규)**
- [ ] 바리케이드가 통로 폭을 막는 스케일로 보이는가
- [ ] 경고 테이프 X자가 "진입 금지" 신호로 즉시 읽히는가
- [ ] 바리케이드 뒤 적 탄이 갑자기 나타나지 않는가 (배치 룰 확인)

### 모바일 공통 원칙

1. **outline 없으면 소형 prop은 불가시** — PropChairPile, PropSafetyCone은 outline 없이 390px 화면에서 소실된다. inverted hull outline 2px 최소값 필수.
2. **바닥 데칼은 opacity 0.9 이상** — PropWarningTape, PropContamPuddle은 opacity가 낮으면 배경 나무 텍스처에 묻힌다.
3. **장애물 높이 ≥ 0.5유닛 = 13px 이상** — 이 이하는 바닥 장식으로 읽혀 이동을 막을 수 없다는 인식을 주지 못한다.
4. **색상 대비 ΔL* ≥ 30** — 밝기 차이가 30 미만이면 소형 화면에서 분리 인식이 어렵다. 특히 PropContamLocker(`#4a5c4a`)와 바닥(`#a99a73`)의 차이를 확인해야 한다.
5. **그림자 필수** — 높이가 있는 prop에는 바닥 타원 그림자를 추가해야 3/4 탑다운에서 "서 있는 물체"로 인식된다.
