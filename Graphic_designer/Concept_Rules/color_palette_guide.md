## 1. 팔레트 선정 근거

탐색 결과 본 게임에 가장 적합한 색채 체계로 **Survival Horror 32**를 기준 팔레트로 채택한다.

선정 이유:
- 어두운 공포 분위기와 카툰 액션 가독성을 동시에 만족하는 32색 구성
- 감염 녹색, 위험 빨간, 공포 보라, 나무/피부 계열이 모두 포함됨
- "어두운 색만 쓰면 지루하다"는 원칙으로 밝은 강조색 포함
- Resident Evil, Silent Hill의 색채 설계 원칙과 동일한 구조

## 4. 색채 설계 원칙

### 원칙 1 — 베이스는 탁하게, 강조는 하나만

Resident Evil 방식: 채도 낮은 베이스 + 캐릭터 1색 강조.  
플레이어의 `0x59c7ff` 하늘색이 유일한 선명한 강조색이 되어야 한다.  
다른 모든 색은 탁한 중간톤 계열을 유지한다.

### 원칙 2 — 차가운 색 = 안전, 따뜻한 색 = 위험

Silent Hill 방식:  
- 바닥·배경: 중립 크림/나무색 계열
- 감염체: 탁한 녹색 계열 (차갑지만 불안함)
- 위험 공격·장판: 주황/빨간 계열 (따뜻하고 즉각적 위험)
- 보스 등장 시: 화면 전체 색온도를 따뜻하게 시프트

### 원칙 3 — 색으로 타입 구분 (Vampire Survivors 방식)

Three.js 캐릭터는 실루엣이 동일하므로 색이 타입 구분의 유일한 수단.  
같은 녹색 계열이어도 명도·채도 차이로 타입을 즉시 구분해야 한다.  
감염 학생(`#95bf91`) vs 감염 학생 떼(`#679a73`) — 명도 차이로 구분.

### 원칙 4 — 역할이 다른 것은 계열이 달라야 한다

| 역할 | 색 계열 |
|------|---------|
| 플레이어 | 하늘색 단색 |
| 일반 적 | 녹색 계열 |
| 엘리트 적 | 보라 계열 |
| 보스 | 어두운 녹+레드 |
| XP 오브 | 밝은 녹색 (Poison) |
| 회복 오브 | 살구/핑크 |
| 위험 이펙트 | 주황/레드 |
| UI | 크림+회청 |

```javascript
// 변경 전 → 변경 후
infectedStudent:      tint: 0x76ef82  →  0x95bf91
infectedStudentSwarm: tint: 0xa4ff70  →  0x679a73
infectedPETeacher:    tint: 0xcc6633  →  0x805947
pollutionSpitter:     tint: 0x7cffc4  →  0x2f7571
frenzyRunner:         tint: 0xff9c3d  →  0xe99039
specialInfectedTeacher: tint: 0xb06dff → 0x6f779e
BOSS_DEFINITION:      tint: 0x9bff64  →  0x95bf91
```

`game.js` VFX 색상 변경:

```javascript
// 감염 장판
fillStyle(0x1a7a3a, 0.32)  →  fillStyle(0x41745a, 0.32)
// 장판 테두리

// 엘리트 충격파 경고
보라 계열                  →  0x693054
```
