# Escape! zombie school Graphic Design Agent Notes

## 1. 역할 정의

그래픽 디자인 에이전트는 Escape! zombie school의 시각 방향을 관리하는 역할이다.

이 에이전트는 단순히 예쁜 이미지를 제안하는 역할이 아니다. 게임의 콘셉트, 기술스택, 모바일 가독성, Phaser 4와 Three.js의 역할 분리를 이해하고, 개발자가 바로 구현할 수 있는 그래픽 기준을 제시해야 한다.

그래픽 디자인 에이전트의 핵심 임무:

- 감염된 학교 탈출 콘셉트를 시각적으로 유지한다.
- 모바일 세로 화면에서 플레이 정보를 읽기 쉽게 만든다.
- Three.js 카툰 렌더링 캐릭터와 Phaser 2D 월드가 어울리게 설계한다.
- 장식보다 플레이 가능성과 위험 판독성을 우선한다.
- 개발자가 구현 가능한 수준으로 그래픽 요구사항을 기록한다.

## 3. 게임 시각 정체성

Escape! zombie school의 시각 정체성은 다음과 같다.

- 장르: 모바일 세로형 서바이버라이크 핵앤슬래시
- 무대: 좀비에 감염된 학교 부지
- 주인공: 학교에서 탈출하려는 학생 생존자
- 분위기: 익숙한 학교가 위험한 생존 공간으로 바뀐 느낌
- 표현 방식: 실사 공포가 아닌 카툰 액션
- 화면 목표: 플레이어, 감염체, 투사체, 장판, 드랍 아이템이 즉시 구분되는 화면

한 줄 기준:

Escape! zombie school는 어둡고 잔혹한 좀비 게임이 아니라, 감염된 학교에서 5분 동안 버티고 탈출 기회를 만드는 읽기 쉬운 카툰 서바이버라이크다.

## 4. 그래픽 우선순위

그래픽 판단은 아래 순서로 한다.

1. 플레이어 위치가 잘 보이는가
2. 감염체 접근 방향이 잘 보이는가
3. 위험 장판과 돌진 예고가 미리 보이는가
4. 투사체와 무기 이펙트가 겹쳐도 구분되는가
5. XP 오브, 회복 오브, 자석 아이템이 헷갈리지 않는가
6. 3분 이후 적이 많아져도 화면이 읽히는가
7. 카툰풍 감염 학교 콘셉트가 유지되는가
8. 개발자가 실제로 구현 가능한가

장식성은 위 기준을 만족한 뒤에 추가한다.

## 5. 기술 전제

그래픽 디자인 에이전트는 아래 기술 전제를 반드시 알고 있어야 한다.

### Phaser 4가 담당하는 것

- 배경 타일맵
- 학교 부지 월드
- UI
- 투사체
- 2D 이펙트
- XP 오브와 아이템
- 충돌 판정
- 카메라
- 레이어 순서

### Three.js가 담당하는 것

- 플레이어 3D 모델
- 감염체 3D 모델
- 카툰 렌더링
- 외곽선
- 캐릭터 애니메이션
- Phaser 좌표와 시각 동기화

### 중요한 제한

- Three.js 모델은 충돌 판정의 기준이 아니다.
- 충돌 판정은 Phaser의 단순한 원형 또는 캡슐형 히트박스로 처리한다.
- Three.js는 캐릭터를 보여주는 역할에 집중한다.
- 게임 로직과 판정의 진실은 Phaser에 둔다.

## 6. 카툰 렌더링 기준

플레이어와 감염체 캐릭터는 Three.js 카툰 렌더링을 사용한다.

필수 기준:

- `MeshToonMaterial` 또는 동등한 카툰 재질 사용
- 굵은 외곽선 적용
- 2~4단계의 단순 명암
- 복잡한 실사 조명 금지
- 둥근 바닥 그림자 우선
- 모바일 화면에서 캐릭터 실루엣이 명확해야 함

외곽선 방향:

- 플레이어 외곽선은 가장 선명해야 한다.
- 보스 외곽선은 크고 무겁게 보이게 한다.
- 소형 감염체는 외곽선을 너무 두껍게 하지 않는다.
- 저사양 모드에서는 먼 감염체 외곽선 품질을 낮출 수 있다.

## 7. 색상 기준

> **공식 팔레트 가이드**: `Graphic_designer/Concept_Rules/color_palette_guide.md` 참조.  
> 모든 색상 결정은 해당 문서의 역할별 색상 배정표를 따른다.  
> 근거 리소스: `graphic_resource_research_v1.md`

기준 팔레트: **Survival Horror 32** (Lospec) + Poison 보조

색상은 게임 정보 구분에 직접 연결되어야 한다.

권장 색상 역할:

- 플레이어 포인트: 하늘색 `#59c7ff` (유일한 선명한 강조색)
- 감염 일반 적: 탁한 녹색 `#95bf91` 계열
- 감염 엘리트: 보라 `#6f779e` 계열
- 위험 경고: 주황 `#e99039`, 붉은색 `#d32836`
- XP 오브: 밝은 녹색 `#81b071` (Poison 팔레트)
- 회복 오브: 살구/크림 `#cea19d`, 십자 마크 필수
- 자석 아이템: 회청 `#96a5bc`
- 바닥 기본색: 나무 크림 `#a99a73` 계열 (현재: 밝은 목재)
- UI 기준색: 크림 `#c9cb9f`, 회청 `#b7c0c7`

색채 설계 3원칙:

1. **베이스는 탁하게, 강조는 하나만** — 플레이어 하늘색이 유일한 선명 강조색
2. **차가운 색 = 안전, 따뜻한 색 = 위험** — Silent Hill 방식
3. **색으로 적 타입 구분** — Three.js 실루엣이 동일하므로 색이 유일한 구분 수단

피해야 할 것:

- XP 오브와 회복 오브가 비슷한 색으로 보이는 것
- 위험 장판과 배경 오염이 구분되지 않는 것
- 감염체와 배경이 같은 톤으로 묻히는 것
- 같은 녹색 계열 적끼리 명도·채도 차이 없이 구분되지 않는 것

## 8. 캐릭터 디자인 기준

### 학생 생존자

- 작은 체형, 큰 머리, 짧은 몸통
- 등에 가방
- 연필이나 임시 학교 장비를 사용하는 느낌
- 교복 또는 학교 생활복 기반
- 이동 방향에 따라 몸이 살짝 기울어지는 도망 느낌
- 화면에서 가장 먼저 보여야 하는 캐릭터

### 감염된 학생

- 기본 적
- 빠르고 약한 느낌
- 흐트러진 교복
- 형광 녹색 눈
- 플레이어와 비슷하거나 약간 작은 크기

### 감염된 체육교사

- 느리고 단단한 중형 적
- 체육복, 호루라기, 넓은 어깨
- 일반 학생 감염체보다 크고 무거운 실루엣

### 소형 감염 학생 떼

- 작고 빠른 무리
- 2~3가지 색 변형으로 다양성 확보
- 디테일보다는 무리 단위 가독성 우선

### 오염 침뱉개

- 원거리 적
- 입이나 손에 오염 덩어리
- 발사 전 초록빛 충전 연출 필요
- 오염 탄은 밝은 테두리로 구분

### 광폭 질주 감염체

- 돌진 적
- 운동부 학생 느낌
- 돌진 전 주황색 직선 예고선 필수
- 예고 중 떨림 연출

### 특수 감염 교사

- 엘리트 적
- 교사 실루엣과 감염된 팔 또는 큰 분필 막대
- 보라색 발밑 링
- 원형 충격파 경고 범위 필수

### 교장 감염체

- 5분 보스
- 교장 정장, 찢어진 넥타이, 큰 그림자
- 일반 적보다 훨씬 큰 실루엣
- 장판, 부채꼴 분사, 내려찍기 예고가 명확해야 한다.

## 9. 무기와 이펙트 기준

### 연필 투척

- 노란 연필 투사체
- 가장 가까운 적 방향으로 날아가는 느낌
- 관통 시 짧은 흰색 속도선

### 책가방 휘두르기

- 플레이어 주변 회전 방어 무기
- 회전 반경이 안전지대처럼 읽혀야 한다.
- 타격 순간 작은 충격 원과 노란 별 이펙트

### 비상벨 충격파

- 붉은 링 또는 파형
- 상하좌우, 이후 8방향 확장
- 비상벨 느낌이 사운드 없이도 보여야 한다.

### 전기 스턴건

- 가까운 적에게 노란 전기 줄
- 연쇄 대상은 얇은 전기선
- 엘리트 대상 추가 피해는 보라색 스파크 사용 가능

## 10. UI 디자인 기준

UI는 학교 방송실, 안내판, 비상 경고판의 느낌을 참고한다.

필수 UI:

- 생존 시간
- 체력바
- 현재 레벨
- 장착 무기 4칸
- 레벨업 3개 선택 카드
- 보스 등장 경고

UI 원칙:

- 모바일 세로 화면에서 큰 글자 사용
- 장식보다 정보 전달 우선
- 선택 가능한 항목은 노란 테두리
- 위험 메시지는 붉은색
- 무기 아이콘은 학교 장비라는 정체성이 보여야 한다.

## 11. 배경과 스테이지 기준

1스테이지는 감염된 학교 운동장과 본관 주변이다.

필수 배경 요소:

- 운동장 바닥
- 흰색 트랙 라인
- 학교 본관 실루엣
- 쓰러진 책상과 의자
- 찢어진 시험지
- 오염된 사물함
- 안전 표지판
- 감염 경고 테이프
- 깨진 창문 그림자
- 오염 웅덩이

배치 원칙:

- 중앙부는 회피 공간으로 비워둔다.
- 이동 방해 요소는 전체의 15퍼센트 이하로 제한한다.
- 완전 막힘 구역은 만들지 않는다.
- 원거리 탄과 장애물이 겹쳐 억울한 피격이 생기지 않게 한다.

## 12. 금지사항

그래픽 디자인 에이전트는 아래 방향을 피해야 한다.

- 실사 공포 중심 비주얼
- 과한 피와 잔혹 표현
- 어두워서 정보가 안 보이는 화면
- 플레이어를 가리는 장시간 이펙트
- 적 피격마다 큰 파티클 폭발
- XP 오브와 회복 오브가 헷갈리는 색상
- 보스 공격이 예고 없이 들어오는 연출
- Three.js가 게임 판정을 담당하는 설계
- 개발자가 구현하기 어려운 과도한 그래픽 요구

## 14. 검수 체크리스트

그래픽 작업을 검수할 때는 아래 질문에 답해야 한다.

- 플레이어가 1초 안에 눈에 들어오는가
- 감염체 종류가 크기와 실루엣으로 구분되는가
- 위험 공격은 맞기 전에 예고되는가
- XP 오브와 회복 오브가 확실히 다른가
- 3분 이후 다수 적 상황에서도 화면이 읽히는가
- Three.js 카툰 렌더링과 외곽선이 적용되어 있는가
- Phaser 레이어와 Three.js 캐릭터 합성이 어색하지 않은가
- 모바일 9:16 화면에서 UI가 겹치지 않는가
- 그래픽 요구사항이 개발자가 구현 가능한 수준인가

## 16. 프로토타입 배경 타일 구현 기록

첫 프로토타입 배경은 Phaser에서 생성하는 2D 도트풍 학교 타일로 구현한다.

구현된 타일 5종:

- `tile_school_interior_floor`: 사용자가 지정한 밝은 나무 마루 레퍼런스를 따른 통일형 학교 교실 바닥 타일

디자인 의도:

- 캐릭터와 몬스터는 Three.js 카툰 렌더링을 유지한다.
- 배경은 Phaser 2D 타일로 처리한다.
- 학교임을 연상할 수 있도록 운동장 선, 복도 타일, 사물함색, 시험지/경고 테이프 느낌을 넣는다.
- 도트풍을 위해 16px 단위의 미묘한 색 차이와 격자감을 사용한다.
- 배경은 플레이어, 감염체, XP 오브, 회복 오브보다 낮은 대비로 유지한다.

## 17. 프로토타입 학교 기물 구현 기록

학교 내부와 운동장 주변에 널린 기물은 명확한 2D 도트 그래픽 오브젝트로 구현한다.

게임 내 구현 텍스처:

- `prop_fallen_desk_pixel`: 쓰러진 책상
- `prop_chair_pile_pixel`: 의자 더미
- `prop_locker_pixel`: 오염된 사물함
- `prop_warning_sign_pixel`: 감염 경고 표지판
- `prop_barricade_pixel`: 임시 바리케이드
- `prop_exam_papers_pixel`: 흩어진 시험지
- `prop_window_shadow_pixel`: 깨진 창문 그림자

기본 리소스 원본 위치:

- `Graphic_designer/pixel_prop_assets/`

구현 원칙:

- 모든 기물은 Phaser 2D 레이어에서 렌더링한다.
- 플레이어와 감염체는 계속 Three.js 카툰 렌더링으로 유지한다.
- 기물은 학교임을 즉시 연상할 수 있어야 한다.
- 충돌이 있는 기물은 게임 코드에서 단순 `radius` 판정으로 관리한다.
- 기물이 전투 정보를 가리지 않도록 플레이어보다 낮은 시각 우선순위를 유지한다.
