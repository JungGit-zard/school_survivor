# Stage 1 Reference Image Implementation Plan

작성일: 2026-05-24  
대상: 첨부 이미지 기반 1스테이지 마룻바닥 배경 그래픽 구현안  
역할: Graphic Designer / Developer handoff  
상태: 구현 방안

## 1. 목표

첨부 이미지의 느낌을 `Escape! zombie school` 1스테이지 플레이 화면에 최대한 가깝게 옮긴다.

핵심은 아래 화면 인상이다.

```text
세로형 모바일 화면
낡은 나무 마룻바닥
중앙은 넓고 비어 있는 플레이 공간
외곽에는 어두운 교실 소품과 오염 흔적
플레이어, 좀비, 드랍, HUD는 배경보다 먼저 읽힘
```

## 2. 첨부 이미지에서 가져올 요소

### 2-1. 전체 화면 구도

- 모바일 세로 화면을 기준으로 한다.
- 카메라는 위에서 아래를 내려다보는 top-down 구도다.
- 중앙 세로축은 플레이어가 움직이는 주 통로처럼 넓게 비워 둔다.
- 소품은 현재 보이는 화면 테두리가 아니라 스테이지 전체의 좌우/상하 외곽 영역에 드문드문 배치한다.
- 카메라가 이동하면서 외곽 영역을 지나갈 때 책상, 의자, 사물함, 종이, 오염 흔적이 간헐적으로 보이게 한다.
- 화면 가장자리의 어두운 분위기는 별도 오버레이로 처리하고, 소품 배치 기준과 혼동하지 않는다.
- 중앙 마룻바닥은 비교적 읽기 쉽다.

### 2-2. 바닥

- 바닥은 넓은 나무 마루 판자다.
- 판자 방향은 화면 세로 진행 방향과 잘 맞게 길게 흐른다.
- 판자 이음새는 보이지만 너무 강한 격자처럼 보이면 안 된다.
- 색은 너무 노랗거나 밝지 않고, `낡은 갈색 + 탁한 회색`에 가깝다.
- 작은 금, 흠집, 먼지 점을 넣어 낡은 학교 느낌을 준다.

### 2-3. 외곽 소품

첨부 이미지처럼 외곽에 배치할 소품:

- 책상
- 의자
- 사물함 또는 캐비닛
- 책과 노트 더미
- 쓰러진 작은 가구
- 어두운 오염 웅덩이
- 찢어진 종이
- 화분 또는 작은 교실 장식
- 경고 테이프

단, 게임플레이를 방해하면 안 되므로 중앙에는 두지 않는다.

### 2-4. 명암

- 중앙 마룻바닥은 플레이어가 잘 보이도록 너무 어둡게 하지 않는다.
- 좌우 외곽과 모서리는 어둡게 눌러서 폐교/감염 분위기를 만든다.
- 창문 그림자처럼 사선으로 들어오는 빛을 일부 넣을 수 있다.
- 오염 웅덩이는 검은색에 가까운 어두운 녹색으로 둔다.

## 3. 현재 프로젝트에 맞춘 구현 전략

현재 구조를 크게 바꾸지 않고 아래 파일들을 조정한다.

### 3-1. 바닥 텍스처 조정

대상:

- `Developer/r3f_prototype/src/components/ClassroomFloor.jsx`

변경 방향:

- 현재 procedural CanvasTexture 방식을 유지한다.
- 텍스처 크기는 유지해도 되지만 판자 느낌을 더 명확히 한다.
- 판자 이음새를 세로 방향으로 더 길게 보이게 조정한다.
- 너무 밝은 노이즈는 줄이고, 중앙은 읽기 쉬운 갈색 계열로 둔다.
- 첨부 이미지처럼 `큰 나무판 여러 장이 길게 깔린 느낌`을 우선한다.

추천 색:

| 역할 | 색 |
|---|---|
| 기본 마루 | `#8f7a56` |
| 밝은 마모 흔적 | `#b7aa7a` |
| 어두운 판자 | `#5f4a35` |
| 이음새 | `#2f241f` |
| 오염 얼룩 | `#18211d` |

기존 정본 색보다 조금 더 어둡고 영화적인 쪽이다. 단, 플레이어가 묻히면 다시 밝힌다.

### 3-2. 외곽 소품 밀도 조정

대상:

- `Developer/r3f_prototype/src/lib/stagePropsLayout.js`
- `Developer/r3f_prototype/src/components/StageProps.jsx`
- `Developer/r3f_prototype/src/components/Props/*`
- `Developer/r3f_prototype/src/components/Atmosphere/*`

변경 방향:

- 기존 외곽 6~12 블록 링 정책은 유지하되, 기준은 화면 테두리가 아니라 스테이지 전체 월드 좌표다.
- 첨부 이미지의 느낌은 "보이는 화면 가장자리를 장식한다"가 아니라 "스테이지 외곽을 이동할 때 교실 소품이 드문드문 발견된다"로 해석한다.
- 단, 중앙 통로는 계속 비운다.
- 소품 수는 늘리되, 충돌 있는 소품은 조심해서 배치한다.
- 시각 전용 종이/그림자/오염은 충돌 없이 더 많이 배치해도 된다.

권장 배치:

- 상단 좌측: 책상, 책 더미, 어두운 벽 그림자
- 상단 우측: 쓰러진 의자, 오염 웅덩이
- 중단 좌측: 작은 책상, 화분, 종이
- 중단 우측: 교탁/테이블 느낌의 소품, 책과 노트
- 하단 좌우: 오염 웅덩이와 종이, 부서진 가구 일부

### 3-3. 어두운 가장자리 분위기

대상 후보:

- `StageProps` 또는 별도 `StageVignette` 컴포넌트
- `Atmosphere/WindowShadowBroken.jsx`
- 새 `Atmosphere/EdgeShadow.jsx`

구현 방향:

- 화면 외곽 소품이 실제로 어둡게 느껴지도록 큰 반투명 그림자 패치를 둔다.
- 카메라에 따라 화면을 따라다니는 UI 오버레이가 아니라, 바닥에 깔린 정적 그림자로 처리한다.
- 중앙 플레이 공간까지 어둡게 덮으면 안 된다.

### 3-4. HUD와 배경의 관계

첨부 이미지는 HUD가 상단과 하단에 강하게 고정되어 있다.

현재 작업의 1차 범위는 배경이므로 HUD는 크게 건드리지 않는다.

다만 배경 구현 후 확인할 것:

- 상단 타이머와 레벨이 배경에 묻히지 않는가
- 하단 HP/XP 영역이 소품과 겹치지 않는가
- 모바일 9:16에서 스테이지 외곽 구역 소품이 화면에 들어왔을 때 HUD를 방해하지 않는가

## 4. 구현 순서

### 1단계: 바닥 톤 맞추기

`ClassroomFloor.jsx`의 procedural texture를 첨부 이미지처럼 더 큰 판자, 더 어두운 갈색, 더 약한 노이즈로 조정한다.

완료 기준:

- 회색 격자처럼 보이지 않는다.
- 밝은 노란 마루처럼 보이지 않는다.
- 넓은 낡은 교실 나무 바닥으로 보인다.

### 2단계: 외곽 소품 배치 조정

`stagePropsLayout.js`에서 화면 기준이 아니라 스테이지 전체 월드 좌표 기준으로 외곽 소품을 재배치한다.
플레이어가 이동하며 외곽 구역을 지나갈 때 책상, 의자, 사물함, 종이, 오염 흔적이 드문드문 나타나야 한다.

완료 기준:

- 중앙 이동 공간은 비어 있다.
- 외곽은 버려진 교실처럼 느껴지되, 모든 가장자리를 빽빽하게 채우지 않는다.
- 이동 중 외곽 구역에서 소품이 간헐적으로 발견된다.
- 플레이어가 소품에 갇히지 않는다.

### 3단계: 종이, 오염, 그림자 추가

충돌 없는 분위기 오버레이를 더 촘촘하게 추가한다.

완료 기준:

- 첨부 이미지처럼 바닥에 종이와 얼룩이 흩어져 있다.
- 보스 장판과 정적 오염이 헷갈리지 않는다.
- 배경이 투사체나 드랍을 가리지 않는다.

### 4단계: 모바일 화면 검증

g-stack 또는 브라우저로 375×667, 390×844 화면을 캡처한다.

확인할 것:

- 플레이어가 1초 안에 보이는가
- 좀비가 배경과 구분되는가
- 바닥이 너무 복잡하지 않은가
- 소품이 HUD를 가리지 않는가
- 3~4분 적이 많을 때도 가독성이 유지되는가

## 5. 하지 않을 것

- 첨부 이미지를 그대로 2D 배경 이미지로 깔지 않는다.
- 캐릭터와 좀비를 2D 스프라이트로 바꾸지 않는다.
- 중앙에 큰 책상이나 장애물을 놓지 않는다.
- 배경을 너무 어둡게 만들어 플레이어를 묻히게 하지 않는다.
- 피, 살점, 잔혹 표현으로 분위기를 만들지 않는다.
- HUD 전체 개편은 이번 작업에 포함하지 않는다.

## 6. 구현 난이도

낮음:

- 바닥 색상과 노이즈 조정
- 종이/오염/그림자 오버레이 추가

중간:

- 외곽 소품 재배치
- 충돌 소품이 길을 막지 않게 조정
- 모바일 화면에서 가독성 유지

높음:

- 첨부 이미지처럼 풍부한 교실 소품을 많이 넣으면서 성능까지 유지
- 모든 소품을 고품질 3D 카툰 오브젝트로 만드는 작업

## 7. 추천 MVP

1차 MVP는 아래까지만 한다.

```text
어두운 낡은 나무 마루
+ 중앙 통로 비움
+ 외곽 책상/의자/사물함/오염 웅덩이
+ 바닥 종이와 창문 그림자
+ 모바일 9:16 캡처 검증
```

이 정도면 첨부 이미지의 핵심 분위기를 가져오면서도 현재 게임의 3D 카툰 정책과 플레이 가독성을 지킬 수 있다.

## 8. 최종 요약

첨부 이미지를 그대로 복사하는 것이 아니라, 현재 R3F 게임 구조 안에서 다음 느낌을 구현한다.

```text
중앙은 플레이 가능한 낡은 마룻바닥
스테이지 전체 외곽은 버려진 교실 소품과 감염 흔적
소품은 이동 중 드문드문 발견됨
화면 가장자리는 별도 오버레이로 어둡고 위험하게 느껴짐
플레이어와 좀비는 항상 배경보다 먼저 보임
```

즉, `배경이 멋진 화면`이 아니라 `게임이 학교 좀비 탈출 게임처럼 보이게 만드는 배경`을 만든다.
