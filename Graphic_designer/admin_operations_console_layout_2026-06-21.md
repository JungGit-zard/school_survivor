# Admin Operations Console Layout

> 작성일: 2026-06-21  
> 역할: 그래픽/레이아웃 기록  
> 상태: 구현 기준 레이아웃

## 1. 시각 방향

- 마케팅 페이지가 아니라 반복 조작용 운영 도구로 보이게 한다.
- 카드 장식을 과하게 늘리지 않고, 입력 폼과 미리보기 정보를 빠르게 스캔할 수 있게 한다.
- 두 흐름은 탭으로 분리한다.
- 위험한 수치 변경은 노란색/붉은색 계열 경고가 아니라 설명 문장과 범위 표시로 차분하게 안내한다.

## 2. 와이어프레임

```text
┌──────────────────────────────────────────────────────────────┐
│ Escape Admin Console                         [게임으로 돌아가기] │
│ 운영 설정은 저장 즉시 새 게임 시작/랭킹 화면에 반영됩니다.        │
├──────────────────────────────────────────────────────────────┤
│ [게임 밸런스] [랭킹/시즌]                                      │
├────────────────────────────────────────────┬─────────────────┤
│ 탭별 운영 입력 폼                           │ 적용 미리보기    │
│                                            │ 저장 상태        │
│ - Stage 1 duration                         │ Stage clear 예시 │
│ - Stage 2 duration                         │ HP/속도 예시     │
│ - HP bonus                                 │ 보상 요약        │
│ - Speed multiplier                         │                 │
│ - Gold multiplier                          │ [저장] [복원]    │
├────────────────────────────────────────────┴─────────────────┤
│ 하단: 현재 설정 JSON 요약 또는 운영 주의문                      │
└──────────────────────────────────────────────────────────────┘
```

## 3. 레이아웃 규칙

- 데스크톱은 2열: 입력 영역 1fr, 미리보기 340px.
- 모바일 폭에서는 1열로 내려가며 탭과 저장 버튼이 먼저 보인다.
- 탭 버튼은 큰 텍스트 박스처럼 만들지 않고, 현재 선택 상태가 분명한 segmented control 형태로 만든다.
- 입력은 숫자 필드, 선택은 select, 보상 티어는 작은 테이블 형태로 구성한다.

## 4. 색상/타이포

- 배경: 짙은 중립색.
- 주요 포인트: 민트, 골드, 코랄을 보조적으로 사용.
- 단일 보라/슬레이트 계열로만 보이지 않게 보상/저장/주의 영역의 색을 분리한다.
- 운영 도구 내부 글자는 작은 화면에서도 잘리지 않도록 줄바꿈 가능한 라벨을 사용한다.
